####
Definición del entorno vacío y la estructura de extensión para un sistema de gestión de ambientes:

```racket
(define empty-env
  (lambda () (list 'empty-env)))

(define extend-env
  (lambda (id value old-env)
    (list 'extend-env id value old-env)))
```

**Conceptos clave**
`ambiente`, `entorno vacío`, `extensión de entorno`, `listas`

**Aplicación**
`empty-env` crea el estado inicial sin asignaciones, mientras que `extend-env` permite añadir un nuevo par identificador-valor a un ambiente existente, manteniendo la estructura necesaria para búsquedas posteriores.

####
Predicados para identificar el tipo de estructura de un ambiente:

```racket
(define empty-env?
  (lambda (e)
    (equal? (car e) 'empty-env)))

(define extend-env?
  (lambda (e)
    (equal? (car e) 'extend-env)))
```

**Conceptos clave**
`predicados`, `car`, `condicionales`, `tipado dinámico`

**Aplicación**
Estos predicados permiten verificar en tiempo de ejecución si un objeto de entorno es de tipo `empty-env` o `extend-env`, facilitando la lógica de despacho en las funciones que operan sobre el ambiente.

####
Funciones de acceso para extraer componentes de un entorno extendido:

```racket
(define extend-env->id
  (lambda (e)
    (cadr e)))

(define extend-env->value
  (lambda (e)
    (caddr e)))

(define extend-env->old-env
  (lambda (e)
    (cadddr e)))
```

**Conceptos clave**
`selectores`, `cadr`, `caddr`, `cadddr`

**Aplicación**
Se utilizan para acceder de forma modular a los campos internos de un nodo de entorno (`id`, `value` y `old-env`), permitiendo que el resto del sistema no dependa de la representación exacta de la lista.

####
Construcción de un ambiente complejo mediante la anidación de extensiones:

```racket
(define e
  (extend-env 'd 6
              (extend-env 'y 8
                          (extend-env 'x 7
                                      (extend-env 'y 14
                                                  (empty-env))))))
```

**Conceptos clave**
`anidamiento`, `entorno`, `ligaduras`

**Aplicación**
Representa la creación de un espacio de nombres con múltiples asignaciones, donde las variables pueden estar sombreadas (como ocurre con 'y' en este ejemplo) según su posición en la cadena de ambientes.

####
Implementación de la búsqueda de valores en un entorno:

```racket
(define apply-env
  (lambda (e var)
    (cond
      [(empty-env? e) (eopl:error "La variable" var "no ha sido encontrada")]
      [(extend-env? e)
       (if (equal? (extend-env->id e) var)
           (extend-env->value e)
           (apply-env (extend-env->old-env e) var))]
      [else (eopl:error "Estructura no válida")]
      )))
```

**Conceptos clave**
`búsqueda recursiva`, `cond`, `manejo de errores`, `scope léxico`

**Aplicación**
Esta función recorre recursivamente el entorno desde el último elemento añadido hacia atrás. Si encuentra el identificador, retorna su valor asociado; de lo contrario, delega la búsqueda en el entorno anterior hasta alcanzar el `empty-env`.

####
Ejecución y visualización de consultas al ambiente:

```racket
(display (apply-env e 'x))
(display "\n")
(display (apply-env e 'y))
```

**Conceptos clave**
`interfaz de salida`, `display`, `aplicación de funciones`

**Aplicación**
Demuestra la recuperación efectiva de valores desde el ambiente `e` definido previamente, permitiendo verificar que la lógica de búsqueda maneja correctamente tanto variables únicas como variables sombreadas.
