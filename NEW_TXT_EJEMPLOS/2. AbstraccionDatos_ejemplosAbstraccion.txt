####
Definición de una interfaz de números naturales representada mediante listas:

```racket
(define zero '())
(define iszero? null?)
(define succ (lambda (rep_n) (cons #t rep_n)))
(define pred cdr)
```

**Conceptos clave**
`listas`, `abstracción de datos`, `predicados`, `constructores`

**Aplicación**
Esta interfaz define números naturales utilizando una representación basada en listas donde `zero` es la lista vacía y el sucesor (`succ`) añade un elemento a la estructura. Se utiliza para modelar aritmética básica sobre estructuras de datos.

####
Implementación de las operaciones aritméticas `plus` y `prod` utilizando recursión:

```racket
(define plus
  (lambda (x y)
    (if (iszero? x)
        y
        (succ (plus (pred x) y)))))

(define prod
  (lambda (x y)
    (if (iszero? x)
        zero
        (if (eqv? x (succ zero))
            y
            (plus y (prod (pred x) y))))))
```

**Conceptos clave**
`recursión`, `aritmética`, `abstracción de datos`, `funciones`

**Aplicación**
`plus` suma dos números naturales de forma recursiva disminuyendo el primer operando. `prod` realiza la multiplicación mediante sumas sucesivas, utilizando las funciones base definidas anteriormente.

####
Definición y procesamiento de árboles binarios (`arbin`):

```racket
;; <arbin> ::= <numero>
;; <arbin> ::= (<simbolo> <arbin> <arbin>)

(define cuantos
  (lambda (arbin)
    (if (number? arbin)
        1
        (+ 1 (cuantos (cadr arbin)) (cuantos (caddr arbin))))))
```

**Conceptos clave**
`BNF`, `árboles binarios`, `recursión estructural`, `procesamiento de listas`

**Aplicación**
La función `cuantos` calcula el número total de nodos en un árbol binario. Si el elemento es un número, se cuenta como una hoja (1 nodo), de lo contrario, suma los nodos de sus dos subárboles más el nodo actual.

####
Sustitución de símbolos en listas (`subst`):

```racket
(define subst
  (lambda (listas old new)
    (if (null? listas)
        '()
        (cons (subst-exp-sim (car listas) old new) 
              (subst (cdr listas) old new)))))

(define subst-exp-sim
  (lambda (exp-sim old new)
    (if (symbol? exp-sim)
        (if (eqv? exp-sim old) new exp-sim)
        (subst exp-sim old new))))
```

**Conceptos clave**
`sustitución`, `listas simbólicas`, `procesamiento recursivo`, `comparación de símbolos`

**Aplicación**
Esta función recorre una lista y reemplaza todas las ocurrencias de un símbolo `old` por `new`. `subst-exp-sim` maneja la lógica de validación para identificar si el elemento actual es un símbolo o una lista anidada.

####
Definición de tipos de datos complejos con `define-datatype`:

```racket
(define-datatype arbin arbin?
  (hoja (n number?))
  (nodo (letra symbol?)
        (izq arbin?)
        (der arbin?)))
```

**Conceptos clave**
`define-datatype`, `tipos de datos algebraicos`, `constructores`, `predicados`

**Aplicación**
Permite definir estructuras de datos personalizadas con validación automática de tipos. Genera automáticamente los constructores (`hoja`, `nodo`) y el predicado (`arbin?`) necesarios para manipular la estructura en el programa.

####
Uso de la macro `cases` para el procesamiento de tipos de datos:

```racket
(define cuantos-cases
  (lambda (un-arbin)
    (cases arbin un-arbin
      (hoja (n) 1)
      (nodo (le izq der) (+ 1 (cuantos-cases izq) (cuantos-cases der))))))
```

**Conceptos clave**
`cases`, `desestructuración`, `patrones`, `tipos de datos`

**Aplicación**
La macro `cases` facilita la selección de la variante de un tipo de dato y la extracción de sus campos. Es la forma estándar en EOPL para implementar funciones que operan sobre tipos definidos mediante `define-datatype`.

####
Representación de expresiones lambda mediante tipos de datos:

```racket
(define-datatype expression expression? 
  (var-exp (id symbol?))
  (lambda-exp (id symbol?)
              (body expression?))
  (app-exp (rator expression?)
           (rand expression?)))
```

**Conceptos clave**
`cálculo lambda`, `árbol sintáctico abstracto`, `abstracción`, `aplicación`

**Aplicación**
Define la gramática de una expresión lambda, permitiendo modelar variables (`var-exp`), abstracciones (`lambda-exp`) y aplicaciones de funciones (`app-exp`) dentro del sistema.

####
Análisis de variables libres y representación de expresiones lambda:

```racket
(define occurs-free?
  (lambda (var exp)
    (cases expression exp
      (var-exp (id) (eqv? id var))
      (lambda-exp (id body)
        (and (not (eqv? id var))
             (occurs-free? var body)))
      (app-exp (rator rand)
        (or (occurs-free? var rator)
            (occurs-free? var rand))))))
```

**Conceptos clave**
`variables libres`, `cálculo lambda`, `recursión`, `álgebra booleana`

**Aplicación**
Determina si una variable ocurre de forma libre en una expresión lambda, siguiendo las reglas del cálculo lambda: ignora la variable si está ligada por un `lambda`, y busca recursivamente en los componentes de las aplicaciones.
