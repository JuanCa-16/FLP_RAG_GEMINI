####
Definición de las operaciones básicas para representar números naturales en el lenguaje `eopl`:

```racket
#lang eopl

(define zero 0)

(define is-zero?
  (lambda (n) 
    (= n 0)))

(define successor
  (lambda (n) 
    (+ n 1)))

(define predecessor
  (lambda (n) 
    (if (is-zero? n)
        (eopl:error "Cero no tiene predecesor" "predecessor")
        (- n 1))))
```

**Conceptos clave**
`abstracción de datos`, `predicados`, `sucesores`, `manejo de errores`, `aritmética básica`

**Aplicación**
Este conjunto de funciones define la interfaz fundamental para manipular números naturales. El predicado `is-zero?` determina la base de la recursión, mientras que `successor` y `predecessor` permiten la transición entre estados numéricos, incluyendo una validación para evitar errores en el dominio de los números naturales.

####
Implementación recursiva de la operación de suma utilizando las funciones primitivas:

```racket
(define plus
  (lambda (a b)
    (cond
      [(is-zero? b) a]
      [else
       (plus (successor a) (predecessor b))])))
```

**Conceptos clave**
`recursión`, `condicionales`, `reducción de parámetros`, `invariante`

**Aplicación**
La función `plus` resuelve la suma mediante un proceso recursivo que transfiere unidades del segundo operando al primero hasta que `b` alcanza el valor `zero`. Es un ejemplo clásico de cómo construir operaciones complejas a partir de primitivas básicas.

####
Implementación de la operación de multiplicación mediante recursión y suma acumulada:

```racket
(define mult
  (lambda (a b)
    (if (is-zero? b)
        zero
        (plus a (mult a (predecessor b))))))
```

**Conceptos clave**
`recursión estructural`, `operadores aritméticos`, `casos base`, `llamadas recursivas`

**Aplicación**
Esta función ilustra la multiplicación como la adición repetida del valor `a` tantas veces como indique `b`. Utiliza la función `plus` previamente definida para combinar los resultados parciales en cada paso de la recursión.

####
Ejecución de una operación aritmética utilizando las funciones implementadas:

```racket
(mult 10 2000)
```

**Conceptos clave**
`evaluación`, `expresiones`, `parámetros`, `procesamiento numérico`

**Aplicación**
Esta expresión invoca el procedimiento `mult` para realizar el cálculo del producto de 10 por 2000, demostrando la operatividad del conjunto de funciones definido anteriormente dentro del entorno `eopl`.
