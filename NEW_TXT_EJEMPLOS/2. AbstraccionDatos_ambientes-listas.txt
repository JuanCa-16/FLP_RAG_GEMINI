####
Definición del entorno vacío mediante `empty-env`:

```racket
(define empty-env
  (lambda () (list 'empty-env)))
```

**Conceptos clave**
`entornos`, `listas`, `representación simbólica`, `constructores`

**Aplicación**
Esta función actúa como el caso base para representar un entorno sin vinculaciones. Se utiliza para inicializar una estructura de datos de entorno vacía al comenzar la evaluación de un programa.

####
Extensión de un entorno existente mediante `extend-env`:

```racket
(define extend-env
  (lambda (var val env)
    (list 'extend-env var val env)))
```

**Conceptos clave**
`vinculaciones`, `entornos`, `abstracción de datos`, `estructuras de datos`

**Aplicación**
Esta función permite añadir una nueva asociación entre un identificador (`var`) y su valor (`val`) a un entorno existente (`env`), creando una nueva estructura que envuelve el entorno anterior.

####
Búsqueda de valores en un entorno mediante `apply-env`:

```racket
(define apply-env
  (lambda (env search-var)
    (cond ((eqv? (car env) 'empty-env)
           (eopl:error 'apply-env "No binding for ~s" search-var))
          ((eqv? (car env) 'extend-env)
           (let ((saved-var (cadr env))
                 (saved-val (caddr env))
                 (saved-env (cadddr env)))
             (if (eqv? search-var saved-var)
                 saved-val
                 (apply-env saved-env search-var))))
          (else (eopl:error 'apply-env "Expecting an environment, given ~s" env)))))
```

**Conceptos clave**
`recursión`, `búsqueda`, `predicados`, `manejo de errores`, `entornos léxicos`

**Aplicación**
Resuelve la búsqueda de un identificador en el entorno. Si el identificador coincide con la variable guardada, retorna su valor; de lo contrario, realiza una llamada recursiva sobre el entorno almacenado (`saved-env`). Si se alcanza el entorno vacío, lanza un error indicando que no existe una vinculación para la variable.
