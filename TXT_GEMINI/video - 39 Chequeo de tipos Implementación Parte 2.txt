### Implementación del Chequeador de Tipos

Se procederá a implementar el chequeador de tipos, comenzando por la definición de `SPAND_TYPE_EXPRESSION`.

#### 1. Procedimiento `SPAND_TYPE_EXPRESSION`

Este procedimiento calcula el tipo denotado por una expresión de tipo. El revisor de tipos lo invocará cada vez que requiera el tipo de una expresión.

*   **Entrada:** Una expresión de tipo (`TYPE_EXPRESSION_EXP`), definida en la especificación gramatical.
*   **Casos:**
    *   Si la expresión es un `INT_EXPRESSION`, retorna `INT_TYPE`. Esto vincula la definición gramatical con la representación interna de tipos.
    *   Si la expresión es un `BULT_TYPE_EXPRESSION`, retorna `BULT_TYPE`.
    *   Para un `PROT_TYPE_EXPRESSION`, que contiene los argumentos (`ARG_TEST`) y el tipo del resultado (`RESULT_T_EXPRESSION`), se construye un `PROT_TYPE`. Esto implica el cálculo recursivo de los tipos: se utiliza `SPAND_TYPE_EXPRESSIONS` para la lista de argumentos y `SPAND_TYPE_EXPRESSION` para el tipo del resultado.

#### 2. Procedimiento `SPAND_TYPE_EXPRESSIONS`

Esta función auxiliar procesa una lista de expresiones de tipo para obtener una lista de tipos.

*   **Entrada:** Una lista de expresiones (ej. `ARG_TEST`).
*   **Funcionamiento:** Aplica la función `SPAND_TYPE_EXPRESSION` (la versión singular) a cada elemento de la lista, utilizando una operación de mapeo (ej. `MAP` con una `LAMBDA`).

Estas funciones constituyen la primera parte de la interfaz para la representación de tipos, mapeando cada tipo a su `data type` correspondiente para la verificación.

### Implementación del Revisor de Tipos (`CHECKER`)

El revisor de tipos (`checker`) se implementará como un procedimiento que recibe una expresión y un ambiente de tipos, retornando el tipo correspondiente o un error. Este revisor se denomina `TAVIO_EXPRESSION` y contendrá cláusulas que implementan las reglas de producción para expresiones tipadas, asociando un tipo a cada expresión de la gramática.

#### 1. Definición Inicial de `TAVIO_EXPRESSION`

`TAVIO_EXPRESSION` recibe una expresión (`EXPRESSION_ES`) y un ambiente de tipos. Se respetarán los nombres definidos previamente para los casos de evaluación.

*   **Casos Iniciales (Literales y Variables):**
    *   Para `LITERAL_EXPRESSION`:
        *   Si es un literal numérico, retorna `INT_TYPE`.
        *   Si es un literal booleano, retorna `BULT_TYPE`.
    *   Para `VAR_EXPRESSION`: Se invoca `APPLY_TM`, un procedimiento similar a `TUEM` pero con un mensaje de error específico para tipos no encontrados o inconsistencias de tipo.

#### 2. Manejo de Expresiones Condicionales (`IF_EXPRESSION`)

El siguiente caso a implementar son las expresiones condicionales (`IF_EXPRESSION`), que reciben `TEST_EXPRESSION`, `TRUE_EXPRESSION`, y `FALSE_EXPRESSION`.

*   **Pasos:**
    1.  Se calculan los tipos de cada componente:
        *   `type_test` para `TEST_EXPRESSION`.
        *   `type_true` para `TRUE_EXPRESSION`.
        *   `type_false` para `FALSE_EXPRESSION`.
        Esto se realiza utilizando `TAVIO_EXPRESSION` en el ambiente actual, almacenando los resultados en una estructura `LET`.
    2.  **Verificación de Reglas de Tipo:**
        *   `type_test` debe ser `BULT_TYPE`.
        *   `type_true` debe ser igual a `type_false`.
        Estas verificaciones se realizan utilizando la función `CHECK_EQUAL_TYPE`, la cual recibe la expresión original para generar mensajes de error contextualizados.

#### 3. Función Auxiliar `CHECK_EQUAL_TYPE`

Esta función compara dos tipos y genera un error si no son iguales.

*   **Definición:** `CHECK_EQUAL_TYPE(T1, T2, S)`
    *   **Lógica:** Si `T1` y `T2` no son iguales (`NOT EQUAL T1 T2`), se genera un error `EOPLERROR` con un mensaje indicando que "los tipos no corresponden" (`S != S`). Los tipos (`T1`, `T2`) y la expresión (`S`) se formatean para el mensaje de error utilizando `TYPE_EXTERNAL_FORM`.

#### 4. Función Auxiliar `TYPE_EXTERNAL_FORM`

Esta función formatea la representación interna de los tipos a una cadena legible para los mensajes de error.

*   **Definición:** `TYPE_EXTERNAL_FORM(T)`
    *   **Casos:**
        *   Si `T` es un `ATOMIC_TYPE`, retorna su nombre.
        *   Si `T` es un `PROT_TYPE`, transforma la lista de tipos de argumentos y el tipo de resultado en una cadena con un formato específico (ej. `INT * BOOL -> INT`), calculando recursivamente los formatos externos de sus componentes.

Con esto se concluye la implementación de los casos iniciales. A continuación, se abordará la implementación de procedimientos, aplicación de procedimientos, `LET` y `LETREC`.
