En nuestro lenguaje de programación, definiremos los tipos de datos básicos, que incluyen enteros (`intype`) y booleanos (`booltype`), así como tipos de procedimiento. Un tipo de procedimiento se compone de los tipos de sus argumentos y el tipo de su valor de retorno. Cada vez que se evalúa una expresión en el intérprete, esta tendrá una expresión de tipo asociada que la clasifica como entero, booleano o procedimiento.

####

La relación entre un valor expresado `v` y su tipo `t` se define de manera inductiva. Un valor expresado es de tipo entero si y solo si es un número entero. Es de tipo booleano si y solo si es un valor booleano. Finalmente, un valor expresado es de tipo procedimiento `(T1 * ... * Tn) -> T_out` si y solo si es un `procval` que requiere exactamente `n` argumentos de tipos `T1` a `Tn` y, al ser evaluado con ellos, retorna un valor de tipo `T_out`.

####

Este lenguaje es de tipado estático, lo que significa que cada valor expresado tiene un único tipo asignado que no puede cambiar durante la ejecución. Sin embargo, determinar el tipo de un valor en tiempo de compilación presenta un desafío, especialmente para el valor de retorno de un procedimiento. Para abordar esto, se utilizan dos estrategias principales: el chequeo de tipos (`type checking`), donde los tipos se especifican explícitamente, y la inferencia de tipos (`type inference`), donde el sistema deduce los tipos automáticamente.

####

A continuación se analizan las expresiones de tipo para diferentes procedimientos:
-   La función `is-even?` determina si un número es par. Recibe un entero y retorna un booleano, por lo que su tipo es `(int -> bool)`.
-   La operación de suma `+` recibe dos enteros y retorna un entero. Su tipo es `(int * int -> int)`.
-   La expresión `lambda x. lambda y. x + y` tiene el tipo `(int -> (int -> int))`. Se infiere que `x` e `y` deben ser enteros porque participan en una suma. El procedimiento externo recibe un entero `x` y retorna un nuevo procedimiento. Este procedimiento interno recibe un entero `y` y retorna el resultado de la suma, que es un entero.

####

El tipo de un procedimiento se distingue del tipo de su evaluación. Por ejemplo, la función `+` tiene el tipo `(int * int -> int)`, que describe una operación que espera dos enteros. En cambio, la expresión `+ 5 2` es la *evaluación* de dicha función, y su resultado, `7`, es de tipo `int`. Es crucial diferenciar el tipo de la función en sí misma del tipo del valor que retorna al ser evaluada.

####

Para la expresión `lambda f. lambda x. is-even?(f (x + 1))`, el tipo es `((int -> int) -> (int -> bool))`. El análisis es el siguiente:
1.  La operación `x + 1` implica que `x` es de tipo `int`, y el resultado también es `int`.
2.  La función `f` se aplica a `(x + 1)`, por lo tanto, `f` debe aceptar un argumento de tipo `int`.
3.  El resultado de `f` se pasa a `is-even?`, que espera un `int`. Por lo tanto, `f` debe retornar un valor de tipo `int`. Concluimos que el tipo de `f` es `(int -> int)`.
4.  La función `is-even?` retorna un `bool`.
5.  El procedimiento completo recibe `f` (de tipo `(int -> int)`) y `x` (de tipo `int`), y retorna el resultado final, que es `bool`.

####

El tipo de la expresión `proc (x) (x 26)` es `((int -> Tx) -> Tx)`. En este caso, `x` es un procedimiento que se aplica al entero `26`. Por lo tanto, `x` debe ser un procedimiento que recibe un `int`. Como no hay información sobre lo que `x` retorna, su tipo de salida se representa con una variable de tipo `Tx`. El procedimiento completo recibe `x` (cuyo tipo es `(int -> Tx)`) y el resultado final de la evaluación es el valor retornado por `x`, que es de tipo `Tx`.

####

La expresión `proc (x) if x then 1 else 3` tiene el tipo `(bool -> int)`. La variable `x` se utiliza como la condición de una estructura `if`, por lo que obligatoriamente debe ser de tipo `bool`. Ambas ramas del condicional, `then` y `else`, retornan valores enteros (`1` y `3`). En nuestro lenguaje, se exige que ambas ramas de un `if` retornen valores del mismo tipo, que en este caso es `int`. Por lo tanto, el procedimiento recibe un booleano y retorna un entero.

####

La expresión `proc (f) proc (x) if (f x) then 1 else 2` tiene el tipo `((Tx -> bool) * Tx -> int)`. Analicemos sus componentes:
1.  `f` es un procedimiento que recibe un argumento `x`. El tipo de `x` es desconocido, por lo que se denota como `Tx`.
2.  El resultado de `(f x)` se usa como condición en un `if`, lo que implica que `f` debe retornar un valor de tipo `bool`. Por lo tanto, el tipo de `f` es `(Tx -> bool)`.
3.  El procedimiento completo recibe dos argumentos: `f` y `x`.
4.  El resultado final del `if` es un entero (`1` o `2`), por lo que el tipo de retorno de todo el procedimiento es `int`.

####

La expresión `proc (x) proc (y) if y then x else 3` tiene el tipo `(int -> (bool -> int))`. El análisis es el siguiente:
1.  La condición del `if` es `y`, por lo que `y` debe ser de tipo `bool`.
2.  Las ramas `then` y `else` deben tener el mismo tipo. Como la rama `else` retorna `3` (un `int`), la rama `then` también debe retornar un `int`. Esto implica que `x` debe ser de tipo `int`.
3.  El procedimiento interno recibe `y` (un `bool`) y retorna un `int`. Su tipo es `(bool -> int)`.
4.  El procedimiento externo recibe `x` (un `int`) y retorna el procedimiento interno.

####

La expresión `proc (x) -(x, 3)` tiene el tipo `(int -> int)`. La operación de resta `-` requiere que sus operandos sean enteros. Por lo tanto, `x` debe ser de tipo `int`. El resultado de la resta también es un entero, que es el valor de retorno del procedimiento.

####

La expresión `proc (f) proc (x) -(f(x), 1)` tiene el tipo `((Tx -> int) -> (Tx -> int))`.
1.  La función `f` recibe un argumento `x` de tipo desconocido `Tx`.
2.  El resultado de `f(x)` se usa en una resta, lo que obliga a que `f` retorne un `int`. Por lo tanto, el tipo de `f` es `(Tx -> int)`.
3.  El procedimiento interno recibe `x` (de tipo `Tx`) y retorna el resultado de la resta, que es `int`. Su tipo es `(Tx -> int)`.
4.  El procedimiento externo recibe `f` y retorna el procedimiento interno.

####

La expresión `proc (x) (x x)` genera un error de tipos debido a una definición recursiva infinita. Para determinar el tipo de `x`, digamos `Tx`, observamos que `x` es un procedimiento que se acepta a sí mismo como argumento. Esto implica que `Tx` debe ser `(Tx -> Ty)`, donde `Ty` es el tipo de retorno. Al sustituir `Tx`, obtenemos `((Tx -> Ty) -> Ty)`, y así sucesivamente. Esta estructura de tipo infinita no es resoluble en el sistema de tipos.

####

La expresión `(proc (x) if x then 88 else 99) 33` produce un error de tipos. El procedimiento `proc (x) if x then 88 else 99` tiene el tipo `(bool -> int)`, ya que espera un argumento booleano para la condición del `if`. Sin embargo, se le está aplicando el valor `33`, que es un entero. Debido a esta discordancia de tipos entre lo esperado (`bool`) y lo proporcionado (`int`), la evaluación falla.

####

La expresión `(proc (p) (p (proc (z) z))) (proc (q) if (q 88) then 99 else 100)` produce un error de tipos. Analicemos la aplicación:
1.  El argumento `p` del procedimiento externo es `proc (q) if (q 88) then 99 else 100`.
2.  El argumento interno `(proc (z) z)` es la función identidad, cuyo tipo es `(Tz -> Tz)`.
3.  Al evaluar `p` con la función identidad, esta última se convierte en `q`. La expresión se transforma en `if ((proc (z) z) 88) then ...`.
4.  La condición del `if` evalúa la función identidad con el entero `88`, lo cual retorna `88`.
5.  El `if` espera una condición de tipo `bool`, pero recibe un `int` (`88`). Esto causa un error de tipos.

####

La expresión `let d = proc (x) (d x) in (d d)` resulta en un error de tipos debido a una definición recursiva. La variable `d` se define como un procedimiento que se llama a sí mismo con su argumento `x`. Al evaluar `(d d)`, `d` se llama a sí misma con `d` como argumento. Esto lleva a una cadena de llamadas `(d (d (d ...)))`, lo que corresponde a un tipo recursivo infinito similar a `proc (x) (x x)`, que no es válido en el sistema.
