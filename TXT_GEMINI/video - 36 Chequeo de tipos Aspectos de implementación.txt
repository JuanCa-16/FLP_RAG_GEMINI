### Tipos en el Lenguaje de Programación

A continuación, se discuten los aspectos de implementación de tipos en nuestro lenguaje de programación.

#### 1. Definición de Tipos Básicos

Los tipos fundamentales que se manejarán son:
*   **IntType (Entero):** Representa números enteros.
*   **BoolType (Booleano):** Representa valores verdadero/falso.
*   **ProcedureType (Procedimiento):** Un procedimiento se define por un conjunto de tipos de argumentos de entrada y un tipo de resultado. Se representa como: `T1 * T2 * ... * Tn -> T_salida`, donde `T1` a `Tn` son los tipos de los argumentos y `T_salida` es el tipo de retorno.

Cada expresión en el interpretador tendrá una expresión de tipo asociada, indicando si es un entero, un booleano o un tipo de procedimiento.

#### 2. Propiedad de un Valor Expresado

La propiedad de que un valor expresado `v` es de tipo `T` se define inductivamente de la siguiente manera:
*   Un valor expresado es de tipo **IntType** si y solo si es un entero.
*   Un valor expresado es de tipo **BoolType** si y solo si es un booleano.
*   Un valor expresado es de tipo **ProcedureType** (`T1 * T2 * ... * Tn -> T_salida`) si y solo si es un valor de procedimiento (`prodval`) que espera exactamente `n` argumentos de tipos `T1` a `Tn` y retorna un valor de tipo `T_salida`. Esto requiere una verificación de cada uno de los tipos correspondientes.

#### 3. Tipado Estático

En este lenguaje, cada valor expresado tiene un único tipo, es decir, es de **tipado estático**. Esto significa que los tipos no pueden cambiar durante la ejecución. No obstante, en algunos escenarios, como el valor retornado por un procedimiento, no siempre es posible determinar el tipo en tiempo de ejecución. Para abordar esto, existen dos estrategias principales:
*   **Chequeo de Tipos:** Los tipos se especifican directamente.
*   **Inferencia de Tipos:** Los tipos se infieren a partir del contexto.

#### 4. Ejemplos de Inferencia de Tipos

A continuación, se presentan varios ejemplos de expresiones y la derivación de sus tipos:

*   **`even` (par):**
    *   Descripción: Función que recibe un entero y retorna un booleano.
    *   Tipo: `IntType -> BoolType`
    *   Ejemplo: `even 5` retorna `false`.

*   **`+` (suma):**
    *   Descripción: Función que recibe dos enteros y retorna un entero.
    *   Tipo: `IntType * IntType -> IntType`
    *   Ejemplo: `+ 5 2` retorna `7`.

*   **`lambda x (lambda y (+ x y))`:**
    *   Explicación: La expresión `(+ x y)` implica que `x` e `y` deben ser `IntType`, y su resultado es `IntType`.
    *   `lambda y (+ x y)`: Recibe `y` (`IntType`) y retorna `IntType`. Su tipo es `IntType -> IntType`.
    *   `lambda x (...)`: Recibe `x` (`IntType`) y retorna el procedimiento anterior.
    *   Tipo: `IntType -> (IntType -> IntType)`

*   **`lambda f (lambda x (even (+ (f x) 1)))`:** (Se asume `f` es una función, `x` una variable)
    *   `+ (f x) 1`: Implica que `(f x)` debe ser `IntType` y el resultado de la suma es `IntType`.
    *   `even (...)`: `even` espera un `IntType`, por lo que `(+ (f x) 1)` debe ser `IntType`. `even` retorna `BoolType`.
    *   `f x`: Como `x` es el argumento de `lambda x`, su tipo es `IntType`. Para que `f x` sea `IntType`, `f` debe ser un procedimiento que toma un `IntType` y retorna un `IntType`. Así, `f: IntType -> IntType`.
    *   La expresión `lambda x (...)` toma `x: IntType` y retorna `BoolType`. Su tipo es `IntType -> BoolType`.
    *   La expresión `lambda f (...)` toma `f: (IntType -> IntType)` y retorna `(IntType -> BoolType)`.
    *   Tipo: `(IntType -> IntType) -> (IntType -> BoolType)`

*   **`- 32 26`:**
    *   Descripción: Resta de dos enteros. El resultado es un entero.
    *   Tipo: `IntType`

*   **`proc x (x 26)`:**
    *   Explicación: `x` es un procedimiento que se aplica a `26` (un `IntType`). Se desconoce el tipo de retorno de `x`, se denota como `Tx`.
    *   Tipo de `x`: `IntType -> Tx`
    *   Tipo de la expresión: `(IntType -> Tx) -> Tx`

*   **`proc x (if x 1 3)`:**
    *   Explicación: La condición `x` de un `if` debe ser `BoolType`. Las ramas `1` y `3` son `IntType`. El `if` retorna `IntType`.
    *   Tipo: `BoolType -> IntType`

*   **`proc f (proc x (if (f x) 1 2))`:**
    *   Explicación:
        *   `if (f x) 1 2`: `(f x)` debe ser `BoolType`. Las ramas `1` y `2` son `IntType`. El `if` retorna `IntType`.
        *   `proc x (...)`: `x` es el argumento, se desconoce su tipo inicial `Tx`. `(f x)` es `BoolType`, entonces `f` toma `Tx` y retorna `BoolType`.
        *   `proc f (...)`: `f` es el argumento, su tipo es `(Tx -> BoolType)`. El procedimiento `proc x (...)` retorna `IntType`.
    *   Tipo: `(Tx -> BoolType) -> (Tx -> IntType)`

*   **`proc x (proc y (if y x 3))`:**
    *   Explicación:
        *   `if y x 3`: `y` es la condición, debe ser `BoolType`. `x` y `3` deben ser del mismo tipo; `3` es `IntType`, entonces `x` debe ser `IntType`. El `if` retorna `IntType`.
        *   `proc y (...)`: Toma `y: BoolType` y retorna `IntType`. Su tipo es `BoolType -> IntType`.
        *   `proc x (...)`: Toma `x: IntType` y retorna `(BoolType -> IntType)`.
    *   Tipo: `IntType -> (BoolType -> IntType)`

*   **`proc x (- x 3)`:**
    *   Explicación: La operación de resta `-` implica que `x` debe ser `IntType`. El resultado es `IntType`.
    *   Tipo: `IntType -> IntType`

*   **`proc f (proc x (- (f x) 1))`:**
    *   Explicación:
        *   `- (f x) 1`: Implica que `(f x)` debe ser `IntType` y el resultado es `IntType`.
        *   `proc x (...)`: Toma `x` (tipo `Tx` desconocido). `f x` es `IntType`, entonces `f` debe ser un procedimiento `Tx -> IntType`. Este procedimiento interno retorna `IntType`.
        *   `proc f (...)`: Toma `f: (Tx -> IntType)` y retorna `(Tx -> IntType)`.
    *   Tipo: `(Tx -> IntType) -> (Tx -> IntType)`

*   **`proc x (proc y (x y))`:**
    *   Explicación:
        *   `x y`: `x` es un procedimiento que toma `y` (tipo `Ty`) y retorna un tipo `Tz`.
        *   `proc y (...)`: Toma `y: Ty` y retorna `Tz`. Su tipo es `Ty -> Tz`.
        *   `proc x (...)`: Toma `x: (Ty -> Tz)` y retorna `(Ty -> Tz)`.
    *   Tipo: `(Ty -> Tz) -> (Ty -> Tz)`

*   **`proc x (x 3)`:**
    *   Explicación: `x` es un procedimiento que toma `3` (`IntType`). Su tipo de retorno es desconocido (`Tx`).
    *   Tipo: `(IntType -> Tx) -> Tx`

*   **`proc x (x x)`:**
    *   Explicación: Esto representa una definición de tipo recursiva, donde `x` toma un argumento de su propio tipo. Generalmente, esto conduce a un **error de tipos** en la mayoría de los sistemas de tipos simples.

*   **`proc x (if x 88 69)`:**
    *   Explicación: La condición `x` debe ser `BoolType`. `88` y `69` son `IntType`. El resultado es `IntType`.
    *   Tipo: `BoolType -> IntType`

*   **`proc x (proc y (if x y 99))`:**
    *   Explicación:
        *   `if x y 99`: `x` es la condición, `BoolType`. `y` y `99` deben ser del mismo tipo; `99` es `IntType`, entonces `y` es `IntType`. El `if` retorna `IntType`.
        *   `proc y (...)`: Toma `y: IntType` y retorna `IntType`. Su tipo es `IntType -> IntType`.
        *   `proc x (...)`: Toma `x: BoolType` y retorna `(IntType -> IntType)`.
    *   Tipo: `BoolType -> (IntType -> IntType)`

*   **`proc x (if x 88 99) 33`:**
    *   Explicación:
        *   El procedimiento `proc x (if x 88 99)` tiene tipo `BoolType -> IntType`.
        *   Se intenta aplicar este procedimiento a `33`.
        *   `33` es `IntType`, pero el procedimiento espera `BoolType`.
    *   Tipo: **Error de tipos** (se esperaba `BoolType`, se recibió `IntType`).

*   **`proc p (if p 88 99) (proc z z)`:**
    *   Explicación:
        *   El procedimiento `proc p (if p 88 99)` tiene tipo `BoolType -> IntType` (similar al ejemplo anterior).
        *   Se intenta aplicar este procedimiento a `(proc z z)`.
        *   El tipo de `(proc z z)` (donde `z` es un procedimiento que se aplica a sí mismo) es una definición de tipo recursiva, lo que no es `BoolType`.
    *   Tipo: **Error de tipos** (se esperaba `BoolType`, se recibió un tipo de procedimiento recursivo).

*   **`proc f (proc g (proc p (if (p (f 1)) (g 1) x)))`:**
    *   Explicación (basado en la derivación paso a paso de la transcripción):
        *   `f 1`: `f` recibe `1` (`IntType`) y retorna un `IntType`. Así, `f: IntType -> IntType`.
        *   `g 1`: `g` recibe `1` (`IntType`) y retorna un `IntType`. Así, `g: IntType -> IntType`.
        *   `p (f 1)`: `(f 1)` es `IntType`. `p` toma este `IntType` y debe retornar `BoolType` (por ser la condición del `if`). Así, `p: IntType -> BoolType`.
        *   `g 1` y `x`: `g 1` es `IntType`. La rama `x` también debe ser `IntType`. Así, `x: IntType`.
        *   `proc p (...)`: Toma `p: (IntType -> BoolType)` y retorna `IntType`.
        *   `proc g (...)`: Toma `g: (IntType -> IntType)` y retorna `((IntType -> BoolType) -> IntType)`.
        *   `proc f (...)`: Toma `f: (IntType -> IntType)` y retorna `((IntType -> IntType) -> ((IntType -> BoolType) -> IntType))`.
    *   Tipo: `(IntType -> IntType) -> ((IntType -> IntType) -> ((IntType -> BoolType) -> IntType))`

*   **`let` con definición recursiva (ejemplo de `f de dd`):**
    *   Explicación: Si una definición como `de` se define en términos de sí misma (e.g., `de` es un procedimiento que recibe a `de`), se produce una definición recursiva.
    *   Tipo: **Error de tipos** debido a la definición recursiva, a menos que el sistema de tipos soporte específicamente tipos recursivos.
