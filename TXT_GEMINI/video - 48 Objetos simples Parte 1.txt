En la implementación simple, la declaración de una clase contiene toda la información necesaria para su representación, incluyendo su nombre, la superclase de la que hereda, sus campos y las declaraciones de sus métodos. De este concepto se derivan dos estructuras de datos principales: la declaración de clase, que agrupa el nombre de la clase, el nombre de la superclase, una lista de campos y una lista de declaraciones de métodos; y la declaración de método, que consiste en un nombre, una lista de argumentos (variables de entrada) y una expresión que conforma su cuerpo. Estas estructuras de datos pueden generarse directamente a partir de la especificación gramatical del lenguaje.

####

Para gestionar las clases definidas, se utiliza una variable de entorno global denominada `de_class_enf`. Esta variable almacena el conjunto de todas las declaraciones de clase del programa. Inicialmente, `de_class_enf` está vacía. A medida que el intérprete procesa las definiciones de clases, un procedimiento llamado `elaborate_class` se encarga de poblar este entorno global, almacenando en él cada nueva declaración de clase. De esta forma, el entorno actúa como un repositorio central de la información de todas las clases disponibles.

####

Para recuperar la información de una clase específica desde el entorno global, se utiliza una función observadora llamada `lookup_class`. Esta función recibe un nombre de clase como argumento y busca su declaración correspondiente dentro de la variable global `de_class_enf`. Si la declaración se encuentra, la función la retorna. Si, tras recorrer todo el entorno, la clase no es encontrada (es decir, el entorno está vacío o la clase no está en él), la función genera un error para indicar que se está haciendo referencia a una clase desconocida.

####

En esta implementación simple, un objeto se representa como una lista de partes. Cada parte en la lista corresponde a una de las clases en la cadena de herencia del objeto, desde su clase más específica hasta la más general. Esta estructura de lista permite encapsular la información y el estado correspondientes a cada nivel de la jerarquía de herencia dentro del propio objeto.

####

La estructura de un objeto como lista de partes sigue un orden específico. La primera parte de la lista representa la clase más específica en la cadena de herencia, es decir, la clase de la cual el objeto es una instancia directa. Las partes siguientes en la lista corresponden a las superclases sucesivas. Por ejemplo, si se crea un objeto de la clase `C3`, que hereda de `C2`, y `C2` hereda de `C1`, el objeto se representará como una lista con las partes de `C3`, luego `C2` y finalmente `C1`. La clase base `Object`, de la cual todas las clases heredan implícitamente, se representa al final de la cadena como una lista vacía.

####

Este modelo de representación de objetos está diseñado para un sistema de herencia simple, lo que significa que cada clase solo puede heredar de una única superclase. No se soporta la herencia múltiple. Por lo tanto, la cadena de herencia de cualquier objeto siempre será una secuencia lineal que asciende desde la clase del objeto hasta la clase base `Object`. Por ejemplo, si una clase `C4` hereda de `C2` (que a su vez hereda de `C1`), la cadena de herencia para un objeto de tipo `C4` será `C4 -> C2 -> C1 -> Object`, manteniendo siempre una estructura lineal.
