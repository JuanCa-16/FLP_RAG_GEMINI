Cuando se escribe un procedimiento, se debe definir qué clase de valores se espera como entrada y como salida.

####
Técnicas

Existen dos técnicas para la definición recursiva de datos:
1.  Especificación inductiva.
2.  Especificación mediante gramáticas.

####
Definición

Se define un conjunto S, el cual es el conjunto más pequeño que satisface las siguientes dos propiedades:
1.  Algunos valores específicos que deben estar en S.
2.  Si algunos valores están en S, entonces otros valores también están en S.

####
Números pares
1.  Si n = 2, entonces n es par.
2.  Si n es par, entonces n + 2 también es par.

Lista de números
1.  `empty` es una lista de números.
2.  Si n es un número y l es una lista, entonces `(cons n l)` es una lista de números.

####
Especificación formal

Ahora formalmente:

Números pares
1.  2 ∈ S
2.  Si n ∈ S, entonces (+ n 2) ∈ S.

Lista de números
1.  `'()` ∈ S
2.  Si l ∈ S y n ∈ N, entonces `(cons n l)` ∈ S.

####
<CODIGO_BLOCK>
;; Demostración de que una lista anidada con `cons`
;; se ajusta a la definición recursiva de lista.
;;
;; Demuestre que (cons 1 (cons 2 (cons 3 '()))) es una lista de números.
;;
;; Solución:
;; 1. '() ∈ S (por la regla base).
;; 2. 3 ∈ N y '() ∈ S, por lo tanto, (cons 3 '()) ∈ S.
;; 3. 2 ∈ N y (cons 3 '()) ∈ S, por lo tanto, (cons 2 (cons 3 '())) ∈ S.
;; 4. 1 ∈ N y (cons 2 (cons 3 '())) ∈ S, por lo tanto, (cons 1 (cons 2 (cons 3 '()))) ∈ S.
;; Se puede seguir hasta llegar al caso fundamental '() ∈ S.
</CODIGO_BLOCK>

####
Especificación formal

Ahora realicemos la especificación inductiva de:
1.  Una lista de números pares.
2.  Múltiplos de 5.

Lista de números pares
1.  2 ∈ P
2.  Si n ∈ P, entonces `(+ n 2)` ∈ P.
3.  `'()` ∈ S
4.  Si l ∈ S y x ∈ P, entonces `(cons x l)` ∈ S.

Múltiplos de 5
1.  5 ∈ S
2.  Si n ∈ S, entonces `(+ n 5)` ∈ S.

####
Una forma sencilla de especificar datos recursivos es con gramáticas regulares en forma Backus-Naur.
Las gramáticas se componen de:
1.  Símbolos no terminales, que son aquellos que se componen de otros símbolos. Son conocidos como categorías sintácticas.
2.  Símbolos terminales: corresponden a elementos del alfabeto.
3.  Reglas de producción.

####
<CODIGO_BLOCK>
;; Conceptos básicos de lenguajes formales y gramáticas.
;;
;; Alfabeto: Conjunto de símbolos, ejemplo P = {a, b, c, ...}
;;
;; Reglas de producción: Construcción del lenguaje.
;;
;; Cerradura de Kleene:
;; {'a}* = {'(), '(a), '(a a), '(a a a), ...}
;;
;; Cerradura positiva:
;; {'b}+ = {'(b), '(b b), '(b b b), ...}
</CODIGO_BLOCK>

####
Lista de números
`<lista-de-enteros> ::= '()`
`::= (<entero> <lista-de-enteros>)`

O de forma compacta:
`<lista-de-enteros> ::= '() | (<entero> <lista-de-enteros>)`

O usando la notación de Kleene:
`<lista-de-enteros> ::= (<entero>)*`

####
Árbol Binario
`<arbol-binario> ::= <entero>`
`::= (<simbolo> <arbol-binario> <arbol-binario>)`

'(foo 1 2)
'(foo 1 (bar 2 3))
</CODIGO_BLOCK>

####
Expresión cálculo λ
`<lambda-exp> ::= <identificador>`
`::= (lambda (<identificador>) <lambda-exp>)`
`::= (<lambda-exp> <lambda-exp>)`

`<identificador> ::= <letra>+`

<CODIGO_BLOCK>
;; Ejemplos de expresiones de cálculo lambda representadas como S-expresiones.
'(lambda (x) (x y))
'((lambda (y) (z y)) (x y))
</CODIGO_BLOCK>

####
La definición inductiva o mediante gramáticas de los conjuntos de datos sirve de guía para desarrollar procedimientos que operan sobre dichos datos. La estructura de los programas debe seguir la estructura de los datos. Para esto, realizamos la especificación recursiva de programas; la idea es utilizar el principio del subproblema más pequeño.

####
<CODIGO_BLOCK>
;; La función estándar de Racket `list-length` retorna el tamaño de una lista.
;; Para el diseño de esta función, debemos retornar a la especificación
;; recursiva de las listas.
;;
;; <Lista> ::= '() | (<numero> <Lista>)
;;
;; Se debe considerar entonces el caso base de la lista vacía,
;; en el cual retornamos 0.
</CODIGO_BLOCK>

####
<CODIGO_BLOCK>
;; Esqueleto de la función `list-length` basado en la definición recursiva.
;; Se identifica el caso base donde la lista está vacía.
;;
;; list-length : list -> number
(define list-length
  (lambda (lst)
    (if (null? lst)
        0
        ...)))
;; Tomando en cuenta la segunda parte de la definición, se debe
;; sumar 1 al tamaño si no encontramos la lista vacía.
</CODIGO_BLOCK>

####
<CODIGO_BLOCK>
;; Implementación completa de `list-length` siguiendo la estructura
;; de la definición recursiva de listas.
;;
;; list-length : list -> number
(define list-length
  (lambda (lst)
    (if (null? lst)
        0
        (+ 1 (list-length (cdr lst))))))

;; De acuerdo a la especificación recursiva de listas, ¿qué se puede
;; decir de este diseño?
</CODIGO_BLOCK>

####
Un árbol binario

Recordando la definición de los árboles vista anteriormente:
`<arbol-binario> ::= <entero>`
`::= (<simbolo> <arbol-binario> <arbol-binario>)`

Este árbol será representado así:
```racket
(define arbolA '(k (h 5 3) (t (s 10 11) 12)))
(define arbolB 2)
```
En DrRacket, el operador `'` (quote) va a generar una lista; toda palabra será convertida en símbolo y todo `(...)` será convertido en lista.

####
Un árbol binario

Procedimiento `sum-arbol`:
```racket
(define sum-arbol
  (lambda (arbol)
    (if (number? arbol)
        arbol
        (+ (sum-arbol (cadr arbol))
           (sum-arbol (caddr arbol))))))
```

####
Lista de números

Procedimiento para generar una lista de números a partir de un árbol.
```racket
;; BNF
;; Entrada: <arbol-binario> ::= <entero> | (<simbolo> <arbol-binario> <arbol-binario>)
;; Salida:  <lista-numeros> ::= '() | (cons <entero> <lista-numeros>)

(define arbol->lista
  (lambda (l)
    (if (number? l)
        (list l)
        (append (arbol->lista (cadr l))
                (arbol->lista (caddr l))))))
```

####
Resumen

En el diseño de programas para datos recursivos, tenga en cuenta:
1.  Identifique el caso terminal (base) o donde termina la especificación recursiva.
2.  La idea es pasar de estados no terminales a uno terminal.
3.  La estrategia es llamar recursivamente la función desde un estado no terminal para llegar a uno terminal.

####
El concepto de variable es fundamental en los lenguajes de programación. Una variable puede ser declarada y posteriormente ligada o referenciada.

Declaración:
```racket
(lambda (x) ...)
(let ((x ...)) ...)
```

Referencia:
```racket
x       ; Como valor
(x y)   ; Como procedimiento
```

####
Una variable está ligada al lugar donde se declara. El valor ligado o referenciado por la variable es su denotación. Cada lenguaje de programación tiene asociadas unas reglas de ligadura que determinan a qué declaración hace referencia cada variable. Dependiendo del momento de aplicación de las reglas, antes o durante la ejecución, los lenguajes se denominan de alcance estático o alcance dinámico.

####
Si la expresión `e` es una variable, la variable `x` ocurre libre si y solo si `x` es igual a `e`.

Si la expresión `e` es de la forma `(lambda (y) e')`, entonces la variable `x` ocurre libre si y solo si `y` es distinto que `x` y `x` ocurre libre en `e'`.

Si la expresión `e` es de la forma `(e1 e2)`, entonces `x` ocurre libre si y solo si `x` ocurre libre en `e1` o en `e2`.

####

```racket
'(lambda (x) (lambda (y) x))
'((lambda (z) (lambda (y) x)) x)
'((lambda (y) (lambda (y) x)) ((lambda (z) x) x))
```

####
Determinar si una variable ocurre libre.

De esta forma se diseña un procedimiento que evalúa si `var` ocurre libre en `exp`.
```racket
(define occurs-free?
  (lambda (var exp)
    (cond
      ((symbol? exp)
       (eqv? var exp))
      ((eqv? (car exp) 'lambda)
       (and
        (not (eqv? (caadr exp) var))
        (occurs-free? var (caddr exp))))
      (else
       (or (occurs-free? var (car exp))
           (occurs-free? var (cadr exp)))))))
```

####
Se define como el alcance de una variable la región dentro del programa en la cual ocurren todas las referencias a dicha variable.
```racket
; Expresión para ilustrar el alcance de las variables.
(lambda (x)       ; Alcance de x1
  (map
   (lambda (x)   ; Alcance de x2, sombrea a x1
     x)          ; Referencia a x2
   ; ... una lista ...
   ))
```
