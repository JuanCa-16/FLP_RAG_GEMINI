La receta para el diseño de interfaces de datos recursivos, o Tipos Abstractos de Datos (TAD), se basa en tres componentes. Primero, se debe incluir un constructor para cada clase de datos, que corresponde a cada regla de producción o variante. Segundo, es necesario un predicado que permita identificar cada tipo de dato específico. Finalmente, se debe implementar un extractor para cada uno de los campos o componentes de los diferentes tipos de datos.

####

En el entorno de Dr. Racket, el proceso de crear constructores, predicados (observadores) y extractores para un tipo de dato se puede automatizar mediante la función de interfaz `define-datatype`. Esta herramienta permite generar todos los procedimientos necesarios de la interfaz en un único paso, simplificando significativamente la definición de tipos de datos abstractos.

####

La sintaxis de `define-datatype` requiere especificar el nombre del nuevo tipo de dato, el nombre del predicado asociado para verificar la pertenencia a ese tipo, el nombre de cada variante posible del tipo y, finalmente, los nombres de los campos que componen cada variante.

####

Al utilizar `define-datatype` se deben seguir ciertas reglas. Dos tipos de datos no pueden tener el mismo nombre. De manera similar, dos variantes no pueden compartir el mismo nombre, incluso si pertenecen a tipos de datos diferentes. El sistema genera automáticamente un procedimiento constructor para cada variante definida. Si una variante se compone de N campos, su constructor correspondiente será una función que recibirá N argumentos.

####

Un ejemplo de aplicación de `define-datatype` es la definición del tipo `ls-expression`. Este tipo puede tener tres variantes: `body-expression` (con un campo `exp`), `lambda-expression` (con los campos `id` y `body`) y `app-expression` (con los campos `rator` y `rand`). Una ventaja clave de este enfoque es que permite asociar un tipo esperado a cada campo (por ejemplo, `symbol?` o `ls-expression?`), habilitando una verificación automática de tipos que no es factible en las implementaciones manuales de constructores y extractores.
