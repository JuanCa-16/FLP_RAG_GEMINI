####
**Receta para el diseño de tipos de datos abstractos (TDA)**

La metodología para el diseño de tipos de datos recursivos sigue las siguientes reglas:
- Incluir un `constructor` para cada clase de datos o variante de la regla de producción.
- Incluir un `predicado` que identifique cada tipo de datos.
- Incluir un `extractor` para cada campo de los diferentes tipos de datos.

####
**Uso de `define-datatype` en Racket**

La función de interfaz `define-datatype` permite implementar la estructura de los tipos de datos recursivos de forma eficiente. El uso de esta función es una de las razones principales para emplear `DrRacket` en el curso, ya que genera automáticamente los constructores, predicados y extractores en un solo paso.

La sintaxis básica para `define-datatype` requiere definir:
- El nombre del tipo de dato.
- El nombre del predicado (para verificar si un elemento pertenece al tipo de dato).
- El nombre de la variante.
- El nombre de los campos asociados a la variante.

####
**Reglas y recomendaciones para `define-datatype`**

Al utilizar `define-datatype`, se deben tener en cuenta las siguientes consideraciones:
- Dos tipos de datos no pueden tener el mismo nombre.
- Dos variantes no pueden compartir el mismo nombre, incluso si pertenecen a tipos diferentes.
- Para cada variante, el sistema genera automáticamente un procedimiento constructor.
- Si una variante tiene `N` campos, el constructor generado recibirá exactamente `N` argumentos.
- Esta herramienta permite verificar que los argumentos recibidos correspondan exactamente con lo definido, garantizando la integridad de los datos.

####
**Ejemplo de implementación: Expresiones de Lenguaje (LC Expression)**

A continuación, se presenta la definición de `lc-expression` utilizando `define-datatype`. Esta estructura permite representar diferentes variantes como `var-exp`, `lambda-exp` y `app-exp`:

```racket
(define-datatype lc-expression lc-expression?
  (var-exp (id symbol?))
  (lambda-exp (id symbol?)
              (body lc-expression?))
  (app-exp (rator lc-expression?)
           (rand lc-expression?)))
```

En este ejemplo, los nombres asignados a los campos sirven para identificar el tipo de dato que se espera, permitiendo al sistema realizar una verificación de tipos automática.

####
**Ejemplo de implementación: Lista de símbolos (S-List)**

Para representar una lista de símbolos (`s-list`), donde un elemento puede ser un símbolo o una sublista, se utiliza la siguiente estructura:

```racket
(define-datatype s-list s-list?
  (empty-s-list)
  (non-empty-s-list (first symbol?)
                    (rest s-list?)))
```

Asimismo, se puede definir una `expression` que contenga símbolos o listas de símbolos, demostrando la versatilidad de `define-datatype` para manejar estructuras recursivas.
