####
**Análisis de tipos en expresiones de procedimientos (Ejemplo 1)**

Considere la siguiente expresión que utiliza `let` para definir procedimientos y aplicarlos:

```racket
let
  j = proc (int x, ? y) +(x, y)
  t = proc (? k, int a, ? b) (k a b)
in
  (t j 2 3)
```

Para determinar el tipo de la expresión, realizamos el análisis de inferencia:
- `y = int`
- `k = (int * int -> int)`
- `b = int`

Tras evaluar la aplicación de los procedimientos, se determina que la expresión resultante es de tipo `int`.

####
**Análisis de tipos en expresiones de procedimientos (Ejemplo 2)**

Considere la siguiente expresión compleja que involucra procedimientos con múltiples argumentos y estructuras condicionales:

```racket
let
  j = proc (int x, ? y)
    if (y 2)
      then +(x, 2)
      else -(x, 3)
  t = proc (? k, int a, ? b, ? c) (k +(a, b) c)
  s = proc (? a) zero?(a)
in
  let
    p = proc (? m, ? n) (m n 5 14 s)
  in
    (p t j)
```

Para determinar el tipo de la expresión, se analiza la firma de cada procedimiento y sus parámetros:
- `y = (int -> bool)`
- `k = (int * (int -> bool) -> int)`
- `b = int`
- `c = (int -> bool)`
- `a = int`
- `m = ((int -> bool) * int * int * (int -> bool) -> int)`
- `n = (int * (int -> bool) -> int)`

Tras la resolución de los tipos de los argumentos y las aplicaciones, la expresión final es de tipo `int`.
