En este momento, tenemos limitaciones con respecto a la presentación de código, habiendo visto representaciones basadas en listas y procedimientos. Sin embargo, al estar limitados por el lenguaje de programación (Racket), utilizaremos los árboles de sintaxis abstracta.

###
Los árboles de sintaxis abstracta (AST) son representaciones desde la gramática que relacionan la estructura de un tipo de dato con su especificación. Para las expresiones de cálculo lambda (lc-exp), la gramática se define así:

<lc-exp> ::= <identifier>
            var-exp(id)
         ::= "lambda" "(" <identifier>* ")" <lc-exp>
          lambda-exp(lid, exp)
         ::= "(" <lc-exp> <lc-exp> ")"
          app-exp(rator, rand)

Usamos los constructores `var-exp`, `lambda-exp` y `app-exp` para construir estos árboles.

###
Un `var-exp` representa un identificador simple, como la variable `x`:

```racket
(var-exp 'x)
```

Su representación gráfica es:
```
graph TD
    var-exp -->|id| x
```

###
Un `lambda-exp` representa una función lambda, especificando sus identificadores de parámetros (lid) y su cuerpo (exp). Aquí, una lambda con parámetros `x`, `y`, `z` y cuerpo `(var-exp 'y)`:

```racket
(lambda-exp
 (list x y z)
 (var-exp 'y))
```

Su representación gráfica es:
```
graph TD
    lambda-exp -->|lid|A[x,y,z]
    lambda-exp -->|exp|B[var-exp]
    B -->|id|y
```

###
Un `app-exp` representa la aplicación de una función a un argumento (rator y rand). Este es un ejemplo más complejo de aplicación anidada de funciones lambda:

```racket
(app-exp
 (app-exp
  (lambda-exp
   '(a b c)
   (lambda-exp
    '(x y z)
    (var-exp 'x)))
  (var-exp 'z))
 (lambda-exp
  '(a b c d e f)
  (var-exp 'e)))
```

Su representación gráfica es:
```
graph TD
    A[app-exp] -->|rator| B[app-exp]
    A[app-exp] -->|rand| C[lambda-exp]
    B -->|rator| D[lambda-exp]
    B -->|rand| E[var-exp]
    D -->|lid| F[a,b,c]
    D -->|exp| G[lambda-exp]
    G -->|lid| H[x,y,z]
    G -->|exp| I[var-exp]
    I -->|id| J[x]
    E -->|id| K[z]
    C -->|lid| L[a,b,c,d,e,f]
    C -->|exp| M[var-exp]
    M -->|id| N[e]
```

###
La representación de árboles de sintaxis abstracta permite saber si el código fuente está bien escrito y representar los datos de una forma independiente de los lenguajes de programación.

###
Los ASTs pueden representar estructuras de código muy complejas, como se muestra en este gran ejemplo de `app-exp` anidado:

```racket
(app-exp
 (app-exp
  (app-exp
   (app-exp
    (var-exp 'x)
    (lambda-exp
     '(a b c)
     (var-exp 'y)))
   (app-exp
    (lambda-exp
     '(x y z)
     (var-exp 'p))
    (lambda-exp
     '(m n o p q r t u)
     (lambda-exp
      '(v w x y z)
      (lambda-exp
       '(a b c)
       (var-exp 'p))))))
  (app-exp
   (var-exp 's)
   (var-exp 'r)))
 (app-exp
  (app-exp
   (var-exp 'a)
   (var-exp 'b))
  (var-exp 'x)))
```

Su representación gráfica es:
```
graph TD
    A[app-exp] -->|rator| B[app-exp]
    A -->|rand| C[app-exp]
    B[app-exp] -->|rator| D[app-exp]
    B -->|rand| E[app-exp]
    D[app-exp] -->|rator| F[app-exp]
    D -->|rand| G[app-exp]
    F -->|rator| H[var-exp]
    H -->|id| I[x]
    F -->|rand| J[lambda-exp]
    J -->|lid| K[a,b,c]
    J -->|exp| L[var-exp]
    L -->|id| M[y]
    G -->|rator| N[lambda-exp]
    G -->|rand| O[lambda-exp]
    N -->|lid| P[x,y,z]
    N -->|exp| Q[var-exp]
    Q -->|id| R[p]
    O -->|lid| S[m n o p q r t u]
    O -->|exp| T[lambda-exp]
    T -->|lid| U[v w x y z]
    T -->|exp| V[lambda-exp]
    V -->|lid| W[a b c]
    V -->|exp| X[var-exp]
    X -->|id| Y[p]
    C -->|rator| Z[app-exp]
    C -->|rand| AA[var-exp]
    AA -->|id| AF[x]
    Z -->|rator| AB[var-exp]
    AB -->|id| AC[a]
    Z --> |rand| AD[var-exp]
    AD -->|id| AE[b]
    E -->|rator| AG[var-exp]
    AG -->|id| AH[s]
    E -->|rand| AI[var-exp]
    AI -->|id| AJ[r]
```
