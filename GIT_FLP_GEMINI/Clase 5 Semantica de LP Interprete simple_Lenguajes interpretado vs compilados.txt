Compilados
Son aquellos que transforman el código en código máquina, directamente se ejecuta sobre la CPU. O bien genera un código binario que puede ejecutar un programa (Maquina Virtual del Lenguaje). Este código usualmente pasa por varias de etapas de optimización para buscar ejecutar el código lo más rapido posible.

El proceso de un lenguaje compilado inicia con el código fuente que es transformado por un Parser a un Árbol de Sintaxis Abstracta (AST):
graph TD
    A["Código fuente"] -->|Parser|B["AST"]

###
En el proceso de traducción el AST es transformado a código binario para el CPU o para un programa que lo ejecute (Máquina Virtual de Java).
Al no tener intermediaros con la CPU, los lenguajes compilados se ejecutan más rapidamente. La sintaxis suele ser más cerca al lenguaje máquina (ej. C++).

###
Interpretados
Un lenguaje interpretado es aquel cuyas instrucciones son leídas por otro programa llamado el intérprete.
¿Alguna vez notaste que cuando había un error de sintaxis en Java no ejecutaba nada y en Python ejecutaba hasta el error? Esto ocurre porque el intérprete lee las instrucciones línea por línea, lo que significa que el código fuente es traducido por otro programa que le envía las instrucciones a la CPU, existiendo un intermediario.

###
El proceso de interpretación puede visualizarse de la siguiente manera, donde el Frontend genera el AST que luego es interpretado:
graph TD
    A["Código fuente"] -->|Frontned|B["AST"]
    B-->|Intepretado|C["Salida"]

El parser puede enviar el AST directamente al intérprete, o bien generar código en un lenguaje intermedio el cual es interpretado. Esto también permite realizar optimizaciones.
graph TD
    A["Parser"]-->|AST|B["Interpretación"]

###
A continuación se presenta un resumen de los aspectos clave entre lenguajes compilados e interpretados:

**Aspecto: Ejecución**
Lenguajes Compilados: Se ejecuta directamente en la CPU o en una máquina virtual.
Lenguajes Interpretados: Requiere un intérprete que lee y ejecuta línea por línea.

**Aspecto: Velocidad**
Lenguajes Compilados: Mayor rendimiento (sin intermediarios).
Lenguajes Interpretados: Menor rendimiento (hay overhead del intérprete).

**Aspecto: Detección de errores**
Lenguajes Compilados: Errores detectados en tiempo de compilación (antes de ejecutar).
Lenguajes Interpretados: Errores detectados en tiempo de ejecución (hasta la línea fallida).

###
**Aspecto: Portabilidad**
Lenguajes Compilados: Depende de la arquitectura (requiere recompilación).
Lenguajes Interpretados: Más portables (el mismo código fuente corre en múltiples plataformas con intérprete).

**Aspecto: Optimización**
Lenguajes Compilados: Optimizaciones agresivas en tiempo de compilación.
Lenguajes Interpretados: Optimizaciones limitadas (en tiempo de ejecución o con JIT).

**Aspecto: Flexibilidad**
Lenguajes Compilados: Menos flexible (cambios requieren recompilación).
Lenguajes Interpretados: Más flexible (cambios en código fuente efectivos inmediatamente).

###
**Aspecto: Ejemplos típicos**
Lenguajes Compilados: C, C++, Rust, Go.
Lenguajes Interpretados: Python, JavaScript, Ruby, PHP.

**Aspecto: Proceso de traducción**
Lenguajes Compilados: Código fuente → AST → Código binario.
Lenguajes Interpretados: Código fuente → AST → Interpretación línea por línea.

**Aspecto: Uso de recursos**
Lenguajes Compilados: Menor uso de memoria en ejecución (estático).
Lenguajes Interpretados: Mayor uso de memoria (por el intérprete y entorno runtime).

**Aspecto: Depuración**
Lenguajes Compilados: Más difícil (depuración a nivel de binario).
Lenguajes Interpretados: Más fácil (depuración a nivel de código fuente).

###
Notas clave:
Los lenguajes compilados generan código binario optimizado para la CPU o una máquina virtual (ej: JVM para Java).
Los lenguajes interpretados dependen de un intérprete que traduce y ejecuta instrucciones en tiempo real.
Muchos lenguajes modernos usan enfoques híbridos (ej: Java compila a bytecode que luego es interpretado/JIT-compilado).
La elección entre compilado e interpretado depende del equilibrio entre rendimiento, portabilidad y flexibilidad.
