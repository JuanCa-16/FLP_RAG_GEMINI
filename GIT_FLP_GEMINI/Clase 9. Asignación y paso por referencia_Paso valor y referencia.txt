Paso por valor: Cuando evaluamos un procedimiento con variables en sus argumentos, hacemos una copia de la referencia que es independiente de la original.

Considera el siguiente procedimiento:
```
let f = proc (x,y)
        begin
            set x = + (x,1);
            set y = + (y,1);
            + (x,y)
        end
    a = 10
    b = 20
in
    (f a b)
```
En este caso, `a` vale 10 y `b` vale 20. `x` es una nueva referencia y `y` tambiĂ©n lo es, manteniendo los valores originales de `a` y `b` inalterados fuera del procedimiento.

```mermaid
flowchart LR
    E["Empty-env"]  
    A["env0 
        f,a,b
        closure(...),10, 20
    "]
    B["envf
    x,y
    10,20   
    "]
    E --> A
    E --> B
```

###
Paso por referencia: Al pasar las variables, estas deben cambiar si dentro del procedimiento se hacen modificaciones.

Usando el mismo procedimiento:
```
let f = proc (x,y)
        begin
            set x = + (x,1);
            set y = + (y,1);
            + (x,y)
        end
    a = 10
    b = 20
in
    (f a b)
```
En este caso, despuĂ©s de la ejecuciĂłn de `f`, `a` es 11 y `b` es 21.

###
ImplementaciĂłn de los valores:
*   Valores expresados: NĂşmero, booleanos y procVal.
*   Valores denotados: Referencia a valor expresado.

La implementaciĂłn tiene un cambio fundamental, cuando tenemos:
*   Blanco directo: Es una referencia normal (valor).
*   Blanco indirecto: Es cuando pasamos una variable a un procedimiento.

###
Cuando se usa paso por referencia, el siguiente procedimiento de ejemplo:
```
let f = proc (x,y)
        begin
            set x = + (x,1);
            set y = + (y,1);
            + (x,y)
        end
    a = 10
    b = 20
in
    (f a b)
```
se visualiza de la siguiente manera:
```mermaid
flowchart LR
    E["Empty-env"]  
    A["env0 
        f,a,b
        dt(closure(...)),dt(10), dt(20)
    "]
    B["envf
    x,y
    id(a),id(b) 
    "]
    E --> A
    E --> B
```
Los targets indirectos permiten relacionar las variables del llamado de `f` con las que se evalĂşan dentro del procedimiento.

###
Consideremos este otro caso donde uno de los argumentos es una expresiĂłn:
```
let f = proc (x,y)
        begin
            set x = + (x,1);
            set y = + (y,1);
            + (x,y)
        end
    a = 10
    b = 20
in
    (f a + (a,b))
```
AquĂ­ se observa la mezcla de `id(a)` (target indirecto) y `dt(30)` (target directo para `a+b`):
```mermaid
flowchart LR
    E["Empty-env"]  
    A["env0 
        f,a,b
        dt(closure(...)),dt(10), dt(20)
    "]
    B["envf
    x,y
    id(a),dt(30)    
    "]
    E --> A
    E --> B
```

###
Cuando evaluamos un procedimiento, se genera:
*   Target directo si pasamos un valor (se genera una nueva referencia).
*   Target indirecto si pasamos una variable (conectamos a la referencia original).

En paso por referencia, cuando enviamos una variable, no enviamos su valor, sino su referencia, para que cualquier efecto que se haga en la variable se vea reflejado en la variable original.

###
Para manejar correctamente las referencias y evitar targets indirectos que apunten a otros targets indirectos, se utiliza la funciĂłn `eval-rand`. Esta funciĂłn genera un `direct-target` si recibe un valor distinto a una variable. Si recibe una variable, y esta es un `direct-target`, genera un `indirect-target` que contiene la referencia; si la variable ya es un `indirect-target`, toma la referencia contenida y la envĂ­a directamente.

AquĂ­ estĂˇ la implementaciĂłn de `eval-rand`:
```scheme
(define eval-rand
  (lambda (rand env)
    (cases expression rand
      (var-exp (id)
        (let ((ref (apply-env-ref env id)))
          (indirect-target
            (cases target (primitive-deref ref)
              (direct-target (e) ref)
              (indirect-target (ref1) ref1)))))
      (else
        (direct-target (eval-expression rand env))))))
```

###
Ejemplo 1: Paso por referencia simple
```
let f = proc (x)
        begin
            set x = + (x,10);
            x
        end
    b = 20
in
    begin
        (f b);
        b
    end
```
Esto retorna 30, ya que `b` se modifica por referencia dentro de `f`.


###
Ejemplo 2: Interacciones complejas con paso por referencia
```
let x = 10
    y = 20
    z = 30
    f = proc (a,b,c)
        begin
            set a = + (a,1);
            set b = + (b,2);
            set c = + (c,3);
            + (a,b,c)
        end
in
    let g = proc (m,n,o)
        begin
            (f m n o);
            (f n m o);
            (f o m n);
            + (x,y,z)
        end
    in
        (g x y + (z,2))
```

###
AnĂˇlisis del Ejemplo 2:
`g(x, y, +(z,2))`
*   `m = x`, `n = y`, `o` es un `dt(32)` (resultado de `z+2`)

Cuando se llama `(f m n o)` (equivalente a `f(x, y, 32)`):
*   `a` referencia a `x`, `b` referencia a `y`, `c` es `dt(32)`.
*   `x` cambia de 10 a 11.
*   `y` cambia de 20 a 22.
*   `c` (el valor 32) cambia a 35, pero esto no afecta a `z`.
*   Valores: `x=11`, `y=22`, `z=30`.

Cuando se llama `(f n m o)` (equivalente a `f(y, x, 32)`):
*   `a` referencia a `y`, `b` referencia a `x`, `c` es `dt(32)`.
*   `y` cambia de 22 a 23.
*   `x` cambia de 11 a 13.
*   `c` (el valor 32) cambia a 35, sin afectar a `z`.
*   Valores: `x=13`, `y=23`, `z=30`.

Cuando se llama `(f o m n)` (equivalente a `f(32, x, y)`):
*   `a` es `dt(32)`, `b` referencia a `x`, `c` referencia a `y`.
*   `a` (el valor 32) cambia a 33, sin afectar a `z`.
*   `x` cambia de 13 a 15.
*   `y` cambia de 23 a 26.
*   Valores: `x=15`, `y=26`, `z=30`.

Finalmente, `+(x,y,z)` evalĂşa `15 + 26 + 30`, resultando en `71`.
