###
Implementación de referencia: Se definen los targets. Un target directo es un valor expresado. Un target indirecto apunta a un target directo, lo que garantiza que al pasar la referencia a un procedimiento, siempre se tenga un target directo.

###
Este código define el tipo de dato `target`, que puede ser un `direct-target` (que contiene un valor expresado) o un `indirect-target` (que contiene una referencia a un `direct-target`).

```racket
(define-datatype target target?
  (direct-target (val expval?))
  (indirect-target (ref ref-to-direct-target?)))
```

###
Un `expval` (valor expresado) puede ser un número, un booleano o un `procval`. La siguiente función define el predicado `expval?`:

```racket
(define expval?
  (lambda (x)
    (or (number? x)
        (boolean? x)
        (procVal? x))))
```

###
Una referencia a un target directo debe ser una referencia válida y su contenido debe ser un `direct-target`. La siguiente función comprueba esta condición:

```racket
(define ref-to-direct-target?
  (lambda (x)
    (and (reference? x)
         (cases reference x
           (a-ref (pos vec)
                  (cases target (vector-ref vec pos)
                    (direct-target (v) #T)
                    (indirect-target (v) #F)))))))
```

###
La función `deref` se modifica para reflejar que un `direct-target` contiene un valor, mientras que un `indirect-target` contiene una referencia a un `direct-target` que a su vez contiene el valor.

```racket
(define deref
  (lambda (ref)
    (cases target (primitive-deref ref)
      (direct-target (val) val)
      (indirect-target (ref1)
                       (cases target (primitive-deref ref1)
                         (direct-target (val) val)
                         (indirect-target (p)
                                          (eopl-error "Illegal reference" ref1)))))))
```

###
La función `setref!` se usa para cambiar el valor de una referencia. Si la referencia es a un target directo, se usa la misma referencia. Si es a un target indirecto, se usa la referencia contenida en él.

```racket
(define setref!
  (lambda (ref val)
    (let ((ref1 (cases target (primitive-deref ref)
                  (direct-target (e) ref)
                  (indirect-target (r) r))))
      (primitive-setref! ref1 (direct-target val)))))
```

###
La expresión `let-exp` se modifica para que genere `direct-target` para los valores de las variables que se enlazan.

```racket
(let-exp (lid lexpr expr)
  (let ((vexpr (map (lambda (x)
                       (direct-target (eval-expression x env)))
                     lexpr)))
    (eval-expression expr
                     (extend-env lid
                                 (list->vector vexpr)
                                 env))))
```

###
El entorno inicial predefinido se modifica para que sus valores iniciales (para variables como x, y, z, a, b, c) sean `direct-target`.

```racket
(define init-env
  (extend-env '(x y z)
              (list->vector (list (direct-target 1)
                                  (direct-target 2)
                                  (direct-target 3)))
              (extend-env '(a b c)
                          (list->vector (list (direct-target 4)
                                              (direct-target 5)
                                              (direct-target 6)))
                          (empty-env))))
```
