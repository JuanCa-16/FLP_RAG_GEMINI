###
El procedimiento `expand-type-expression` se utiliza para convertir una expresión de tipo en un tipo (AST). Recibe una expresión de tipo y retorna su representación de tipo.

```racket
;expand-type-expression: expresion de tipos -> tipo
(define expand-type-expression
  (lambda (texp)
    (cases type-exp texp
      (int-exp () int-type)
      (bool-exp () bool-type)
      (proc-type-exp (targs tresult)
        (proc-type
          (expand-type-expressions targs)
          (expand-type-expression tresult))))))
```

También se define `expand-type-expressions` para manejar listas de expresiones de tipo:

```racket
;expand-type-expressions: list of type-exp -> list of types
(define expand-type-expressions
  (lambda (texps)
    (map expand-type-expression texps)))
```

Este procedimiento permite transformar las expresiones de tipo en un tipo (AST), por ejemplo:

```racket
(expand-type-expression (int-exp))
;; #(struct:atomic-type int)
```

Una expresión booleana se expandiría a `bool-type`. Un ejemplo más complejo para un tipo de procedimiento es:

```racket
(expand-type-expression
  (proc-type-exp
    (list (int-exp) (int-exp))
    (bool-exp)))
;; #(struct:proc-type
;;   (#(struct:atomic-type int)
;;    #(struct:atomic-type int))
;;   #(struct:atomic-type bool))
```

###
El diseño del interpretador de tipos para expresiones literales y variables se basa en los siguientes principios: si es un número, retorna `int-type`; si es `true` o `false`, retorna `bool-type`. Si es una variable, busca su tipo en el ambiente de tipos actual.

```racket
(define type-of-expression
  (lambda (exp tenv)
    (cases expression exp
      (lit-exp (d) int-type)
      (true-exp () bool-type)
      (false-exp () bool-type)
      (var-exp (id)
        (deref (apply-tenv tenv id))))))
```

El ambiente de tipos se inicializa con `init-tenv`:

```racket
;Type enviroment
(define init-tenv
  (extend-env
    '(x y z)
    (list->vector (list int-type int-type int-type))
    (extend-env
      '(a b c)
      (list->vector (list int-type int-type int-type))
      (empty-env))))
```

La función `apply-tenv` es responsable de buscar el tipo de una variable en el ambiente:

```racket
; apply-env: environment -> type
(define apply-tenv
  (lambda (env var)
    (cases environment env
      (empty-env ()
        (eopl:error "Variable not found" var)) ; Error si no se encuentra
      (extend-env (lid vec old-env)
        (letrec ((search-var
                   ; Búsqueda recursiva en la lista de bindings
                   (lambda (lid lval [pos 0])
                     (cond
                       [(null? lid)
                        (apply-tenv old-env var)] ; Buscar en entorno anterior
                       [(equal? (car lid) var)
                        (a-ref pos vec)] ; Voy a generar una referencia cuando encuentro la variable
                       [else
                        (search-var (cdr lid) vec (+ pos 1))]))) ; Continuar búsqueda
                 ))
          (search-var lid vec))))))
```

###
Ejemplos de cómo `type-of-expression` evalúa el tipo de expresiones literales y variables usando `init-tenv`:

```racket
(type-of-expression
  (cases program (parser "5") (a-program (exp) exp))
  init-tenv)
;; #(struct:atomic-type int)

(type-of-expression
  (cases program (parser "false") (a-program (exp) exp))
  init-tenv)
;; #(struct:atomic-type bool)

(type-of-expression
  (cases program (parser "true") (a-program (exp) exp))
  init-tenv)
;; #(struct:atomic-type bool)

(type-of-expression
  (cases program (parser "x") (a-program (exp) exp))
  init-tenv)
;; #(struct:a-ref 0
;;   #(#(struct:atomic-type int)
;;     #(struct:atomic-type int)
;;     #(struct:atomic-type int)))

(type-of-expression
  (cases program (parser "y") (a-program (exp) exp))

;; #(struct:a-ref 1
;;   #(#(struct:atomic-type int)
;;     #(struct:atomic-type int)
;;     #(struct:atomic-type int)))
```

###
Para las expresiones condicionales (`if`), el `test-exp` debe ser de tipo `bool`, y el `true-exp` debe tener el mismo tipo que el `false-exp`. La siguiente sección de `type-of-expression` maneja estas reglas:

```racket
(define type-of-expression
  (lambda (exp tenv)
    (cases expression exp
      ...
      (if-exp (test-exp true-exp false-exp)
        (let ((test-type  (type-of-expression test-exp tenv))
              (true-type  (type-of-expression true-exp tenv))
              (false-type (type-of-expression false-exp tenv)))
          (begin
            (check-equal-type! test-type bool-type test-exp)
            (check-equal-type! true-type false-type exp)
            true-type)))
      ...)))
```

###
Se utilizan procedimientos auxiliares para la verificación y reporte de errores de tipo en expresiones condicionales:

*   `check-equal-type!` verifica que dos tipos sean iguales y genera un error si no lo son.
*   `type-to-external-form` y `arg-types-to-external-form` se encargan de formatear los tipos para que los mensajes de error sean legibles.

```racket
;check-equal-type: type, type -> type
(define check-equal-type!
  (lambda (t1 t2 exp)
    (if (not (equal? t1 t2))
      (eopl:error 'type-check
                  "Types didn't match ~s != ~s in ~%~s"
                  (type-to-external-form t1)
                  (type-to-external-form t2)
                  exp)
      t1)))
```

```racket
(define type-to-external-form
  (lambda (t)
    (cases type t
      (atomic-type (name) name)
      (proc-type (targs tresult)
        (append
          (arg-types-to-external-form targs)
          '(->)
          (list (type-to-external-form tresult)))))))
```

```racket
(define arg-types-to-external-form
  (lambda (ty)
    (cond
      [(null? ty) '()]
      [(null? (cdr ty))
       (list (type-to-external-form (car ty)))]
      [else
       (append
         (list (type-to-external-form (car ty)))
         (list "*")
         (arg-types-to-external-form (cdr ty)))])))
```

```

;;  #(struct:true-exp)
;;  #(struct:true-exp)
;;  #(struct:lit-exp 2))
```

###
Los procedimientos siguen la regla de tipo `t1 * t2 * ... * tn -> t`. Una expresión `proc` recibe tres elementos: una lista de expresiones de tipos para los argumentos, una lista de identificadores para los argumentos y un cuerpo (expresión).

El proceso de verificación de tipo para un procedimiento implica:
1.  Convertir la lista de expresiones de tipos en una lista de tipos.
2.  Calcular el tipo resultante del cuerpo con un ambiente extendido que incluye los identificadores y sus tipos.
3.  Generar el `proc-type` con los tipos de argumentos y el tipo de resultado calculados.

```racket
(define type-of-expression
  (lambda (exp tenv)
    (cases expression exp
      ....
      (proc-exp (targs args body)
        (type-of-proc-exp targs args body tenv))
      ...)))
```

La función auxiliar `type-of-proc-exp` realiza el cálculo de tipos para los procedimientos:

```racket
;type-of-proc-exp list of types, list of idenfier, exp => type
(define type-of-proc-exp
  (lambda (targs args body tenv)
    (let* ((args-types    (expand-type-expressions targs))
           (result-type   (type-of-expression
                            body
                            (extend-env args
                                        (list->vector args-types)
                                        tenv))))
      (proc-type args-types result-type))))
```
