Este contenido trata sobre la evaluación de procedimientos recursivos.

Consideremos el siguiente procedimiento recursivo definido con `letrec`:

```
letrec f (x,y) = if >(x,0) then +(y, (f -(x,1) y)) else y in (f 2 3)
```

###
La evaluación de la función `f` con los argumentos `(2,3)` se puede visualizar a través de un diagrama de flujo de entornos:

```mermaid
flowchart LR
    E["Empty-env"]
    A["envr0
    '(f)
    '((x,y))
    '(if..)"
    ]
    F1["envf1
    (x,y)
    (2,3)
    "]  
    F2["envf2
    (x,y)
    (1,3)
    "]
    F3["envf3
    (x,y)
    (0,3)
    "]      
    E --> A
    A --> F1
    A --> F2
    A --> F3
```

El proceso de evaluación detallado es el siguiente:
*   **Envf1:** `if >(x,0) then +(y, (f -(x,1) y)) else y` se evalúa como `if >(2,0) then +(3, (f 1 3))`.
*   **Envf2:** `if >(x,0) then +(y, (f -(x,1) y))` se evalúa como `if >(1,0) entonces +(3, (f 0 3))`.
*   **Envf3:** `if >(x,0) then +(y, (f -(x,1) y)) else y`. Dado que `>(0,0)` es falso, ejecuta `else y`, retornando `3`.

###
Observe la suma acumulada en cada paso:
1.  `+(3, (f 1 3))`
2.  `+(3, +(3, (f 0 3)))`
3.  `+(3, +(3, 3)) = 9`

###
Ahora examinemos un ejemplo más complejo que incluye anidamiento de `let` y `letrec`, así como múltiples funciones:

```
let x = let x = 3 in +(x,3) y = 8 z = proc (a,b) if >(a,b) then a else b in
letrec f (x,y) = if >(x,0) then +(y, (g -(x,2) +(y,1))) else let t = (z (z x y) (z y x)) in +(t,1)
g (m,n) = if >(m,0) then *(n, (f -(m,1) -(n,1))) else n in
(f x y)
```
El resultado de esta evaluación es `809`.

###
El diagrama de flujo de entornos para la evaluación de este procedimiento es el siguiente:

```mermaid

    envg1
    (m,n)

    (3,8)"
    ]
    ENVG2["

    (m,n)
    (1,9)"

    E --> ENVZ2
    E --> ENVZ3
```

###
Detalles de la evaluación:
*   **Envr0:** `(f x y)` se evalúa a `(f 6 8)`. `f` es un closure `((x,y) if .. envR0)`.
*   **Envf1:** `if >(x,0)` se evalúa a `>(6,0)` que es verdadero. Se ejecuta `+(8, (g 4 9))`.
*   **Envg1:** `if >(m,0)` se evalúa a `>(4,0)` que es verdadero. Se ejecuta `*(9, (f 3 8))`.
*   **Envf2:** `if >(x,0)` se evalúa a `(3,0)` que es verdadero. Se ejecuta `+(8, (g 1 9))`.
*   **Envg2:** `if >(m,0)` se evalúa a `(1,0)` que es verdadero. Se ejecuta `*(9, (f 0 8))`.
*   **Envf3:** `if >(x,0)` se evalúa a `(0,0)` que es falso. Se ejecuta la rama `ELSE`. `t = (z (z 0 8) (z 8 0))`.
    *   `(z 0 8)`: `if >(0,8)` es falso, retorna `8`. (Envz1)
    *   `(z 8 0)`: `if >(8,0)` es verdadero, retorna `8`. (Envz2)
    *   `t = (z 8 8)`: `if >(8,8)` es falso, retorna `8`. (Envz3)
*   **Envt:** Retorna `+(t,1)` que es `+(8,1) = 9`.

###
Evaluación total acumulada:
1.  `+(8, (g 4 9))`
2.  `+(8, *(9, (f 3 8)))`
3.  `+(8, *(9, +(8, (g 1 9))))`
4.  `+(8, *(9, +(8, *(9, (f 0 8)))))`
5.  `+(8, *(9, +(8, *(9, 9))))`
6.  `+(8, *(9, +(8, 81)))`
7.  `+(8, *(9, 89))`
8.  `+(8, 801)`
9.  `809`
