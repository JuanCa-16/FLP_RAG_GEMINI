Los objetos planos son los que utiliza Java, es básicamente un vector de campos, donde las primeras posiciones corresponden a las clases de mayor jerarquía y las últimas a las de menor jerarquía.

###
Aquí se define una clase `c1` que extiende de `object` con dos campos `x` e `y`, y un método `initialize`. Luego se instancia `o1`.

```
class c1 extends object
  field x
  field y
  method initialize () 1

let o1 = new c1 ()
in
??
```

Cuando es `o1` se muestra:
```
#(struct:an-object c1 #(0 0))
```
Una parte contiene el nombre de la clase, y los valores de los campos.

###
A continuación, se define una clase `c2` que extiende de `c1` y se instancia `o2`.

```
class c2 extends c1
  field z
  field w
  method initialize () 1

let o2 = new c2 ()
in
??
```
Cuando hago `c2` observe que:
```
#(struct:an-object c2 #(0 0 0 0))
```
El primer elemento es la clase `c2` y el segundo la clase `c1`. Se asume `object` como la lista vacía.

###
Este ejemplo extiende la jerarquía con `c3` que hereda de `c2`.

```
class c3 extends c2
  field a
  field b
  method initialize () 1

let o3 = new c3 ()
in
o3
```
En este caso vemos:
```
#(struct:an-object c3 #(0 0 0 0 0 0))
```
Entonces nos va a construir lo siguiente:
graph TD
    A["C3 (0 0 0 0 0 0)"]
Los primeros elementos del vector representan la clase de mayor jerarquía `c1`, y los últimos de la menor jerarquía que sería `c3`. Esta implementación es utilizada por Java para representar los objetos.

###
Este es un ejemplo de clases más complejas con métodos y herencia.

```
class p1 extends object
  field a
  field b
  method initialize (x,y)
    begin
      set a = x;
      set b = y
    end
  method sum (x)
    begin
      set a = +(a,1);
      +(a,b,x, send self m1())
    end
  method m1 () 1

class p2 extends p1
  field c
  field d
  field e
  method initialize (x,y,z)
    begin
      super initialize (x,y);
      set c = +(x,1);
      set d = +(y,2);
      set e = z
    end
  method sum (s)
    +(c,d,e, send self m1())

class p3 extends p2
  field f
  field g
  method initialize (u)
    begin
      set f = u;
      set g = +(u,1);
      super initialize (u,+(u,1), +(u,2))
    end
  method sum (a)
    +(a, super sum (+(a,2)))
  method m1 () 10
```

###
Instanciación de los objetos `p1`, `p2`, `p3` y ejecución de sus métodos `sum`.

```
let o1 = new p1 (2,3)
    o2 = new p2 (3,4,5)
    o3 = new p3 (6)
in
  list (send o1 sum (2),
        send o2 sum (3),
        send o3 sum (4))
```
Esto da:
```
(9 16 38)
```

###
Vamos a dibujar los ambientes de los llamados de los objetos, comenzamos con `o1` en `o1 = new p1(2,3)`.
graph TD
    A["p1 (2,3)"]
```
#(struct:an-object p1 #(2 3))
```
Vamos con `o2 = new p2(3,4,5)`.
graph TD
    A["p2 3 4 4 6 5"]
```
#(struct:an-object p2 #(3 4 4 6 5))
```
Vamos con `o3 = new p3(6)`.
graph TD
    A["p3 (6 7 7 9 8 6 7)"]
```
#(struct:an-object p3 #(6 7 7 9 8 6 7))
```
Aquí `6,7` son de `p1`, `7,9,8` son de `p2` y `6,7` son de `p3`.

###
**Ejecución de métodos**
Cuando se invoca un método se tiene un diagrama de clases independiente (propiedad de encapsulación), en este se tiene cada llamado es independiente, llamar dos métodos no se afectan entre sí, a excepción si uno de ellos ha modificado los campos del objeto.

**Llamado `send o1 sum(2)`**
flowchart LR
    A["empty-env"]
    B["env1
    a,b
    2,3
    "]
    C["envsum
    x,super,self
    2,object,o1
    "]
    A --> B
    B --> C
Sobre el ambiente `envsum` vamos a ejecutar:
```
begin
  set a = +(a,1);
  +(a,b,x, send self m1())
end
```
Al hacer `set a = +(a,1)` cambia la representación del objeto:
flowchart LR
    A["p1 (3,3)"]
Cambia el ambiente `env1`:
flowchart LR
    A["empty-env"]
    B["env1
    a,b
    3,3
    "]
    C["envsum
    x,super,self
    2,object,o1
    "]
    A --> B
    B --> C
Cuando ejecutamos `+(a,b,x, send self m1())`:
`+(3,3,2, send o1 m1())`
`send o1 m1()`:
flowchart LR
    A["empty-env"]
    B["env1
    a,b
    3,3
    "]
    C["envm1
    super,self
    object,o1
    "]
    A --> B
    B --> C
`+(3,3,2,1) = 9`

**Llamado `send o3 sum(4)`**

Continuando con la ejecución de `send o3 sum(4)`:

`send o3 m1()`:

Por lo que al evaluar: `+(7,9,8, 10) = 34`
Luego: `+(4, 34) = 38`
El comportamiento es exactamente el mismo, solo que cambia la representación de los objetos.
