Los condicionales permiten un flujo de ejecución condicional: si se cumple una condición, se ejecuta cierto número de instrucciones; si no se cumple, se ejecutan otras. Para los condicionales, necesitamos booleanos porque es necesario evaluar si una condición es verdadera o falsa.
Los valores expresados y denotados en este contexto son números y booleanos.

###
La gramática se modifica para incluir la expresión condicional `if`:

`<expression> ::= "if" <expresion> "then" <expresion> "else" <expresion>`

Esto se representa para la construcción de la expresión condicional (`if-exp`) en la gramática. Un ejemplo de cómo se estructuraría sería:

`if (> x 3) then 4 else 5`

###
La función `eval-expression` se modifica para manejar las expresiones `if-exp`. Cuando se evalúa una `if-exp` con una expresión de condición (`cond-exp`), una expresión para el caso verdadero (`true-exp`) y una para el caso falso (`false-exp`), se sigue este proceso:

1. Se evalúa la `cond-exp` para obtener un `cond-value`.
2. Se verifica que `cond-value` sea un booleano. Si no lo es, se lanza un error.
3. Si `cond-value` es verdadero, se evalúa `true-exp`.
4. Si `cond-value` es falso, se evalúa `false-exp`.

Este es el fragmento de código que implementa esta lógica (dentro de un `cond` o similar):

```racket
(if-exp (cond-exp true-exp false-exp)
        (let ((cond-value (eval-expression cond-exp env)))
          (if (boolean? cond-value)
              (if cond-value
                  (eval-expression true-exp env)
                  (eval-expression false-exp env))
              (eopl:error "The condition must be a boolean " cond-exp))))
```
