###
Este documento describe la implementación de un intérprete para un lenguaje de programación con asignaciones, procedimientos, y recursión, utilizando la biblioteca `sllgen` de EOPL para el análisis léxico y sintáctico.

La especificación léxica define los tokens que el analizador léxico reconocerá. Esto incluye comentarios (ignorados), espacios en blanco (ignorados), números (positivos y negativos) e identificadores (letras seguidas de letras o dígitos).

```racket
(lang eopl)

; Especificación léxica: define tokens y patrones regex
(define lexical
  '(
    (comment ("%" (arbno (not #\newline))) skip)
    ; Comentarios: % hasta newline (se ignoran)
    (whitespace (whitespace) skip)
    ; Espacios en blanco (se ignoran)
    (number (digit (arbno digit)) number)
    ; Números positivos: dígitos
    (number ("-" digit (arbno digit)) number)
    ; Números negativos: - seguido de dígitos
    (identifier (letter (arbno (or letter digit))) symbol)
    ; Identificadores: letra seguida de letras/dígitos
    ))
```

###
La gramática define la estructura sintáctica del lenguaje, especificando cómo se combinan los tokens para formar expresiones y programas válidos. Un programa es simplemente una expresión. Las expresiones básicas incluyen variables, literales numéricos, y booleanos `true` y `false`.

```racket
; Gramática para el parser (producciones y constructores del AST)
(define grammar
  '(
    (program (expression) a-program)
    ; Programa: una expresión
    (expression (identifier) var-exp)
    ; Expresión: variable
    (expression (number) lit-exp)
    ; Expresión: literal numérico
    (expression ("true") true-exp)
    ; Expresión: booleano true
    (expression ("false") false-exp)
    ; Expresión: booleano false
    ))
```

###
La gramática también soporta expresiones más complejas como llamadas a primitivas, condicionales `if-then-else`, y bloques `let` para definir variables locales.

```racket
(define grammar
  '(
    ... ; Definiciones previas omitidas para brevedad

    (expression (primitive "(" (separated-list expression ",") ")") prim-exp)
    ; Expresión primitiva con argumentos
    (expression ("if" expression "then" expression "else" expression) if-exp)
    ; Expresión condicional
    (expression ("let" (arbno identifier "=" expression) "in" expression) let-exp)
    ; Bloque let para definir variables
    ))
```

###
Además, la gramática incluye soporte para procedimientos (`proc`), invocaciones de procedimientos (`app-exp`), procedimientos recursivos (`letrec`), secuencias de expresiones (`begin`) y asignación a variables (`set`).

```racket
(define grammar
  '(
    ... ; Definiciones previas omitidas para brevedad

    (expression ("proc" "(" (separated-list identifier ",") ")" expression) proc-exp)
    ; Expresión de procedimiento (lambda)
    (expression ("letrec" (arbno identifier "(" (separated-list identifier ",") ")" "=" expression) "in" expression) letrec-exp)
    ; Bloque letrec para procedimientos recursivos
    (expression ("set" identifier "=" expression) set-exp)
    ; Expresión de asignación
    (expression ("begin" expression (arbno ";" expression) "end") begin-exp)
    ; Bloque begin para secuenciar expresiones
    (expression ("(" expression (arbno expression) ")") app-exp)
    ; Expresión de aplicación de procedimiento
    ))
```

###
Las primitivas soportadas por el lenguaje incluyen operaciones aritméticas (suma, resta, multiplicación, división), lógicas (AND, OR), y de comparación (menor, menor o igual, mayor, mayor o igual, igualdad, desigualdad).

```racket
(define grammar
  '(
    ... ; Definiciones previas omitidas para brevedad

    (primitive ("+") add-prim)
    ; Primitiva: suma
    (primitive ("-") sub-prim)
    ; Primitiva: resta
    (primitive ("*") prod-prim)
    ; Primitiva: multiplicación
    (primitive ("/") div-prim)
    ; Primitiva: división
    (primitive ("and") and-prim)
    ; Primitiva: and lógico
    (primitive ("or") or-prim)
    ; Primitiva: or lógico
    (primitive ("<") less-prim)
    ; Primitiva: menor que
    (primitive ("<=") lesseq-prim)
    ; Primitiva: menor o igual
    (primitive (">") more-prim)
    ; Primitiva: mayor que
    (primitive (">=") moreeq-prim)
    ; Primitiva: mayor o igual
    (primitive ("==") eq-prim)
    ; Primitiva: igualdad
    (primitive ("!=") neq-prim)
    ; Primitiva: desigualdad
    ))
```

###
La biblioteca `sllgen` se utiliza para generar automáticamente los tipos de datos (datatypes) que representan el Árbol de Sintaxis Abstracta (AST) a partir de las especificaciones léxica y gramatical. Luego, se definen las funciones `scanner` y `parser` que convierten una cadena de texto en una lista de tokens y luego en un AST, respectivamente.

```racket
;; Generar datatypes automáticamente a partir de lexical/grammar
(sllgen:make-define-datatypes lexical grammar)

; Scanner: convierte string en lista de tokens
; scanner -> string -> lista de tokens
(define scanner
  (lambda (program)
    ((sllgen:make-string-scanner lexical grammar) program)))

; Parser: convierte string en AST usando la gramática
; parser: string -> ast
(define parser
  (lambda (program)
    ((sllgen:make-string-parser lexical grammar) program)))
```

###
El entorno (`environment`) es una estructura de datos crucial para el intérprete, ya que almacena las asociaciones entre identificadores y sus valores. Se define como un tipo de dato con tres constructores:
*   `empty-env`: representa un entorno vacío.
*   `extend-env`: extiende un entorno existente con una lista de nuevos identificadores y sus valores correspondientes.
*   `extend-recursively-env`: utilizado para extender el entorno con definiciones de procedimientos recursivos.

```racket
; Definición del entorno (environment) como estructura de datos
(define-datatype environment environment?
  (empty-env)
  ; Entorno vacío
  (extend-env
   ; Extender entorno con nuevos bindings
   (lid (list-of symbol?))
   ; Lista de identificadores
   (lval vector?)
   ; Lista de valores
   (old-env environment?))
  ; Entorno anterior
  (extend-recursively-env
   (procname (list-of symbol?))
   (argss (list-of (list-of symbol?)))
   (bodies (list-of expression?))
   (old-env environment?)))

; Predicado value?: cualquier valor es válido (implementación simplificada)
(define value?
  (lambda (v) #T))
```

###
Las funciones `apply-env` y `apply-env-ref` son responsables de buscar el valor o la referencia de un identificador dentro de un entorno. `apply-env` devuelve el valor directamente, mientras que `apply-env-ref` devuelve una referencia mutable al valor, lo cual es necesario para la asignación (`set!`). La búsqueda se realiza de forma recursiva, primero en el entorno actual y luego en el entorno anterior (`old-env`).

```racket
; Aplicar entorno: busca un símbolo en el entorno y retorna su valor
; apply-env: environment -> valor
(define apply-env
  (lambda (env var)
    (deref (apply-env-ref env var))))

; apply-env-ref: environment -> referencia
(define apply-env-ref
  (lambda (env var)
    (cases environment env
      (empty-env ()
        (eopl:error "Variable not found" var))
      ; Error si no se encuentra
      (extend-env (lid vec old-env)
        (letrec ((search-var
                  ; Búsqueda recursiva en la lista de bindings
                  (lambda (lid lval [pos 0])
                    (cond
                      [(null? lid) (apply-env-ref old-env var)]
                      ; Buscar en entorno anterior
                      [(equal? (car lid) var) (a-ref pos vec)]
                      ; Voy a generar una referencia cuando encuentro la variable
                      [else (search-var (cdr lid) vec (+ pos 1))])))
                  ; Continuar búsqueda
                  )
          (search-var lid vec)))
      (extend-recursively-env (procnames llargs bodies old-env)
        (letrec ( (search-proc
                    (lambda (procs args bodies)
                      (cond
                        [(null? procs) (apply-env-ref old-env var)]
                        [(eqv? (car procs) var) (a-ref 0 (list->vector (list (closure (car args) (car bodies) env))))]
                        [else (search-proc (cdr procs) (cdr args) (cdr bodies))]))))
          (search-proc procnames llargs bodies))))))
```

###
La función `eval-program` es el punto de entrada para la evaluación de un programa. Toma un AST de programa y evalúa su expresión principal en un entorno inicial predefinido. El `init-env` define algunas variables (`x, y, z`, `a, b, c`) con valores numéricos iniciales.

```racket
; Evaluar programa: punto de entrada, evalúa la expresión del programa
; eval-program: program -> value
(define eval-program
  (lambda (pgm)
    (cases program pgm
      (a-program (exp)
        (eval-expression exp init-env))
      ; Evalúa la expresión con entorno inicial
      )))

; Entorno inicial predefinido (variables x,y,z y a,b,c con valores numéricos)
(define init-env
  (extend-env
   '(x y z)
   (list->vector '(1 2 3))
   (extend-env
    '(a b c)
    (list->vector '(4 5 6))
    (empty-env))))
```

###
La función `eval-expression` es el corazón del intérprete, realizando la evaluación recursiva de las expresiones.
*   `var-exp`: busca el valor de la variable en el entorno.
*   `lit-exp`: retorna el valor literal (número).
*   `true-exp` y `false-exp`: retornan los booleanos `#t` y `#f`.
*   `prim-exp`: evalúa los argumentos y luego aplica la función primitiva correspondiente.

```racket
; Evaluar expresión: recursivamente evalúa según el tipo de expresión
; eval-expression: expression -> value
(define eval-expression
  (lambda (exp env)
    (cases expression exp
      (var-exp (id)
        (apply-env env id))
      ; Variable: busca en entorno
      (lit-exp (datum)
        datum)
      ; Literal: retorna el número
      (true-exp ()
        #t)
      ; True: retorna #t
      (false-exp ()
        #f)
      ; False: retorna #f
      (prim-exp (prim lrands)
        ; Primitiva: evalúa argumentos y aplica
        (let ((values (map (lambda (x) (eval-expression x env)) lrands)))
          (apply-primitive prim values)))
      )))
```

###
Continuando con `eval-expression`:
*   `if-exp`: evalúa la condición. Si es un booleano y es `#t`, evalúa la rama "then"; de lo contrario, evalúa la rama "else". Lanza un error si la condición no es booleana.
*   `let-exp`: evalúa las expresiones de los enlaces, crea un nuevo entorno extendido con estos enlaces y evalúa el cuerpo de la expresión `let` en este nuevo entorno.

```racket
(define eval-expression
  (lambda (exp env)
    (cases expression exp
      ... ; Definiciones previas omitidas para brevedad

      (if-exp (cond-exp true-exp false-exp)
        (let ((cond-value (eval-expression cond-exp env)))
          (if (boolean? cond-value)
              (if cond-value
                  (eval-expression true-exp env)
                  (eval-expression false-exp env))
              (eopl:error "The condition must be a boolean " cond-exp))))
      (let-exp (lid lexpr expr)
        (let ((vexpr (map (lambda (x) (eval-expression x env)) lexpr)))
          (eval-expression expr (extend-env lid (list->vector vexpr) env))))
      )))
```

###
En `eval-expression`, los procedimientos y sus aplicaciones se manejan de la siguiente manera:
*   `proc-exp`: crea un `closure` (cierre), que encapsula los parámetros, el cuerpo del procedimiento y el entorno en el que fue definido.
*   `app-exp`: evalúa el operador (que debe ser un procedimiento) y los operandos. Si el operador es un cierre válido y el número de argumentos coincide, se evalúa el cuerpo del procedimiento en un nuevo entorno extendido con los argumentos pasados y el entorno del cierre.

```racket
(define eval-expression
  (lambda (exp env)
    (cases expression exp
      ... ; Definiciones previas omitidas para brevedad

      (proc-exp (lid exp)
        (closure lid exp env))
      (app-exp (rator rands)
        (let ((procv (eval-expression rator env))
              (vrands (map (lambda (x) (eval-expression x env)) rands)))
          (if (and (procVal? procv) (= (length (procVal->lid procv)) (length vrands)))
              (cases procVal procv
                (closure (lid exp old-env)
                  (eval-expression exp (extend-env lid (list->vector vrands) old-env))))
              (eopl:error "Not a procedure or incorrect of number of args"))))
      )))
```

###
La expresión `letrec` en `eval-expression` permite definir procedimientos recursivos.
*   `letrec-exp`: crea un entorno extendido recursivamente. Esto significa que los procedimientos definidos dentro del `letrec` pueden referenciarse entre sí y a sí mismos. Luego, evalúa la expresión principal del cuerpo (`expn`) dentro de este entorno.

```racket
(define eval-expression
  (lambda (exp env)
    (cases expression exp
      ... ; Definiciones previas omitidas para brevedad

      ;; Manejo de expresiones letrec en eval-expression
      (letrec-exp (procnames llargs bodies expn)
        ; Componentes de la expresión letrec
        (eval-expression
         expn
         ; Expresión del cuerpo principal (después de "in")
         (extend-recursively-env
          ; Extender el ambiente con definiciones recursivas
          procnames
          ; Lista de nombres de procedimientos
          llargs
          ; Lista de listas de argumentos (parámetros formales)
          bodies
          ; Lista de cuerpos de los procedimientos
          env)))
        ; Ambiente actual (base para la extensión)
      )))
```

###
Las expresiones `begin` y `set` se manejan en `eval-expression` de la siguiente manera:
*   `begin-exp`: evalúa secuencialmente una lista de expresiones. El valor retornado es el resultado de la última expresión.
*   `set-exp`: evalúa la expresión del lado derecho, luego actualiza el valor de la referencia asociada al identificador en el entorno usando `setref!`. Retorna `void` para indicar que no hay un valor significativo.

```racket
(define eval-expression
  (lambda (exp env)
    (cases expression exp
      ... ; Definiciones previas omitidas para brevedad

      (begin-exp (exp lexp)
        (letrec ((evaluation
                  (lambda (lexp)
                    (cond
                      [(null? (cdr lexp)) (eval-expression (car lexp) env)]
                      [else
                       (begin
                         (eval-expression (car lexp) env)
                         (evaluation (cdr lexp)))]))))
          (evaluation (cons exp lexp))))
      (set-exp (id exp)
        (begin
          (setref! (apply-env-ref env id) (eval-expression exp env))
          (void)))
      (else "Not implemented yet")) ; Default (no debería ocurrir)
    ))
```

###
La función `operation` es una utilidad genérica para aplicar una función binaria acumulativa a una lista de valores, comenzando con un acumulador inicial. Esto es útil para operaciones como la suma o el producto que pueden tomar múltiples argumentos.

```racket
; Operación genérica: aplica una función binaria acumulativa a una lista
; operation: (T,T)->T, lista de valores, T -> value
(define operation
  (lambda (f lst acc)
    (cond
      [(null? lst) acc]
      ; Caso base: retorna acumulador
      [else (operation f (cdr lst) (f (car lst) acc))])))
    ; Aplica f y acumula
```

###
La función `apply-primitive` toma un tipo de primitiva y una lista de valores, y ejecuta la operación correspondiente. Maneja operaciones aritméticas, lógicas y de comparación. Para operaciones con múltiples argumentos (como `+`, `*`, `and`, `or`), usa la función `operation`. Para las binarias (`-`, `/`, comparaciones), opera sobre los dos primeros elementos.

```racket
; Aplicar primitiva: ejecuta la operación correspondiente sobre los valores
; apply-primitive: primitive, lista valores -> value
(define apply-primitive
  (lambda (prim values)
    (cases primitive prim
      (add-prim () (operation + values 0))
      ; Suma acumulativa
      (sub-prim () (- (car values) (operation + (cdr values) 0)))
      ; Resta (first - sum(rest))
      (prod-prim () (operation * values 1))
      ; Producto acumulativo
      (div-prim () (/ (car values) (operation * (cdr values) 1)))
      ; División (first / product(rest))
      (and-prim () (operation (lambda (a b) (and a b)) values #t))
      ; AND acumulativo
      (or-prim () (operation (lambda (a b) (or a b)) values #f))
      ; OR acumulativo
      (less-prim () (< (car values) (cadr values)))
      ; < (2 args)
      (lesseq-prim () (<= (car values) (cadr values)))
      ; <= (2 args)
      (more-prim () (> (car values) (cadr values)))
      ; > (2 args)
      (moreeq-prim () (>= (car values) (cadr values)))
      ; >= (2 args)
      (eq-prim () (= (car values) (cadr values)))
      ; == numérico (2 args)
      (neq-prim () (not (= (car values) (cadr values)))))))
    ; != numérico (2 args)
```

###
Los procedimientos se representan mediante el tipo de dato `procVal`, específicamente con el constructor `closure`. Un cierre encapsula:
*   `lid`: la lista de identificadores (parámetros formales).
*   `exp`: el cuerpo de la expresión del procedimiento.
*   `old-env`: el entorno léxico en el que se definió el procedimiento.
También se define un extractor `procVal->lid` para obtener los identificadores de los parámetros de un cierre.

```racket
; Closure
; Representación de los procedimientos
(define-datatype procVal procVal?
  (closure
   (lid (list-of symbol?))
   (exp expression?)
   (old-env environment?)))

;;Extractor para lid en un closure
(define procVal->lid
  (lambda (cls)
    (cases procVal cls
      (closure (lid exp old-env) lid))))
```

###
Para implementar la asignación (`set!`), se utiliza un sistema de referencias mutables.
*   El tipo de dato `reference` con el constructor `a-ref` almacena la posición de un valor dentro de un vector, que a su vez está contenido en el entorno.
*   `deref` y `primitive-deref`: toman una referencia y devuelven el valor almacenado en ella.
*   `setref!` y `primitive-setref!`: toman una referencia y un nuevo valor, y actualizan el valor en la posición indicada por la referencia.

```racket
;;References
(define-datatype reference reference?
  (a-ref
   (position integer?)
   (vec vector?)))

;;dref: reference -> value
(define deref
  (lambda (ref)
    (primitive-deref ref)))

;primitive-refef: reference -> valor
(define primitive-deref
  (lambda (ref)
    (cases reference ref
      (a-ref (pos vec)
        (vector-ref vec pos)))))

;setref
;Setref es un procedimiento para cambiar el valor de una referencia
(define setref!
  (lambda (ref val)
    (primitive-setref! ref val)))

(define primitive-setref!
  (lambda (ref val)
    (cases reference ref
      (a-ref (pos vec)
        (vector-set! vec pos val)))))
```

###
Finalmente, se define un tipo de dato `nulltype` con el constructor `void` para representar un valor nulo o sin un resultado significativo (utilizado por ejemplo, después de una asignación). El intérprete interactivo se inicia utilizando `sllgen:make-rep-loop`, proporcionando un prompt `-->` y las funciones de evaluación y parsing.

```racket
;;defininición del valor nulo
(define-datatype nulltype nulltype?
  (void))

; Intérprete interactivo: loop de lectura-evaluación-impresión
(define interpreter
  (sllgen:make-rep-loop
   "-->"
   ; Prompt
   eval-program
   ; Función de evaluación
   (sllgen:make-stream-parser lexical grammar)))
   ; Parser para input stream

; Iniciar el intérprete
(interpreter)
```
