Las ligaduras locales permiten introducir nuevas ligaduras, teniendo en su alcance léxico. Se entiende que las ligaduras del ambiente inicial tienen alcance global; sin embargo, pueden ser ocultadas o "shadowing" con ligaduras locales.

Un ejemplo de una ligadura local con `let`:
```
let x = 8
    y = 9
    z = +(x,1)
in +(x,y,z)
```

###
Tener en cuenta que `let` tiene dos áreas:
1.  **Área de declaración:** `identifier = expression`. Aquí se toma el ambiente anterior.
2.  **El área del `let` (después del `in`):** A partir de este punto, empiezan a existir las ligaduras declaradas.

Concepto: Al crear una ligadura local, se debe extender el ambiente actual incluyendo las nuevas ligaduras. Para el ejemplo anterior, el flujo de ambientes se vería así:

###
Para soportar la estructura de ligaduras locales `let`, se necesita una modificación en la gramática de las expresiones.

La modificación a la gramática para incluir `let-exp` es la siguiente:
```
(expression
 (let
  (arbno identifier "=" expression)
  in
  expression)
 let-exp)
```

###
También se debe modificar la función `eval-expression` para manejar las ligaduras locales.

La modificación a `eval-expression` para `let-exp` se implementa como:
```racket
(let-exp (lid lexpr expr)
  (let ((vexpr (map (lambda (x)
                       (eval-expression x env))
                     lexpr)))
    (eval-expression expr
      (extend-env lid vexpr env))))
```

###
Consideremos un ambiente inicial `(extend-env '(x y z) '(1 2 3) (extend-env '(a b c) '(4 5 6) (empty-env)))`.

Aquí hay un ejemplo más complejo de anidación de ligaduras `let`:
```
let x = let a = 3
            b = 4
            c = 5
        in +(x,y,z)
    y = +(y,x)
    z = *(z,
          let p = x
          in +(p,z))
in
let m = let f = let f = +(x,y)
                in f
        in +(f,z)
    n = +(a,b)
in +(x,y,z,m,n)
```
El resultado de evaluar esta expresión es `51`.
