####
**Expresiones de tipo en el lenguaje**

Para la implementación de un interpretador con chequeo de tipos, el lenguaje debe soportar expresiones que incluyan las siguientes formas:

- `proc` con argumentos tipados: `proc (int x) ...`
- Procedimientos de orden superior: `proc ((int * int -> int) x) ...` y `proc ((int -> (int * int -> int)) x) ...`
- Definiciones `letrec` con tipos explícitos: `letrec int f (int x, int y) = ...` y `(int * int -> int) g (int x) = ...`

####
**Modificaciones en la gramática para tipos**

La gramática del lenguaje se extiende para incluir la sintaxis de los tipos en las expresiones `proc` y `letrec`, así como las definiciones de los `type-exp`:

```racket
(expression ("proc" "(" (separated-list type-exp identifier ",") ")" expression) proc-exp)
(expression ("letrec" (arbno type-exp identifier "(" (separated-list type-exp identifier ",") ")" "=" expression) "in" expression) letrec-exp)
(type-exp ("int") int-exp)
(type-exp ("bool") bool-exp)
(type-exp ("(" (separated-list type-exp "*") "->" type-exp ")") proc-type-exp)
```

####
**Representación de tipos en Racket**

Se distinguen dos categorías de tipos: `atomic-type`, que representa tipos primitivos no compuestos, y `proc-type`, que describe la firma de un procedimiento especificando los tipos de los argumentos y el tipo de retorno.

```racket
(define-datatype type type?
  (atomic-type (nombre symbol?))
  (proc-type (arg-type (list-of type?))
             (result-t type?)))
```

####
**Definición de tipos primitivos**

Utilizando la estructura definida anteriormente, se establecen los tipos base `int` y `bool` mediante el constructor `atomic-type`.

```racket
; int
(define int-type (atomic-type 'int))

; bool
(define bool-type (atomic-type 'bool))
```
