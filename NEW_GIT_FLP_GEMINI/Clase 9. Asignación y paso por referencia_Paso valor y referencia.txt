####
**Paso por valor**

En el paso por valor, al evaluar un procedimiento con variables en sus argumentos, se crea una copia de la referencia que es independiente de la original.

```racket
let f = proc (x, y) 
        begin 
          set x = + (x, 1); 
          set y = + (y, 1); 
          + (x, y) 
        end 
    a = 10 
    b = 20 
in (f a b)
```

**Representación de entorno en paso por valor**

```mermaid
graph LR
    E["Empty-env"]
    A["env0 
        f,a,b
        closure(...),10, 20
    "]
    B["envf
    x,y
    10,20   
    "]
    E --> A
    E --> B
```

En este contexto, `a` vale 10 y `b` vale 20. Las variables `x` e `y` actúan como nuevas referencias independientes.

####
**Paso por referencia**

Al utilizar paso por referencia, las variables deben cambiar si dentro del procedimiento se realizan modificaciones, afectando directamente a la variable original.

```racket
let f = proc (x, y) 
        begin 
          set x = + (x, 1); 
          set y = + (y, 1); 
          + (x, y) 
        end 
    a = 10 
    b = 20 
in (f a b)
```

En este caso, tras la ejecución del procedimiento, `a` es 11 y `b` es 21.

####
**Implementación: valores expresados y denotados**

La implementación distingue los siguientes tipos de valores:
- `Valores expresados`: `Numero`, `booleanos` y `procVal`.
- `Valores denotados`: Referencia a un `valor expresado`.

Existe una diferencia fundamental en la estructura:
- `Blanco directo`: Una referencia normal (valor).
- `Blanco indirecto`: Cuando se pasa una variable a un procedimiento.

```racket
let f = proc (x, y) 
        begin 
          set x = + (x, 1); 
          set y = + (y, 1); 
          + (x, y) 
        end 
    a = 10 
    b = 20 
in (f a b)
```

```mermaid
graph LR
    E["Empty-env"]
    A["env0 
        f,a,b
        dt(closure(...)),dt(10), dt(20)
    "]
    B["envf
    x,y
    id(a),id(b) 
    "]
    E --> A
    E --> B
```

Los `targets` indirectos permiten relacionar las variables del llamado de `f` con las que se evalúan dentro del procedimiento.

####
**Paso por referencia con expresiones**

Cuando se pasan expresiones en lugar de variables simples, el comportamiento cambia.

```racket
let f = proc (x, y) 
        begin 
          set x = + (x, 1); 
          set y = + (y, 1); 
          + (x, y) 
        end 
    a = 10 
    b = 20 
in (f a + (a, b))
```

```mermaid
graph LR
    E["Empty-env"]
    A["env0 
        f,a,b
        dt(closure(...)),dt(10), dt(20)
    "]
    B["envf
    x,y
    id(a),dt(30)    
    "]
    E --> A
    E --> B
```

Al evaluar un procedimiento:
- Se genera un `target directo` si se pasa un valor (se genera una nueva referencia).
- Se genera un `target indirecto` si se pasa una variable (se establece una conexión).

####
**Lógica de evaluación de argumentos**

La función `eval-rand` determina si el argumento pasado es un valor o una variable para asignar el tipo de `target` correspondiente, evitando crear `targets` indirectos apuntando hacia otros `targets` indirectos.

```racket
(define eval-rand
  (lambda (rand env)
    (cases expression rand
      (var-exp (id)
        (let ((ref (apply-env-ref env id)))
          (indirect-target
            (cases target (primitive-deref ref)
              (direct-target (e) ref)
              (indirect-target (ref1) ref1)))))
      (else
        (direct-target (eval-expression rand env))))))
```

####
**Ejemplo de efectos secundarios**

En el siguiente ejemplo, el cambio en `x` dentro del procedimiento se refleja en la variable original `b`.

```racket
let f = proc (x) 
        begin 
          set x = + (x, 10); 
          x 
        end 
    b = 20 
in begin (f b); b end
```

####
**Análisis de múltiples llamadas**

Análisis de la ejecución con variables `x, y, z`:

```racket
let x = 10 y = 20 z = 30
    f = proc (a, b, c) 
        begin 
          set a = + (a, 1); 
          set b = + (b, 2); 
          set c = + (c, 3); 
          + (a, b, c) 
        end
in let g = proc (m, n, o) 
           begin 
             (f m n o); 
             (f n m o); 
             (f o m n); 
             + (x, y, z) 
           end
   in (g x y + (z, 2))
```

**Secuencia de cambios:**
1. `(f m n o)`: `x`=11, `y`=22, `z`=33.
2. `(f n m o)`: `y`=23, `x`=13, `z`=36.
3. `(f o m n)`: `z`=37, `x`=15, `y`=26.
Resultado final: 78.
