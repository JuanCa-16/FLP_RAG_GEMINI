####
**Definición del tipo de dato `environment`**

Para representar entornos en `Racket` utilizando `eopl`, se define una estructura que permite gestionar el almacenamiento de variables y sus valores.

```racket
(define-datatype environment environment?
  (empty-env)
  (extend-env
   (lid (list-of symbol?))
   (lval (list-of value?))
   (old-env environment?)))
```

La gramática para esta estructura es:
- `<environment> ::= '() | empty-env()`
- `<environment> ::= <symbol>* <values>* <environment> | extend-env(lid lval old-env)`

Se define también un predicado `value?` que, en este contexto básico, acepta cualquier elemento.

```racket
(define value?
  (lambda (x) #t))
```

####
**Instanciación de un entorno de ejemplo**

El entorno `env1` se construye mediante el anidamiento de `extend-env`, permitiendo enlazar múltiples símbolos con sus respectivos valores y manteniendo una referencia al entorno anterior.

```racket
(define env1
  (extend-env '(x y z)
              '(1 2 3)
              (extend-env '(a b c)
                          '(4 5 6)
                          (empty-env))))
```

En esta estructura, el entorno superior contiene las variables `x`, `y` y `z` con los valores `1`, `2` y `3`, respectivamente, mientras que el entorno inferior contiene las variables `a`, `b` y `c` con los valores `4`, `5` y `6`.

####
**Implementación de `apply-env`**

La función `apply-env` busca el valor asociado a una variable `var` dentro del entorno `env`. Si la variable no se encuentra en el entorno actual, la función realiza una búsqueda recursiva en `old-env`.

```racket
(define apply-env
  (lambda (env var)
    (cases environment env
      (empty-env ()
                 (eopl:error "No encuentro a " var))
      (extend-env (lid lval old-env)
                  (letrec ((search-var
                            (lambda (li lv)
                              (cond
                                [(null? li) (apply-env old-env var)]
                                [(equal? (car li) var) (car lv)]
                                [else (search-var (cdr li) (cdr lv))]))))
                    (search-var lid lval))))))
```

La función utiliza `cases` para distinguir entre un entorno vacío y un entorno extendido. Si se trata de un `extend-env`, una función auxiliar `search-var` recorre las listas de símbolos `lid` y valores `lval` en paralelo.

####
**Pruebas de acceso a variables en el entorno**

A continuación, se muestra el proceso de recuperación de valores almacenados en `env1` utilizando `apply-env`.

```racket
(display (apply-env env1 'x))
(display "\n")
(display (apply-env env1 'y))
(display "\n")
(display (apply-env env1 'z))
(display "\n")
(display (apply-env env1 'a))
(display "\n")
(display (apply-env env1 'b))
(display "\n")
(display (apply-env env1 'c))
(display "\n")
(display (apply-env env1 'w))
```

Al ejecutar estas instrucciones, se imprimirán los valores asociados a las variables presentes en el entorno. En caso de solicitar una variable inexistente, como `'w`, el sistema lanzará un error a través de `eopl:error`.
