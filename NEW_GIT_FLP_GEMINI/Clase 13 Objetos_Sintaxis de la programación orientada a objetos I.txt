####
**Fundamentos de la programación orientada a objetos: valores**

En esta extensión del lenguaje para soporte de programación orientada a objetos, se distinguen los siguientes elementos:

*   **Valores denotados**: Corresponden a `ref(valor expresado)`.
*   **Valores expresados**: Comprenden números, booleanos, `procVal` y objetos.

####
**Definición de una clase simple**

El siguiente ejemplo define la clase `c1`, la cual hereda de `object` y contiene campos para almacenar valores, junto con métodos para manipular su estado.

```racket
class c1 extends object
  field i
  field j

  method initialize(x)
  begin
    set i = x;
    set j = -(0, x)
  end

  method countup(d)
  begin
    set i = +(i, d);
    set j = -(j, d)
  end

  method getstate()
    list(i, j)
```

Al instanciar esta clase, el constructor `initialize` asigna el valor `x` al campo `i` y el valor `-x` al campo `j`. Los métodos `countup` y `getstate` permiten modificar y consultar el estado interno del objeto, respectivamente.

####
**Ejemplo de uso e instanciación de clases**

Para utilizar una clase, se realiza la creación de una instancia mediante el operador `new`, seguido de la invocación de métodos mediante el comando `send`.

```racket
let t1 = 0
    t2 = 0
    o1 = new c1(3)
in
begin
  set t1 = send o1 getstate();
  send o1 countup(2);
  set t2 = send o1 getstate();
  list(t1, t2)
end
```

En este caso, tras inicializar `o1` con el valor 3, el estado inicial es `list(3, -3)`. Tras ejecutar `countup(2)`, los valores se actualizan a 5 y -5, resultando en un valor de retorno final de `list(list(3, -3), list(5, -5))`.

####
**Estructura de árbol mediante POO**

Es posible construir estructuras de datos complejas mediante la composición de objetos. A continuación, se define un nodo (`node`) que puede contener otros nodos o hojas (`leaf`).

```racket
class node extends object
  field left
  field right
  field val

  method initialize(l, r, v)
  begin
    set left = l;
    set right = r;
    set val = v
  end

  method sum()
    +(send left sum(), send right sum(), val)

class leaf extends object
  field val

  method initialize(v)
  begin
    set val = v
  end

  method sum()
    val
```

####
**Construcción y ejecución de un árbol**

Utilizando las clases `node` y `leaf`, se puede instanciar una estructura jerárquica y calcular la suma total de sus valores.

```racket
let tree = new node(
  new node(new leaf(3), new leaf(5), 3),
  new node(new node(new leaf(4), new leaf(3), 10), new leaf(20), 19),
  14
)
in
  send tree sum()
```

####
**Despacho dinámico de métodos**

El concepto de **despacho dinámico** permite que el método a ejecutar sea determinado en tiempo de ejecución, basándose en el tipo real del objeto y no en el tipo declarado. Esto es fundamental para el `polimorfismo`. En el ejemplo anterior, el método `sum()` se ejecuta de forma recursiva sin que el sistema necesite conocer a priori si el objeto es un `node` o una `leaf`.

####
**Resumen de conceptos de sintaxis POO**

A continuación se detallan los componentes fundamentales de la estructura orientada a objetos:

*   **Clase**: Plantilla para crear objetos (ej. `class c1 extends object`).
*   **Campo**: Variable de instancia del objeto (ej. `field i`).
*   **Método**: Función asociada a un objeto (ej. `method sum() val`).
*   **Constructor**: Método que inicializa el objeto (ej. `method initialize(x)`).
*   **Instanciación**: Creación de un objeto a partir de una clase (ej. `new c1(3)`).
*   **Envío de mensaje**: Invocación de un método en un objeto (ej. `send o1 getstate()`).
*   **Herencia**: Relación entre clases padre e hijas (ej. `extends object`).
*   **Despacho dinámico**: Selección del método en tiempo de ejecución (ej. `send left sum()`).
*   **Composición**: Objetos que contienen otros objetos (ej. nodos que contienen hojas).
