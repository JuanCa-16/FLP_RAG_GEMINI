####

Para implementar las reglas de tipado de procedimientos, es necesario asegurar que cada argumento tenga un tipo asociado y que el tipo de la salida dependa del tipo de la expresiĂłn interna. Para el manejo de aplicaciones de procedimientos, utilizaremos un procedimiento auxiliar llamado `type-of-application`. Este debe verificar que el operador sea efectivamente un procedimiento antes de intentar aplicarlo. El proceso sigue estos pasos:

1. EvalĂşa el `rator` (operador) para obtener su tipo y verifica que sea un `proc-type`.
2. Verifica que el nĂşmero de argumentos esperados por el procedimiento coincida con el nĂşmero de argumentos proporcionados (`rands`).
3. Utiliza un ciclo `for-each` para validar que cada tipo del argumento esperado sea igual al tipo del operando correspondiente.
4. Si la validaciĂłn es exitosa, devuelve el tipo de resultado del `proc-type`.

En caso de error, el sistema debe reportar un mensaje indicando la discrepancia entre el tipo esperado y el tipo recibido utilizando una forma de manejo de errores, como `eopl:error`.

####

Dentro de `type-of-expression`, debemos aĂ±adir el caso correspondiente a `app-expression`. Esta funciĂłn invocarĂˇ a `type-of-application` enviando el tipo resultante de evaluar el operador y los tipos resultantes de evaluar cada uno de los operandos (`rands`).


####

Las primitivas del lenguaje (como `+`, `incrementar` o comparaciones) actĂşan como procedimientos. Para manejarlas, definimos `type-of-primitive`, el cual determina el tipo de la primitiva basĂˇndose en el valor y el nĂşmero de argumentos.

Para primitivas numĂ©ricas que reciben una cantidad variable de argumentos (a diferencia del libro, donde se restringen a dos), implementamos una funciĂłn recursiva llamada `primitive-numeric`:

```racket

  (cond
    [(null? rands) '()]
    [else (cons (int-type) (primitive-numeric (cdr rands)))]))
```

Esta funciĂłn construye una lista de `int-type` por cada argumento recibido. AsĂ­, una primitiva como `mayor` resultarĂˇ en un `proc-type` que recibe una lista de `int-type` y retorna un `bool-type`. Para operaciones unarias como `incrementar` o `decrementar`, se restringe la entrada a un Ăşnico `int-type`.

####

Finalmente, para procesar una lista de expresiones (como los argumentos de una aplicaciĂłn), se utiliza la funciĂłn `type-of-expressions`, la cual mapea `type-of-expression` sobre la lista de operandos:
