#### **Implicaciones de la asignación en los procedimientos**

Al introducir la asignación en un lenguaje de programación, surgen problemas con el manejo de procedimientos. Hasta ahora, el `ambiente` se entiende como una estructura que asocia identificadores a valores, compuesta por una lista de identificadores y una lista de valores, además de un enlace al ambiente superior.

Cuando incluimos procedimientos, aparece un problema de consistencia. Consideremos el siguiente ejemplo:
`f = proc(x) 10`
`f(10)`

Al invocar `f(10)`, surge la duda sobre a qué `x` se refiere el sistema. Para resolver esto, se utiliza la `clausura` (`closure`), que guarda una copia del ambiente existente al momento de crear el procedimiento. Sin embargo, al introducir la asignación (`set`), la clausura presenta una limitación: si el valor en el ambiente cambia, la copia guardada en la clausura no se actualiza, manteniendo el valor original.

#### **El problema de la clausura con asignación**

Analicemos el siguiente ejemplo para observar el fallo de consistencia:
```racket
let z = 0
in let f = proc(x) z
   in begin
        set z = 1
        f(2)
      end
```
Lo lógico sería que al ejecutar `f(2)`, el resultado fuera `1`, pues `z` fue modificado mediante un `set`. Sin embargo, el sistema devuelve `0`. Esto ocurre porque la clausura guardó el ambiente en el momento en que `z` valía `0`. Para solucionar esto, debemos introducir el concepto de **referencia**.

#### **Introducción a las referencias**

Para manejar la asignación correctamente, cambiaremos la forma en que las variables se vinculan a los valores. En lugar de que la variable guarde el valor directamente, la relación se hará a través de una `referencia` o `ubicación`. La variable apuntará a la dirección donde reside el elemento en memoria.

En esta arquitectura:
- El `L-valor` (`L-value`) es la ubicación o dirección (dónde está guardado el dato).
- El `R-valor` (`R-value`) es el valor almacenado en dicha ubicación.

La referencia actúa como el conector entre ambos. La analogía es la dirección de una casa (L-valor) frente a la casa misma (R-valor).

#### **Interfaz de referencias**

Para implementar las referencias, definimos un tipo de dato que contiene una posición (entero) y un vector donde reside el valor. La interfaz incluye procedimientos para interactuar con estas referencias:

- `aRef`: Constructor de la referencia.
- `deref`: Procedimiento observador que obtiene el valor almacenado en la referencia.
- `setref`: Procedimiento para cambiar el valor asociado a la referencia.

#### **Implementación de la estructura de datos**

Utilizamos un `data type` para definir la referencia, que contiene una posición y un vector:

```racket
(define-datatype referencia referencia?
  (a-ref (pos number?)
         (vec vector?)))
```

La función `deref` utiliza `vector-ref` para extraer el valor en la posición indicada:

```racket
(define (deref ref)
  (primitive-deref ref))

(define (primitive-deref ref)
  (cases referencia ref
    (a-ref (pos vec)
           (vector-ref vec pos))))
```

#### **Implementación de la asignación de referencias**

Para modificar el valor de una referencia, utilizamos `setref`. Esta función emplea `vector-set!` para actualizar el contenido del vector en la posición específica:

```racket
(define (setref ref val)
  (primitive-setref ref val))

(define (primitive-setref ref val)
  (cases referencia ref
    (a-ref (pos vec)
           (vector-set! vec pos val))))
```

Con esta interfaz, el sistema ya es capaz de gestionar cambios en memoria a través de referencias, permitiendo que las clausuras y las variables apunten a ubicaciones que pueden ser actualizadas dinámicamente. El siguiente paso consiste en integrar este mecanismo con los `ambientes` y el resto del interpretador.
