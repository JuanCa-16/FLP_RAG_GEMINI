####
Para implementar el manejo de `literal-expression` y `var-expression`, es necesario definir la estructura de los `ambientes`. Los ambientes se definen mediante un `define-datatype` que contempla dos variantes: `empty-env`, que representa un ambiente vacío, y `extended-env`, que almacena una lista de identificadores (`list-of-symbol`) y una lista de valores (`list-of-values`) correspondientes, además de una referencia al ambiente previo.

```racket
(define-datatype ambiente ambiente?
  (empty-env)
  (extended-env
   (ids (list-of symbol?))
   (vals (list-of scheme-value?))
   (env ambiente?)))
```

####
La función `apply-env` es la encargada de buscar un símbolo dentro de un ambiente determinado. Esta función discrimina entre los tipos de ambiente definidos: si es un `empty-env`, reporta que la variable no se encuentra; si es un `extended-env`, busca el identificador en la lista correspondiente y, de no hallarlo, delega la búsqueda de manera recursiva al ambiente al cual extiende.

```racket
(define apply-env
  (lambda (env sym)
    (cases ambiente env
      (empty-env ()
                 (eopl:error 'apply-env "No se encuentra la variable: ~s" sym))
      (extended-env (ids vals prev-env)
                    (let ((pos (buscar-posicion ids sym)))
                      (if (number? pos)
                          (list-ref vals pos)
                          (apply-env prev-env sym)))))))
```

####
Para facilitar la búsqueda dentro de la lista de identificadores en `apply-env`, se requiere una función auxiliar que retorne la posición del símbolo si existe, o un valor que indique que no se ha encontrado.

```racket
(define buscar-posicion
  (lambda (ids sym)
    (let loop ((ids ids) (pos 0))
      (cond
        ((null? ids) #f)
        ((eqv? (car ids) sym) pos)
        (else (loop (cdr ids) (+ pos 1)))))))
```

####
Para fines de prueba, se define un `ambiente-inicial` que contiene variables predefinidas. Este ambiente es un `extended-env` que permite validar la recuperación de datos mediante `apply-env`.

```racket
(define ambiente-inicial
  (extended-env '(x y z) '(1 2 3)
                (extended-env '(a b c) '(4 5 6)
                              (empty-env))))

;; Ejemplos de uso:
;; (apply-env ambiente-inicial 'x) ; retorna 1
;; (apply-env ambiente-inicial 'b) ; retorna 5
```

####
La evaluación de expresiones se realiza mediante la función `evaluar-expresion`, la cual recibe una expresión y un ambiente. Esta función utiliza `cases` para manejar las distintas variantes de la gramática. Para el caso de `program`, se invoca la evaluación inicial con el `ambiente-inicial`.

```racket
(define evaluar-programa
  (lambda (prog)
    (cases programa prog
      (a-program (exp)
                 (evaluar-expresion exp ambiente-inicial)))))
```

####
Dentro de `evaluar-expresion`, se implementan los casos iniciales: `literal-expression`, que devuelve el valor numérico directamente, y `var-expression`, que utiliza `apply-env` para obtener el valor asociado al identificador en el ambiente actual.

```racket
(define evaluar-expresion
  (lambda (exp env)
    (cases expression exp
      (literal-expression (num) num)
      (var-expression (id) (apply-env env id))
      (else (eopl:error 'evaluar-expresion "Tipo de expresión no implementado")))))
```
