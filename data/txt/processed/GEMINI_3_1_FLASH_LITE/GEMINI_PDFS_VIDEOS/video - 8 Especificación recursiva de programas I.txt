####

**Especificación de recursión de datos y programas**

Para la especificación de recursión de datos utilizamos la notación de `Backus-Naur Form` (`BNF`) y definiciones inductivas. Cuando pasamos a la especificación de programas, la definición inductiva sirve como guía para desarrollar los procedimientos que operan sobre dichos datos.

Toda función que trabaja sobre un tipo de datos debe considerar la estructura de este. Por ejemplo, una `lista` de enteros siempre posee una estructura definida: un `entero` seguido de otra `lista` de enteros, o una `lista vacía`. Para acceder a sus componentes utilizamos `car` y `cdr`. Por lo tanto, el diseño de la función debe contemplar ambos casos: el caso base y el caso recursivo.

####

**El caso base o condición de parada**

El caso base es aquel donde el problema alcanza su dimensión más pequeña, permitiendo entregar una respuesta inmediata sin realizar cálculos complejos ni llamadas recursivas. Esto se identifica mediante un símbolo terminal en la gramática.

Por ejemplo, al definir una función para sumar los elementos de una lista, el caso base ocurre cuando la lista está vacía. En este escenario, la respuesta es trivial: el valor de retorno es `0`, ya que no hay elementos que sumar.

```racket
(if (null? lista)
    0
    ; caso recursivo)
```

En este código, `null?` actúa como la condición de parada. Si se cumple, el procedimiento devuelve `0` y no se ejecutan más llamadas recursivas.

####

**El caso recursivo**

El caso recursivo se presenta cuando la estructura del dato contiene elementos que requieren un procesamiento posterior mediante el mismo procedimiento. Siguiendo la estructura de la lista (un `entero` seguido de una `lista` de enteros), el caso recursivo debe aprovechar esta composición.

Para realizar una suma de elementos, el procedimiento debe:
1. Extraer el primer elemento (`car`) de la lista.
2. Realizar un llamado recursivo con el resto de la lista (`cdr`).

Como la estructura de una lista garantiza que tras un `car` siempre encontraremos otra `lista`, el llamado recursivo permite procesar el resto de los elementos hasta alcanzar el caso base.

####

**Ejemplo: Función para contar elementos**

Para diseñar una función que cuente el número de elementos en una lista (`lislen`), debemos considerar la estructura gramatical: un símbolo terminal (lista vacía) y un caso recursivo (número seguido de una lista).

Si la lista no está vacía, sabemos que hay al menos un elemento. Por tanto, la lógica consiste en sumar `1` al resultado del llamado recursivo sobre el `cdr` de la lista.

```racket
(define (lislen lista)
  (if (null? lista)
      0
      (+ 1 (lislen (cdr lista)))))
```

En esta implementación, el caso base devuelve `0` cuando la lista está vacía. En el caso recursivo, la función suma `1` por el elemento actual (`car`) y se invoca a sí misma con el resto de la lista (`cdr`), asegurando que el problema se reduzca en cada iteración hasta llegar a la condición de parada.
