####
**Representación de ambientes con `set`**

Para representar los cambios en los ambientes durante la ejecución de expresiones `set`, observamos el siguiente caso: `let x = 2 in begin set x = 3; x end`. 

Al no especificarse un ambiente inicial, se parte de un `empty-env`. El proceso es el siguiente:
1. Se crea el ambiente con `x` vinculado al valor `2`. Este valor se almacena en un vector de memoria.
2. Al ejecutar `set x = 3`, se modifica el valor almacenado en la posición correspondiente a `x` dentro del vector, actualizándolo a `3`.

En este ejemplo, la operación `set` altera directamente el valor contenido en la referencia de `x` en el ambiente actual.

####
**Ejemplo de ambientes con procedimientos**

Consideremos un ejemplo más elaborado que involucra `procedimientos`: `let x = 100 in let p = proc (x) begin set x = (x + 1); x end in +(p(x), p(x))`.

El análisis del ambiente se realiza de la siguiente manera:
- Se inicia en `empty-env`.
- Se crea un ambiente donde `x` tiene el valor `100`.
- Se define `p` como una `clausura` que contiene el código del procedimiento.
- Al evaluar `+(p(x), p(x))`, los operandos se ejecutan de izquierda a derecha.
- Cada llamado a `p(x)` genera un nuevo ambiente donde el parámetro `x` se vincula al valor `100`.
- Dentro del procedimiento, la instrucción `set x = (x + 1)` afecta únicamente a la variable `x` del ambiente local creado por el llamado al procedimiento. Por lo tanto, `x` cambia de `100` a `101`.
- El primer llamado a `p(x)` retorna `101` y el segundo llamado también retorna `101`, resultando en una suma de `101 + 101 = 202`.

####
**Verificación de ejecución**

Para validar el comportamiento del `scope` y la mutación con `set`, observamos los siguientes resultados en el intérprete:

```racket
;; Primer caso
let x = 2 in begin set x = 3; x end
;; Resultado: 3

;; Segundo caso
let x = 100 in 
  let p = proc (x) begin set x = (x + 1); x end 
  in +(p(x), p(x))
;; Resultado: 202
```

Cada llamado a un `procedimiento` genera un ambiente nuevo. Es fundamental notar que el `set` modifica la variable local del ambiente más cercano donde se encuentra definida, sin afectar las variables del mismo nombre en ambientes superiores o en otros ámbitos de llamada. Es necesario tener cuidado con estos cambios en la representación de ambientes, especialmente al trabajar con `procedimientos recursivos`.
