####
**Implementación de `spant-type-expression`**

Para realizar la verificación de tipos, es necesario definir un procedimiento que calcule el tipo denotado por una expresión de tipo. Este procedimiento conecta la gramática con el sistema de tipos.

```racket
(define spant-type-expression
  (lambda (type-expression)
    (cases type-expression
      (int-type-expression () (int-type))
      (bool-type-expression () (bool-type))
      (proc-type-expression (arg-types result-type)
        (proc-type
          (spant-type-expressions arg-types)
          (spant-type-expression result-type))))))
```

El procedimiento `spant-type-expressions` se encarga de procesar una lista de tipos utilizando un `map`:

```racket
(define spant-type-expressions
  (lambda (type-expressions)
    (map (lambda (x) (spant-type-expression x))
         type-expressions)))
```

####
**Implementación del `checker`: `type-of-expression`**

El `checker` se implementa como un procedimiento que recibe una expresión y un ambiente de tipos, retornando el tipo correspondiente o un error. Las primeras cláusulas manejan literales y variables:

```racket
(define type-of-expression
  (lambda (exp env)
    (cases expression exp
      (literal-expression (n) (int-type))
      (var-expression (var) (apply-tenv env var))
      (bool-expression (b) (bool-type))
      (else (eopl-error "Tipo no soportado")))))
```

Para las variables, se utiliza `apply-tenv`, que funciona de forma similar a `apply-env` pero emite mensajes de error específicos para fallos en la verificación de tipos.

####
**Verificación de tipos en condicionales**

Para la estructura `if-expression`, se requiere que la expresión de prueba sea de tipo `bool-type` y que las ramas `true` y `false` compartan el mismo tipo.

```racket
(if-expression (test-exp true-exp false-exp)
  (let ([type-test (type-of-expression test-exp env)]
        [type-true (type-of-expression true-exp env)]
        [type-false (type-of-expression false-exp env)])
    (begin
      (check-equal-type! type-test (bool-type) test-exp)
      (check-equal-type! type-true type-false true-exp)
      type-true)))
```

####
**Procedimiento de utilidad `check-equal-type!`**

Esta función verifica si dos tipos son idénticos y, en caso contrario, genera un error utilizando `eopl-error`.

```racket
(define check-equal-type!
  (lambda (t1 t2 exp)
    (if (not (equal? t1 t2))
        (eopl-error "Los tipos no corresponden: ~s distinto a ~s en la expresión ~s"
                    (type-external-form t1)
                    (type-external-form t2)
                    exp))))
```

####
**Formateo de tipos para mensajes de error**

La función `type-external-form` transforma los tipos internos (representados como `struct` o tipos de datos) a un formato legible por el usuario, útil para la depuración de errores.

```racket
(define type-external-form
  (lambda (t)
    (cases type t
      (int-type () "int")
      (bool-type () "bool")
      (proc-type (arg-types result-type)
        (string-append "(" 
                       (string-join (map type-external-form arg-types) " * ")
                       " -> " 
                       (type-external-form result-type) 
                       ")")))))
```

Esta función realiza la "carpintería" necesaria para visualizar correctamente los tipos, especialmente en el caso de los procedimientos (`proc-type`), donde se requiere mostrar la lista de tipos de argumentos y el tipo de retorno.
