####

**Gramática para Clases y Objetos**

Para extender nuestra gramática con soporte para orientación a objetos, debemos establecer una estructura que permita la declaración de clases antes de la expresión principal del programa. La estructura propuesta es:

- `Program`: `ClassDecl*` `Expression`
- `ClassDecl`: `class` `identifier` `extends` `identifier` `{` `field*` `methodDecl*` `}`
- `MethodDecl`: `method` `identifier` `(` `identifier*` (`,` `identifier*`)* `)` `Expression`

Una clase se compone de un nombre, la herencia (mediante la palabra reservada `extends`), una serie de campos (`field`) y declaraciones de métodos. Cada método posee un identificador, una lista de parámetros y una expresión que define su comportamiento.

####

**Reglas de Producción para Expresiones de Objetos**

Para manipular objetos, invocarlos y realizar llamadas a clases superiores, añadimos las siguientes reglas de producción a las expresiones de nuestro lenguaje:

- `new` `identifier` `(` `expression*` (`,` `expression*`)* `)`: Crea una instancia de la clase especificada.
- `send` `expression` `identifier` `(` `expression*` (`,` `expression*`)* `)`: Invocación de un método sobre un objeto.
- `super` `identifier` `(` `expression*` (`,` `expression*`)* `)`: Llamado a un método definido en la clase padre.

Los valores denotados y expresados ahora incluyen números, `PROCVAL`, objetos y listas de valores.

####

**Ejemplo de Declaración de Clases y Semántica**

Consideremos la declaración de una clase `C1` que extiende `Object`. Cuando una clase no posee una superclase, utilizamos la palabra reservada `Object`.

```racket
class C1 extends Object {
  field i
  field j
  method initialize(i_val, j_val) {
    begin
      set i = i_val;
      set j = j_val
    end
  }
  method get_state() { list(i, j) }
  method con_up(n) {
    begin
      set i = i + n;
      set j = j - n
    end
  }
}
```

El método `initialize` actúa como constructor; se ejecuta automáticamente al crear el objeto mediante `new`. En este caso, `C1` contiene dos campos (`i`, `j`) y tres métodos.

####

**Creación y Manipulación de Objetos**

Utilizando la clase `C1` definida anteriormente, podemos instanciar objetos y enviar mensajes:

```racket
let o1 = new C1(3, -3)
in begin
  send o1 get_state();
  send o1 con_up(2);
  send o1 get_state()
end
```

Al ejecutar `new C1(3, -3)`, se invoca `initialize`, asignando `i = 3` y `j = -3`. Posteriormente, al invocar `con_up(2)`, el primer campo se incrementa en 2 y el segundo se decrementa en 2, resultando en `5` y `-5` respectivamente. El método `get_state` retorna el estado actual del objeto como una lista.

####

**Polimorfismo y Despacho Dinámico**

El polimorfismo permite que diferentes clases implementen un mismo método con comportamientos distintos. Consideremos una estructura de árbol:

```racket
class interior_node extends Object {
  field left
  field right
  method initialize(l, r) { begin set left = l; set right = r end }
  method sum() { send left sum() + send right sum() }
}

class leaf_node extends Object {
  field value
  method initialize(v) { set value = v }
  method sum() { value }
}
```

Cuando invocamos `send node sum()`, el programa no necesita conocer la clase específica del nodo (`interior_node` o `leaf_node`). Si es `interior_node`, delega la suma a sus hijos; si es `leaf_node`, retorna el valor. Este mecanismo de resolución automática es conocido como `despacho dinámico de métodos`.

####

**Uso de `self` para Invocación Interna**

El identificador `self` permite a un objeto enviar mensajes a sí mismo, funcionando de manera análoga al `this` en otros lenguajes de programación.

```racket
class odd_even extends Object {
  method even(n) {
    if zero?(n) then 1
    else send self odd(n - 1)
  }
  method odd(n) {
    if zero?(n) then 0
    else send self even(n - 1)
  }
}
```

Al utilizar `send self method_name(...)`, el sistema vincula el mensaje al objeto que recibe la invocación original, permitiendo que la lógica interna utilice el polimorfismo definido en la jerarquía de clases.

####

**Herencia y Ocultamiento de Campos**

La herencia permite definir clases basadas en otras existentes. Si una clase hija redefine un campo, el acceso al mismo se realiza sobre la versión más reciente en la jerarquía.

```racket
class C1 extends Object { field x }
class C2 extends C1 { field y }
```

Si invocamos métodos de `C2` que acceden a `x`, el sistema buscará primero en `C2`; si el campo no existe, escalará a `C1`. El `despacho dinámico` garantiza que, incluso si un método es definido en la clase padre, al ejecutarse mediante `self` en una clase hija, se respetará el contexto del objeto actual.

####

**Uso de `super` para Inicialización en Cadena**

La palabra reservada `super` permite invocar métodos de la clase padre, lo cual es fundamental para la inicialización jerárquica de objetos.

```racket
class colored_point extends point {
  field color
  method initialize(x, y, c) {
    begin
      super initialize(x, y);
      set color = c
    end
  }
}
```

Si no se invoca `super initialize`, los campos de la clase padre mantendrán su valor por defecto (`0`). El uso de `super` permite asegurar que cada nivel de la jerarquía configure sus propios atributos correctamente durante el proceso de creación del objeto.
