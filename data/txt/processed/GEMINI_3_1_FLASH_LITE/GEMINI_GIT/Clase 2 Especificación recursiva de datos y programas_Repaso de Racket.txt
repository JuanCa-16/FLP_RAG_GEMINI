####
**Definición de Racket**
`Racket` es un lenguaje de programación multiparadigma que permite trabajar con diversos enfoques, tales como funcional, procedural, orientado a objetos y basado en eventos. En este curso, nos enfocaremos específicamente en el paradigma funcional.

####
**Características de la sintaxis**
El lenguaje utiliza notación prefija, donde primero se especifica la función y posteriormente los argumentos. Además, emplea una estructura basada en paréntesis. Para definir el lenguaje en el archivo fuente, se utiliza la directiva `#lang eopl`.

```racket
#lang eopl

(define pollito 5)

(define gallinita (* pollito 2))
```

####
**Declaración de funciones con lambda**
En este enfoque, las funciones se declaran exclusivamente utilizando `lambda`, ya que las funciones son tratadas como valores.

```racket
(define funcion 
  (lambda (x y) 
    (+ x y)))

(funcion 1 2)
```

####
**Funciones con parámetros por defecto**
Es posible definir funciones con valores predeterminados para sus parámetros, de modo que si no se proporciona un argumento, se utilice el valor especificado.

```racket
(define funcionPro 
  (lambda (x [p 8]) 
    (+ x p)))

(funcionPro 4)   ; Retorna 12
(funcionPro 4 9) ; Retorna 13
```

####
**Estructuras condicionales**
Para el control de flujo, se utiliza la estructura `cond`, la cual evalúa múltiples condiciones y ejecuta la expresión correspondiente a la primera que resulte verdadera.

```racket
(define funcion 
  (lambda (x y)
    (cond
      [(> x 5) (+ x y)]
      [(and (> x 0) (< x 5)) (* x y)]
      [else (+ x y 2)])))
```

####
**Listas en Racket**
Las listas pueden construirse explícitamente mediante `list` o mediante el uso recursivo de `cons` para añadir elementos a una lista vacía `empty`. También se pueden definir listas literales utilizando el apóstrofo (`'`).

```racket
(list 1 2 3 5)

(cons 1 (cons 2 (cons 3 (cons 5 empty))))

'(1 2 3 4 a b (1 2 3))

(list 1 2 3 'a 'b (list 1 2 3))
```

####
**Operaciones sobre listas: `car` y `cdr`**
Para manipular listas, se utilizan las funciones `car` (que retorna la cabeza de la lista) y `cdr` (que retorna la cola, es decir, el resto de la lista).

```racket
(define l '(1 2 3 4 5))

(car l) ; Retorna 1
(cdr l) ; Retorna '(2 3 4 5)

(car (cdr l))             ; Retorna 2
(car (cdr (cdr l)))       ; Retorna 3
(cadr l)                  ; Retorna 2 (car del cdr)
(caddr l)                 ; Retorna 3 (car del cdr del cdr)
```

####
**Recursión sobre listas**
Es posible procesar listas mediante funciones recursivas que verifican si la lista está vacía utilizando `null?`.

```racket
(define sumarL
  (lambda (lst)
    (cond
      [(null? l) 0]
      [else (+ (car l) (sumarL (cdr l)))])))
```

####
**Recursión de cola**
La recursión de cola utiliza un acumulador (`acc`) para optimizar el proceso de suma de elementos de una lista.

```racket
(define sumarColita
  (lambda (lst [acc 0])
    (cond
      [(null? l) acc]
      [else (sumarL (cdr l) (+ acc (car l)))])))
```
