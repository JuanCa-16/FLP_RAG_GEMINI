####
**Definición de función en Racket**
```racket
let func = proc(?f, ?x) (f +(x, 1) >(x, 0))
in (func proc(?a, ?b) a 10)
```
Este fragmento presenta una expresión `let` que define un procedimiento `func`. Este procedimiento recibe dos argumentos, `f` (una función) y `x` (un entero), y aplica `f` sobre los resultados de `(x + 1)` y `(x > 0)`.

####
**Tabla de variables de tipo**
Para realizar la inferencia, se establecen las siguientes variables de tipo para las expresiones y subexpresiones:

| Variable | Expresión | Tipo inferido |
| :--- | :--- | :--- |
| `t0` | `let ... in ...` | `int` |
| `tfunc` | `proc(?f, ?x) ...` | `(int * bool -> int) * int -> int` |
| `tf` | `f` | `int * bool -> int` |
| `tx` | `x` | `int` |
| `t1` | `(f +(x, 1) >(x, 0))` | `int` |
| `t2` | `+(x, 1)` | `int` |
| `t3` | `>(x, 0)` | `bool` |
| `t4` | `(func proc(?a, ?b) a 10)` | `int` |
| `t5` | `proc(?a, ?b) a` | `int * bool -> int` |
| `ta` | `a` | `int` |
| `tb` | `b` | `bool` |

La variable resultado final del sistema es `t0`.

####
**Declaraciones y evaluación de tipos**
El proceso de inferencia de tipos se basa en las siguientes igualdades derivadas de las definiciones de los procedimientos y las aplicaciones:

- `tfunc = tf * tx -> t1`
- `tf = t2 * t3 -> t1`
- `tx * int -> t2` (donde `int * int -> int`, por tanto `tx = int` y `t2 = int`)
- `tx * int -> t3` (donde `int * int -> bool`, por tanto `tx = int` y `t3 = bool`)
- `tfunc = t5 * int -> t4`
- `t5 = tf`, `tx = int`, `t4 = t1`
- `t5 = ta * tb -> ta`
- `ta = t2`, `tb = t3`, `ta = t1`
- `t0 = t4`

####
**Demostración mediante el analizador de tipos**
El siguiente código muestra la verificación de tipos utilizando una estructura de datos `type-of-expression` sobre el programa definido:

```racket
(type-of-expression
  (cases program
    (scan&parse
      "let func = proc(?f, ?x) (f +(x, 1) >(x, 0))
       in let res = (func proc(?a, ?b) a 10) in res")
    (a-program (exp) exp))
  (empty-tenv))
```

El resultado de esta evaluación es una estructura que representa un tipo atómico `int`:
`#(struct:tvar-type 16 #(struct:atomic-type int))`

####
**Obtención de la forma externa del tipo de función**
Para visualizar el tipo inferido de `func` en una forma legible, se utiliza la función `type-to-external-form`:

```racket
(type-to-external-form
  (type-of-expression
    (cases program
      (scan&parse
        "let func = proc(?f, ?x) (f +(x, 1) >(x, 0))
         in let res = (func proc(?a, ?b) a 10) in func")
      (a-program (exp) exp))
    (empty-tenv)))
```

El resultado obtenido es:
`((int * bool -> int) * int -> int)`
