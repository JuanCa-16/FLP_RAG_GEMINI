####
**Concepto de estructuras condicionales**
Los condicionales permiten un flujo de ejecución condicional: si se cumple una condición, se ejecuta cierto número de instrucciones; si no se cumple, se ejecutan otras. Para implementar condicionales, se requiere el uso de `booleanos`, ya que es necesario evaluar si una condición es `true` o `false`. En este sistema, los `valores expresados` y `valores denotados` incluyen tanto números como booleanos.

####
**Modificación a la gramática para expresiones condicionales**
Para incluir los condicionales en el lenguaje, se extiende la gramática definiendo la estructura de la expresión `if`. La regla sintáctica se define como:

`<expression> ::= "if" <expresion> "then" <expresion> "else" <expresion>`

En la implementación, esto se añade como una `if-exp` a la gramática:

```racket
(expression
  ("if" expression "then" expression "else" expression)
  if-exp)
```

Un ejemplo de uso de esta sintaxis sería:
```racket
if (> x 3) then 4 else 5
```

####
**Implementación de eval-expression para condicionales**
La función `eval-expression` debe ser modificada para procesar la `if-exp`, la cual contiene una `cond-exp`, una `true-exp` y una `false-exp`. El proceso evalúa primero la condición y verifica que el resultado sea un valor booleano antes de proceder con la evaluación de las ramas.

```racket
(if-exp (cond-exp true-exp false-exp)
  (let ((cond-value (eval-expression cond-exp env)))
    (if (boolean? cond-value)
        (if cond-value
            (eval-expression true-exp env)
            (eval-expression false-exp env))
        (eopl:error "The condition must be a boolean" cond-exp))))
```

Si la expresión no corresponde a un tipo esperado, el sistema lanza un error mediante `eopl:error`.
