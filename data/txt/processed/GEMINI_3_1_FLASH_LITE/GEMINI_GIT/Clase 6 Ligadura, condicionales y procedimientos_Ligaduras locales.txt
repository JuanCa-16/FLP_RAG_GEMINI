####
**Concepto de ligaduras locales**

Las `ligaduras locales` permiten introducir nuevas asociaciones en un programa, definiendo su propio `alcance léxico`. Si bien las ligaduras del ambiente inicial poseen un `alcance global`, estas pueden ser ocultadas mediante el fenómeno conocido como `shadowing` cuando se definen ligaduras locales.

Un ejemplo básico de esta estructura es:

```racket
let
  x = 8
  y = 9
  z = +(x, 1)
in
  +(x, y, z)
```

**Estructura del `let`**

El `let` se compone de dos áreas claramente diferenciadas:
- **Área de declaración:** Contiene las asociaciones `identifier = expression`. En esta zona se evalúan las expresiones utilizando el ambiente anterior.
- **Área del cuerpo:** Comienza después de la palabra clave `in`. A partir de este punto, las nuevas ligaduras declaradas comienzan a existir y forman parte del ambiente extendido para evaluar la expresión resultante.

Al crear una ligadura local, el sistema extiende el ambiente actual incluyendo las nuevas asociaciones.

####
**Diagrama de ambientes**

Este diagrama representa la relación entre el ambiente vacío, el ambiente inicial (`env0`) y las extensiones generadas por las ligaduras locales (`env1`, `env2`).

```mermaid

    B["env0: a,b,c = 4,5,6"]
    C["env2: x,y,z = 8,9,2"]

    B --> A
    A --> C
```

####
**Modificación a la gramática**

Para soportar `ligaduras locales`, la gramática de las expresiones se extiende incluyendo la estructura `let-exp`.

```racket
(expression
  ("let"
    (arbno identifier "=" expression)
    "in" expression
  )
  let-exp
)
```

**Implementación de `eval-expression`**

La función `eval-expression` se modifica para procesar el `let-exp` evaluando las expresiones en el ambiente actual antes de extenderlo para el cuerpo del `let`.

```racket
(let-exp (lid lexpr expr)
  (let ((vexpr (map (lambda (x) (eval-expression x env)) lexpr)))
    (eval-expression expr (extend-env lid vexpr env))
  )
)
```

####
**Ejemplo complejo de ligaduras locales**

Considerando el ambiente inicial definido como:
`(extend-env '(x,y,z) '(1,2,3) (extend-env '(a,b,c) '(4,5,6) (empty-env)))`

Se analiza la siguiente expresión `let` anidada:

```racket
let
  x = let a = 3 b = 4 c = 5 in +(x, y, z)
  y = +(y, x)
  z = *(z, let p = x in +(p, z))
in
  let
    m = let f = let f = +(x, y) in f in +(f, z)
    n = +(a, b)
  in
    +(x, y, z, m, n)
```

Esta evaluación resulta en `51`.
