####
**Definición del tipo de dato `target`**
La implementación del paso por referencia utiliza el tipo `target`, que distingue entre valores directos e indirectos. Esto garantiza que al pasar una referencia a un procedimiento, siempre se apunte hacia un `direct-target`.

```racket
(define-datatype target target?
  (direct-target (val expval?))
  (indirect-target (ref ref-to-direct-target?)))
```

El `direct-target` contiene un valor expresado (`expval`), mientras que el `indirect-target` contiene una referencia que apunta a un `direct-target`.

####
**Definición de valores expresados (`expval`)**
Un `expval` representa los valores permitidos en el lenguaje, los cuales pueden ser números, booleanos o valores de procedimiento (`procVal`).

```racket
(define expval?
  (lambda (x)
    (or (number? x)
        (boolean? x)
        (procVal? x))))
```

####
**Validación de referencias a `direct-target`**
Para asegurar la integridad de las referencias, el predicado `ref-to-direct-target?` verifica que una referencia contenga efectivamente un `direct-target` en la posición indicada del vector de memoria.

```racket
(define ref-to-direct-target?
  (lambda (x)
    (and (reference? x)
         (cases reference x
           (a-ref (pos vec)
                  (cases target (vector-ref vec pos)
                    (direct-target (v) #t)
                    (indirect-target (v) #f)))))))
```

####
**Operación de desreferenciación (`deref`)**
La función `deref` se modifica para gestionar la jerarquía de los targets. Si el objetivo es un `direct-target`, retorna el valor directamente. Si es un `indirect-target`, accede a la referencia contenida para extraer el valor del `direct-target` subyacente.

```racket
(define deref
  (lambda (ref)
    (cases target (primitive-deref ref)
      (direct-target (val) val)
      (indirect-target (ref1)
        (cases target (primitive-deref ref1)
          (direct-target (val) val)
          (indirect-target (p)
            (eopl-error "Illegal reference" ref1)))))))
```

####
**Operación de asignación (`setref!`)**
El procedimiento `setref!` permite actualizar el valor de una referencia. Si el objetivo es directo, utiliza la referencia actual; si es indirecto, utiliza la referencia contenida dentro del objetivo.

```racket
(define setref!
  (lambda (ref val)
    (let ((ref1 (cases target (primitive-deref ref)
                  (direct-target (e) ref)
                  (indirect-target (r) r))))
      (primitive-setref! ref1 (direct-target val)))))
```

####
**Modificación de la expresión `let`**
Para implementar correctamente el paso por referencia, la evaluación de las expresiones dentro de un `let` debe generar objetos de tipo `direct-target`.

```racket
(let-exp (lid lexpr expr)
  (let ((vexpr (map (lambda (x)
                      (direct-target (eval-expression x env)))
                    lexpr)))
    (eval-expression expr
                     (extend-env lid (list->vector vexpr) env))))
```

####
**Ambiente inicial (`init-env`)**
El ambiente inicial se define pre-cargando las variables con `direct-target`, asegurando que todas las variables disponibles desde el inicio cumplan con la estructura requerida para la gestión de referencias.

```racket
(define init-env
  (extend-env '(x y z)
              (list->vector (list (direct-target 1)
                                  (direct-target 2)
                                  (direct-target 3)))
              (extend-env '(a b c)
                          (list->vector (list (direct-target 4)
                                              (direct-target 5)
                                              (direct-target 6)))
                          (empty-env))))
```
