####
**Sistema de tipos y expresiones de tipo**

En el contexto de un lenguaje con `sistema de tipos`, una `expresión de tipo` indica el tipo de una variable, no su representación en memoria. Por ejemplo, al escribir `int a = 3`, `int` representa la `expresión de tipo`.

La `gramática de tipos` se define de la siguiente manera:

- `<tipo-exp> ::= int` (`int-type-exp()`)
- `<tipo-exp> ::= bool` (`bool-type-exp()`)
- `<tipo-exp> ::= (<tipo-exp>*(,) -> <tipo-exp>)` (`proc-type-exp(targs result-type)`)

####
**Ejemplos de expresiones tipadas**

Las expresiones pueden ser anotadas con sus respectivos tipos:

- `proc(int a, int b) + (a, b)` ; `(int * int) -> int`
- `5` ; `int`
- `true` ; `bool`

####
**Ejemplos de tipos de procedimientos**

Los procedimientos poseen una firma que describe sus parámetros y su tipo de retorno:

- `even?(x)` ; `int -> bool`
- `+` ; `int * int -> int`
- `(lambda (x) (lambda (y) (+ x y)))` ; `int -> (int -> int)`
- `(lambda (f x) (zero? (f (+ x 1))))` ; `((int -> int) * int) -> bool`

####
**Ejercicio de inferencia de tipos**

Se pueden deducir los tipos basándose en la estructura de las expresiones:

- `-(32, 26)` ; `int`
- `proc(x) (x 26)` ; `(int -> t) -> t`
- `proc(x) if x then 1 else 2` ; `bool -> int`
- `proc(f, x) if (f x) then 1 else 2` ; `(tx -> bool) * tx -> int`
- `proc(x) proc(y) if y then x else 3` ; `int -> (bool -> int)`

Cuando un tipo no se puede determinar unívocamente, se denomina `tipo polimórfico`.

####
**Especificación del lenguaje**

El lenguaje es `fuertemente y estáticamente tipado`. Se generan errores de tipo en las siguientes condiciones:

- Se intenta aplicar algo que no es un `procVal` (evaluación de no-procedimiento).
- Se envía un número incorrecto de argumentos a un procedimiento.
- Se aplica un argumento de tipo incorrecto a una primitiva (ej. usar un booleano en una operación suma).
- Se utiliza algo diferente a un booleano en la condición de un condicional.

Nota: Las divisiones por cero no se consideran errores de tipo, sino errores semánticos.

####
**Implementación del verificador de tipos**

El verificador se basa en el procedimiento `type-of-expression`, el cual recibe una expresión y un `ambiente de tipos` para asignar un tipo.

**Comportamiento de type-of-expression:**

- `Número`: retorna `int`.
- `Booleano`: retorna `bool`.
- `Variable`: busca la ligadura en el `ambiente de tipos`.
- `Condicional (if)`:
    - La condición debe ser de tipo `bool`.
    - Las expresiones `then` y `else` deben tener el mismo tipo.
    - Retorna el tipo común de `then` y `else`.
- `Procedimiento (proc)`: retorna `(t1 * t2 * ... * tn) -> t`, donde `ti` son los tipos de los parámetros y `t` el tipo del cuerpo.
- `Aplicación`: para `(rator rand1 ... randn)`:
    - `rator` debe tener tipo `(t1 * t2 * ... * tn) -> t`.
    - Cada `randi` debe tener tipo `ti`.
    - Retorna tipo `t`.

####
**Limitaciones y estrategias de resolución**

Al evaluar una aplicación de procedimiento, es necesario contrastar el `rator` con los `rands`. El problema ocurre cuando los tipos de los parámetros del `rator` no pueden determinarse directamente, por ejemplo en: `let f = proc (x, y) ... in (f 1 2)`.

Existen dos estrategias principales para solucionar esto:

1. **Chequeo de tipos explícito**: Colocar anotaciones de tipo directamente en los procedimientos, por ejemplo: `let f = proc (int x, int y) ... in (f 1 2)`.
2. **Inferencia de tipos**: Utilizar un sistema de ecuaciones de tipos y unificación para determinar los tipos automáticamente según el contexto: `let f = proc (? x, ? y) ... in (f 1 2)`.
