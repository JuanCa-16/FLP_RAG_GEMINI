####
**Arquitectura del intérprete**

El diseño de un intérprete requiere componentes fundamentales para transformar el código fuente en resultados ejecutables. El proceso consta de:

*   `Scanner`: Transforma el código fuente (string) en una lista de `tokens` basándose en una especificación léxica.
*   `Parser`: Convierte la secuencia de `tokens` en un `AST` (Abstract Syntax Tree) siguiendo una gramática formal definida en BNF.
*   `Ambiente`: Estructura de datos que gestiona las `ligaduras` (bindings) entre identificadores y valores. Es independiente del código fuente.
*   `REPL` (Read-Eval-Print Loop): Interfaz interactiva que permite al usuario introducir código y obtener una respuesta evaluada.

####
**Especificación léxica y gramatical**

El lenguaje, basado en el estándar `EOPL`, define tanto la forma de los `tokens` como las reglas de producción para las expresiones.

```racket
#lang eopl

; Especificación léxica: define tokens y patrones regex
(define lexical
  '((comment ("%" (arbno (not #\newline))) skip)
    (whitespace (whitespace) skip)
    (number (digit (arbno digit)) number)
    (number ("-" digit (arbno digit)) number)
    (identifier (letter (arbno (or letter digit))) symbol)))

; Gramática para el parser
(define grammar
  '((program (expression) a-program)
    (expression (identifier) var-exp)
    (expression (number) lit-exp)
    (expression ("true") true-exp)
    (expression ("false") false-exp)
    (expression (primitive "(" (separated-list expression ",") ")") prim-exp)
    (primitive ("+") add-prim)
    (primitive ("-") sub-prim)
    (primitive ("*") prod-prim)
    (primitive ("/") div-prim)
    (primitive ("and") and-prim)
    (primitive ("or") or-prim)
    (primitive ("<") less-prim)
    (primitive ("<=") lesseq-prim)
    (primitive (">") more-prim)
    (primitive (">=") moreeq-prim)
    (primitive ("==") eq-prim)
    (primitive ("!=") neq-prim)))

(sllgen:make-define-datatypes lexical grammar)
```

####
**Implementación del entorno**

El `environment` permite gestionar el `scope léxico` buscando identificadores desde el entorno más reciente al más antiguo.

```racket
(define-datatype environment environment?
  (empty-env)
  (extend-env
    (lid (list-of symbol?))
    (lval (list-of value?))
    (old-env environment?)))

(define apply-env
  (lambda (env var)
    (cases environment env
      (empty-env () (eopl:error "Variable not found" var))
      (extend-env (lid lval old-env)
        (letrec ((search-var
                  (lambda (lid lval)
                    (cond [(null? lid) (apply-env old-env var)]
                          [(equal? (car lid) var) (car lval)]
                          [else (search-var (cdr lid) (cdr lval))]))))
          (search-var lid lval))))))
```

####
**Evaluación de expresiones**

La función `eval-expression` recorre el `AST` y realiza la computación según el tipo de nodo. Para las `prim-exp`, se evalúan los argumentos y se aplica la primitiva correspondiente.

```racket
(define eval-expression
  (lambda (exp env)
    (cases expression exp
      (var-exp (id) (apply-env env id))
      (lit-exp (datum) datum)
      (true-exp () #t)
      (false-exp () #f)
      (prim-exp (prim lrands)
        (let ((values (map (lambda (x) (eval-expression x env)) lrands)))
          (apply-primitive prim values))))))
```

####
**Aplicación de primitivas**

Las primitivas utilizan una función de operación acumulativa para procesar los argumentos. Nota: las operaciones `sub` y `div` calculan el primer elemento menos/dividido por la suma/producto del resto.

```racket
(define operation
  (lambda (f lst acc)
    (cond [(null? lst) acc]
          [else (operation f (cdr lst) (f (car lst) acc))])))

(define apply-primitive
  (lambda (prim values)
    (cases primitive prim
      (add-prim () (operation + values 0))
      (sub-prim () (- (car values) (operation + (cdr values) 0)))
      (prod-prim () (operation * values 1))
      (div-prim () (/ (car values) (operation * (cdr values) 1)))
      (and-prim () (operation (lambda (a b) (and a b)) values #t))
      (or-prim () (operation (lambda (a b) (or a b)) values #f))
      (less-prim () (< (car values) (cadr values)))
      (lesseq-prim () (<= (car values) (cadr values)))
      (more-prim () (> (car values) (cadr values)))
      (moreeq-prim () (>= (car values) (cadr values)))
      (eq-prim () (= (car values) (cadr values)))
      (neq-prim () (not (= (car values) (cadr values)))))))
```

####
**Ejecución del intérprete**

Para interactuar con el lenguaje, se inicializa el entorno y se ejecuta el bucle `REPL`.

```racket
(define init-env
  (extend-env '(x y z) '(1 2 3)
    (extend-env '(a b c) '(4 5 6) (empty-env))))

(define interpreter
  (sllgen:make-rep-loop "-->" eval-program
    (sllgen:make-stream-parser lexical grammar)))

; (interpreter) ; Descomentar para iniciar el REPL
```

Ejemplos de evaluación directa:
* `(eval-program (parser "x"))` retorna `1`.
* `(eval-program (parser "+(x,y)"))` retorna `3`.
