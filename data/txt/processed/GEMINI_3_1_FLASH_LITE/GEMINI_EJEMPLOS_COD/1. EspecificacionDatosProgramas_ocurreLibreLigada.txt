####
Definición de la función `occurs-free?` para determinar si una variable ocurre libremente en una expresión lambda:

```racket
(define occurs-free?
  (lambda (var exp)
    (cond
      ((symbol? exp) (eqv? var exp))
      ((eqv? (car exp) 'lambda) 
       (and (not (eqv? (caadr exp) var))
            (occurs-free? var (caddr exp))))
      (else (or (occurs-free? var (car exp))
                (occurs-free? var (cadr exp)))))))
```

**Conceptos clave**
`cálculo lambda`, `variables libres`, `recursión`, `predicados`, `estructuras de datos`

**Aplicación**
Esta función implementa la lógica formal para identificar si una variable `var` no está ligada por un `lambda` dentro de una expresión `exp`. Es fundamental en la construcción de intérpretes y sistemas de manipulación de expresiones simbólicas para verificar el alcance de las variables.

####
Evaluación de `occurs-free?` en una expresión lambda compleja:

```racket
(define exp1 '(lambda (x y) ((lambda (a) (x (a y)))x)))
(occurs-free? 'x exp1)
```

**Conceptos clave**
`expresiones simbólicas`, `listas`, `evaluación`, `scope`

**Aplicación**
Este ejemplo ilustra cómo el algoritmo procesa expresiones anidadas con múltiples parámetros. Al ejecutar `(occurs-free? 'x exp1)`, el sistema determina si la variable `x` tiene una ocurrencia libre en la estructura proporcionada, retornando un valor booleano tras evaluar la jerarquía de las funciones `lambda`.

####
Verificación de ocurrencia libre en expresiones con variables ligadas y no ligadas:

```racket
(define exp2 '((lambda (x) x) y))
(occurs-free? 'y exp2)
```

**Conceptos clave**
`variables ligadas`, `alcance`, `procedimientos`, `S-expressions`

**Aplicación**
Se utiliza para verificar la presencia de variables libres en una aplicación de función. En este caso, la variable `y` aparece fuera del alcance del `lambda` interno, por lo que la función identifica correctamente su estado como variable libre.

####
Detección de ocurrencia libre en expresiones lambda anidadas:

```racket
(define exp4 '(lambda (y) ((lambda (x) x) y)))
(occurs-free? 'y exp4)
```

**Conceptos clave**
`anidamiento`, `abstracción`, `variables libres`, `lexical scoping`

**Aplicación**
Demuestra el comportamiento de la función ante estructuras donde una variable es capturada por un `lambda` externo. Permite discernir si, a pesar de las abstracciones internas, la variable `y` mantiene una presencia libre respecto al bloque principal.

####
Validación de ausencia de variables libres en una expresión:

```racket
(define exp5 '(lambda (x y z) (a b)))
(occurs-free? 'x exp5)
```

**Conceptos clave**
`variables ligadas`, `ocultamiento`, `abstracción`, `sintaxis de Racket`

**Aplicación**
Ilustra el caso donde una variable `x` es declarada como parámetro en un `lambda`, lo que la convierte en una variable ligada. Al evaluar `(occurs-free? 'x exp5)`, la función retorna `#f`, confirmando que `x` no ocurre libremente en el cuerpo de la expresión.
