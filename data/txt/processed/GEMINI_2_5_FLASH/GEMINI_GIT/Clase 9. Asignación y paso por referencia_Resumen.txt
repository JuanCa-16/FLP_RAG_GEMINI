Exploramos lo que son las tuplas, la parametrización de tipos para que las funciones acepten cualquier tipo de dato (siempre y cuando se pueda operar), y las funciones de alto orden. Vimos Map para la transformación de listas, Filter para el filtrado de listas, y Reduce para transformar listas a un valor. ReduceLeft (izquierda) y ReduceRight (derecha) toman un elemento de la lista como acumulador. FoldLeft (Izquierda) y FoldRight (derecha) permiten establecer el acumulador, retornando un reduce con este ya establecido.

###
Las Tuplas agrupan valores heterogéneos. Son inmutables, no recursivas y permiten pattern matching. Un ejemplo sería `(1, "hola", 3.14)` con el tipo `(Int, String, Double)`.

###
Map se utiliza para transformar elementos de una lista. Cambia el tipo de los elementos pero mantiene la estructura de la lista.
Ejemplo:
```scala
List(1,2,3).map(_ * 2)
```
Resultado: `List(2,4,6)`

###
Filter se usa para filtrar elementos de una lista basándose en un predicado booleano. Mantiene el tipo de los elementos.
Ejemplo:
```scala
List(1,2,3,4).filter(_ % 2 == 0)
```
Resultado: `List(2,4)`

###
Reduce permite reducir una lista a un único valor. Los elementos y el resultado son del mismo tipo y no funciona con listas vacías.
Ejemplo:
```scala
List(1,2,3).reduce(_ + _)
```
Resultado: `6`

###
Fold se utiliza para reducir una lista a un valor, permitiendo especificar un acumulador inicial. Puede cambiar el tipo de retorno y funciona con listas vacías.
Ejemplo:
```scala
List(1,2,3).fold(0)(_ + _)
```
Resultado: `6`

###
Las diferencias clave entre Reduce y Fold son:
*   **Tipo de retorno:** Reduce devuelve el mismo tipo que los elementos, mientras que Fold puede devolver un tipo diferente.
*   **Listas vacías:** Reduce lanza una excepción con listas vacías, Fold retorna el acumulador.
*   **Acumulador inicial:** Reduce usa el primer/último elemento como acumulador, Fold requiere que se especifique.
*   **Asociatividad:** En ambos, Left/Right es importante.

###
La asociatividad en operaciones puede ser crucial:
*   **Suma:** Es asociativa.
    *   Left: `(((1+2)+3)+4)=10`
    *   Right: `(1+(2+(3+4)))=10`
    *   El resultado es el mismo.
*   **Resta:** No es asociativa.
    *   Left: `(((1-2)-3)-4)=-8`
    *   Right: `(1-(2-(3-4)))=-2`
    *   El resultado es diferente.

###
El Pattern Matching con Tuplas puede simplificar código anidado, haciéndolo más legible al evitar `nested matches`. Por ejemplo, en el cálculo de un producto punto, podemos transformar código anidado complejo como:
```scala
case x :: xs => v match {
  case y :: ys => prodI(xs)(ys)(x * y + acc)
}
```
En un código más limpio usando tuplas:
```scala
case (x :: xs, y :: ys) => prodI(xs)(ys)(x * y + acc)
```

###
Las Funciones Genéricas permiten la reutilización con diferentes tipos de datos. Se definen con un parámetro de tipo `[T]`:
```scala
def funcion[T](l: List[T]): ...
```
Esta función puede ser usada con listas de enteros:
```scala
funcion[Int](List(1, 2, 3))
```
Y también con listas de cadenas:
```scala
funcion[String](List("a", "b"))
```

###
Las aplicaciones prácticas de estos conceptos incluyen:
*   **Producto punto:** Para combinar elementos de listas paralelas.
*   **Concatenación tipada:** Para crear tuplas de diferentes tipos.
*   **Filtrado genérico:** Para aplicar predicados a cualquier tipo de dato.
*   **Reducción flexible:** Para sumas, multiplicaciones o concatenaciones.
*   **Transformación de tipos:** Por ejemplo, de una lista de strings a la suma de sus longitudes.

###
Las ventajas de este enfoque son:
*   **Type safety:** Scala detecta errores en tiempo de compilación.
*   **Reutilización:** Las funciones genéricas trabajan con múltiples tipos.
*   **Expresividad:** El pattern matching simplifica la lógica compleja.
*   **Eficiencia:** La recursión de cola (tail recursion) optimiza el uso de memoria.
*   **Flexibilidad:** Fold supera las limitaciones de reduce.
