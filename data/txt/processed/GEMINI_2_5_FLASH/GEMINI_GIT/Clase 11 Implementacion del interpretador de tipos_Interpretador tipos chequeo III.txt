Evaluación de procedimientos:
En la evaluación se deben validar:
El rator debe ser un procVal.
El número de argumentos debe ser igual al número que espera el procVal.
Los tipos de los argumentos deben ser iguales a lo que espera el procedimiento.

###
Al evaluar el procedimiento, el resultado es el tipo de retorno del proc. Por ejemplo, en la expresión `let f = proc (int x) x in (f 3)`:
`f` es `int -> int`
`(f 3)` es `int`

###
Para evaluar esto se modifican los ambientes iniciales. Dentro de `type-of-expression`, la construcción `app-exp` se encarga de calcular los tipos del `rator` (que debe ser un procedimiento) y los de los argumentos (una lista de tipos), para luego invocar `type-of-application`.

```racket
(define type-of-expression
  (lambda (exp tenv)
    ...
    (app-exp (rator rands)
      (let ((trator (type-of-expression rator tenv))
            (trands (map (lambda (x) (type-of-expression x tenv)) rands)))
        (type-of-application trator trands rator rands exp)))
    ...))
```

###
La función `type-of-application` es la encargada de realizar las validaciones de tipo para una aplicación de procedimiento.

```racket
(define type-of-application
  (lambda (trator trands rator rands exp)
    (cases type trator
      (proc-type (args-types result-type)
        (if (= (length args-types) (length trands))
            (begin
              (for-each check-equal-type! trands args-types rands)
              result-type)
            (eopl:error 'type-of-application
                        "Length of the args mismatch ~s ~s"
                        args-types trands)))
      (else
        (eopl:error 'type-of-expression
                    "Rator isn't a proc-type ~s is a type ~s"
                    rator
                    (type-to-external-form trator))))))
```

###
En `type-of-application` se realizan las siguientes verificaciones:
Verifica que el tamaño de los argumentos enviados sea igual al número de argumentos esperados.
Verifica que cada tipo enviado sea igual al tipo esperado para el argumento correspondiente.
Si todas las validaciones son correctas, retorna el tipo resultado del procedimiento.

###
Estos son los ambientes iniciales `env0`, `init-env` y `init-tenv` definidos para la evaluación y chequeo de tipos:

```racket
(define env0
  (extend-env '(a b c)
              (list->vector '(4 5 6))
              (empty-env)))

(define init-env
  (extend-env '(x y z f)
              (list->vector
               (list 1 2 3
                     (closure '(x y)
                              (prim-exp (add-prim)
                                        (list (var-exp 'x) (var-exp 'y)))
                              env0)))
              env0))

(define init-tenv
  (extend-env '(x y z f)
              (list->vector
               (list int-type
                     int-type
                     int-type
                     (proc-type (list int-type int-type int-type)
                                int-type)))
              (extend-env '(a b c)
                          (list->vector
                           (list int-type int-type int-type))
                          (empty-env))))
```

###
Ejemplos de evaluación de tipos utilizando `type-of-expression` con el ambiente de tipos inicial `init-tenv`:

```racket
(type-of-expression
  (cases program (parser "f") (a-program (exp) exp))
  init-tenv)
```
