Exploramos lo que son las tuplas, la parametrización de tipos para que las funciones acepten cualquier tipo de dato (siempre y cuando se pueda operar), y vimos las funciones de alto orden. Estas incluyen Map para la transformación de listas, Filter para el filtrado, y Reduce para transformar listas a un único valor. También diferenciamos entre ReduceLeft (izquierda) y ReduceRight (derecha), que toman un elemento de la lista como acumulador, y FoldLeft (izquierda) y FoldRight (derecha), que permiten establecer un acumulador inicial y retornan un reduce con este acumulador.

###
Una tupla permite agrupar valores heterogéneos. Son inmutables y no recursivas. Un ejemplo de tupla es `(1, "hola", 3.14)`, que tiene el tipo `(Int, String, Double)`.

El pattern matching con tuplas simplifica el código anidado, evitando `nested matches` y haciéndolo más legible. Por ejemplo:

```scala
(u,v) match { case (x::xs, y::ys) => ... }
```

###
Map es una operación para transformar los elementos de una lista. Cambia el tipo de los elementos, pero mantiene la estructura de la lista.

Por ejemplo, para duplicar cada número en una lista:

```scala
List(1,2,3).map(_ * 2)
// Resultado: List(2,4,6)
```

###
Filter es una operación para filtrar elementos de una lista. Aplica un predicado booleano y mantiene el tipo de los elementos restantes.

Por ejemplo, para obtener solo los números pares de una lista:

```scala
List(1,2,3,4).filter(_ % 2 == 0)
// Resultado: List(2,4)
```

###
Reduce es una operación para reducir una lista a un único valor. Los elementos y el valor de retorno deben ser del mismo tipo. No funciona con listas vacías, lanzando una excepción.

Por ejemplo, para sumar todos los elementos de una lista:

```scala
List(1,2,3).reduce(_ + _)
// Resultado: 6
```

###
Fold es una operación para reducir una lista a un valor utilizando un acumulador inicial. Permite que el tipo de retorno sea diferente al tipo de los elementos de la lista y funciona correctamente con listas vacías.

Por ejemplo, para sumar los elementos de una lista partiendo de un acumulador inicial de 0:

```scala
List(1,2,3).fold(0)(_ + _)
// Resultado: 6
```

###
Las diferencias clave entre Reduce y Fold se pueden resumir en la siguiente tabla:

| Aspecto           | Reduce                 | Fold                         |
| :---------------- | :--------------------- | :--------------------------- |
| Tipo de retorno   | Mismo que elementos    | Puede ser diferente          |
| Listas vacías     | Lanza excepción        | Retorna acumulador           |
| Acumulador inicial | Primer/último elemento | Se especifica                |
| Asociatividad     | Left/Right importante  | Left/Right importante        |

###
La asociatividad en operaciones es importante para el orden de evaluación.
La suma es asociativa, por lo que el orden no altera el resultado:
`(((1+2)+3)+4) = 10` (Left) y `(1+(2+(3+4))) = 10` (Right) dan el mismo resultado.

La resta no es asociativa, por lo que el orden sí altera el resultado:
`(((1-2)-3)-4) = -8` (Left) y `(1-(2-(3-4))) = -2` (Right) dan resultados diferentes.

###
Una implementación práctica es el producto punto con tuplas, que puede simplificar código anidado complejo.

Antes, con código anidado:

```scala
case x :: xs =>
  v match {
    case y :: ys =>
      prodI(xs)(ys)(x * y + acc)
  }
```

Después, con código limpio usando tuplas en el pattern matching:

```scala
case (x :: xs, y :: ys) =>
  prodI(xs)(ys)(x * y + acc)
```

###
Las funciones genéricas permiten la reutilización con diferentes tipos de datos.

```scala
// Permite reutilización con diferentes tipos
def funcion[T](l: List[T]): ...

funcion[Int](List(1, 2, 3))
funcion[String](List("a", "b"))
```

###
Las aplicaciones prácticas de estos conceptos incluyen:
*   Producto punto: Combinar elementos de listas paralelas.
*   Concatenación tipada: Crear tuplas de diferentes tipos.
*   Filtrado genérico: Aplicar predicados a cualquier tipo de dato.
*   Reducción flexible: Realizar sumas, multiplicaciones o concatenaciones.
*   Transformación de tipos: Por ejemplo, convertir una lista de strings a la suma de sus longitudes.

###
Este enfoque ofrece varias ventajas:
*   **Type safety**: Scala detecta errores en tiempo de compilación.
*   **Reutilización**: Las funciones genéricas trabajan con múltiples tipos.
*   **Expresividad**: El pattern matching simplifica la lógica compleja.
*   **Eficiencia**: La recursión de cola (tail recursion) optimiza el uso de memoria.
*   **Flexibilidad**: Fold supera las limitaciones de reduce.
