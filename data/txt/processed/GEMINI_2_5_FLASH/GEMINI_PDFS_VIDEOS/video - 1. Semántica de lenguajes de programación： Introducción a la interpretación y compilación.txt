La semántica de los lenguajes de programación aborda los conceptos de interpretación y compilación, procesos fundamentales para transformar el código fuente en un formato ejecutable. El primer paso en ambos procesos consiste en convertir el código en una estructura de datos intermedia conocida como árbol de sintaxis abstracta (AST), que representa la estructura gramatical del programa.

####

El proceso de transformación del código fuente comienza en una etapa denominada *frontend*, la cual se encarga de dos tareas principales. Primero, un *escáner* o analizador léxico revisa el texto para identificar componentes significantes (tokens), como palabras clave, identificadores y operadores, mientras ignora elementos sin valor semántico, como espacios en blanco y comentarios. A continuación, un *parser* o analizador sintáctico utiliza la gramática del lenguaje para construir un árbol de sintaxis abstracta (AST) a partir de los tokens generados por el escáner.

####

El árbol de sintaxis abstracta (AST) cumple dos funciones esenciales. En primer lugar, permite validar que el código fuente está escrito correctamente de acuerdo con las reglas gramaticales del lenguaje de programación, detectando errores de sintaxis. En segundo lugar, una vez validada la estructura, el AST sirve como la representación intermedia del programa que se utilizará para su posterior ejecución o traducción a otro lenguaje.

####

Un interpretador es un programa que ejecuta las instrucciones definidas en un árbol de sintaxis abstracta directamente. El interpretador está escrito en un lenguaje de implementación (también llamado lenguaje de definición), que es el lenguaje utilizado para construir el propio interpretador. Por ejemplo, el interpretador principal de Python está escrito en C++.

####

Ejemplos de lenguajes que utilizan un modelo de interpretación incluyen Python, Java y JavaScript. En Python, el código se transforma en un AST que es procesado por una máquina virtual escrita en C++. En Java, el código se compila a un formato binario intermedio llamado *bytecode* (archivos `.class`), que luego es interpretado por la Máquina Virtual de Java (JVM). En JavaScript, el motor del navegador web (como V8 en Chrome) construye el AST y ejecuta el código para producir una respuesta.

####

Los programas interpretados suelen ser más lentos que los compilados debido al paso de intermediación que supone la traducción y ejecución en tiempo real por parte del interpretador o máquina virtual. Sin embargo, esta desventaja de rendimiento se compensa con una mayor facilidad de programación y una sintaxis a menudo más sencilla. Debido a esta sobrecarga de procesamiento, los lenguajes puramente interpretados no son ideales para tareas de computación intensiva, aunque existen excepciones como Python, que utiliza librerías optimizadas escritas en lenguajes compilados para tareas de alto rendimiento como el *machine learning*.

####

En los lenguajes compilados, como C++ o Assembly, el proceso transforma directamente el código fuente en código máquina ejecutable por el sistema operativo. Durante la compilación, se genera el árbol de sintaxis abstracta, el cual es analizado y traducido a un lenguaje de bajo nivel, como el lenguaje ensamblador (*assembly*). Por ejemplo, al compilar un archivo C++ con `g++`, se genera un archivo objeto (`.o`) que contiene una representación binaria del AST y otras informaciones, para finalmente producir un archivo ejecutable que el sistema operativo puede correr directamente.

####

El compilador está compuesto por un analizador y un traductor. El analizador procesa el árbol de sintaxis abstracta para deducir información útil sobre el programa y realizar optimizaciones. Posteriormente, el traductor utiliza esta información para convertir el código a un lenguaje destino. Este lenguaje destino es típicamente código máquina o lenguaje ensamblador, que puede ser ejecutado directamente por el hardware.

####

Algunos lenguajes utilizan un modelo híbrido, compilando el código fuente a un lenguaje intermedio de propósito especial, conocido como *bytecode*. Este *bytecode* es más fácil de interpretar que el código fuente original. Un ejemplo clásico es Java, donde el código se compila a archivos `.class` (bytecode). Este código intermedio es independiente de la plataforma y puede ser ejecutado en cualquier sistema que tenga instalada una Máquina Virtual de Java (JVM). La JVM actúa como el interpretador del *bytecode*. Otros lenguajes que utilizan máquinas virtuales son Scala, Haskell y Erlang.
