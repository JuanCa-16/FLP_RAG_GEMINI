La sintaxis abstracta se representa mediante tipos de datos, como los definidos con `define-datatype`, que estructuran el código en forma de Árboles de Sintaxis Abstracta (AST). Dada una gramática, como la de las expresiones del cálculo lambda, cada expresión puede ser representada unívocamente utilizando una estructura de datos de este tipo.

####

La representación textual del código que un programador escribe se conoce como sintaxis concreta o representación externa. Sin embargo, para que este código pueda ser procesado por un compilador o un intérprete, debe ser transformado en una representación interna, conocida como sintaxis abstracta. Esta estructura interna permite al sistema verificar la validez del código y facilitar su ejecución.

####

Para crear una sintaxis abstracta a partir de una gramática de sintaxis concreta, es necesario nombrar cada regla de producción. Por ejemplo, en la gramática del cálculo lambda, las producciones se pueden nombrar de la siguiente manera:
- `var-expression`: para una variable o identificador.
- `lambda-expression`: para una abstracción, compuesta por un identificador (el parámetro) y una expresión (el cuerpo).
- `app-expression`: para una aplicación, compuesta por dos expresiones, el operador (radar) y el operando (ram).

####

Consideremos la expresión de cálculo lambda `λx. f(f x)`. Su representación como Árbol de Sintaxis Abstracta (AST) se construye de la siguiente manera:
1.  La expresión principal es una `lambda-expression`, con el identificador `x` como parámetro y la expresión `f(f x)` como cuerpo.
2.  El cuerpo, `f(f x)`, es una `app-expression` (aplicación).
    -   El operador (radar) de esta aplicación es `f`, representado como una `var-expression`.
    -   El operando (ram) es la subexpresión `f x`, que a su vez es otra `app-expression`.
3.  Dentro de la subexpresión `f x`:
    -   El operador (radar) es `f`, representado como una `var-expression`.
    -   El operando (ram) es `x`, también representado como una `var-expression`.

####

La capacidad de construir un Árbol de Sintaxis Abstracta (AST) a partir de un fragmento de código es el mecanismo fundamental que utilizan los lenguajes de programación para verificar si el código está sintácticamente correcto. Si el árbol puede generarse sin errores, significa que la expresión se adhiere a las reglas de la gramática del lenguaje y puede proceder a las siguientes fases de compilación o interpretación. Muchas herramientas de análisis y pruebas de validez de código operan directamente sobre el AST, ya que esta es una representación estructurada y validada del programa, a diferencia del código fuente nativo.
