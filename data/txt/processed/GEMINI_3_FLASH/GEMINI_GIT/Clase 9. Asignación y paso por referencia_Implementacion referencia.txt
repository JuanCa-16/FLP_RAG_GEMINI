####
**Implementación de referencias y tipos de target**

En la implementación de paso por referencia, se definen los `targets`. Un `direct-target` representa un valor expresado, mientras que un `indirect-target` apunta a un `direct-target`. Esta estructura garantiza que, al pasar una referencia a un procedimiento, siempre se cuente con un vínculo hacia un `direct-target`.

```racket
;; Paso por referencia
(define-datatype target target?
  (direct-target
   (val expval?))
  (indirect-target
   (ref ref-to-direct-target?)))
```

####
**Definición de valores expresados y validación de referencias**

Un `expval` (valor expresado) se define como un número, un booleano o un `procval`. Por otro lado, la validación `ref-to-direct-target?` asegura que una referencia apunte efectivamente a un contenedor de tipo `direct-target`.

```racket
; Expval is a number, boolean or procval (expressed value)
(define expval?
  (lambda (x)
    (or (number? x)
        (boolean? x)
        (procVal? x))))

; Ref a target direct must be a reference and this reference have to content a direct target
(define ref-to-direct-target?
  (lambda (x)
    (and (reference? x)
         (cases reference x
           (a-ref (pos vec)
                  (cases target (vector-ref vec pos)
                    (direct-target (v) #t)
                    (indirect-target (v) #f)))))))
```

####
**Operación de desreferenciación con targets**

El procedimiento `deref` se modifica para manejar la jerarquía de `targets`. Un `direct-target` contiene directamente el valor, mientras que un `indirect-target` contiene una referencia a un `direct-target` que es la que finalmente posee el valor.

```racket
(define deref
  (lambda (ref)
    (cases target (primitive-deref ref)
      (direct-target (val)
                     val)
      (indirect-target (ref1)
                       (cases target (primitive-deref ref1)
                         (direct-target (val)
                                        val)
                         (indirect-target (p)
                                          (eopl-error "Illegal reference" ref1)))))))
```

####
**Modificación de setref! para manejo de referencias**

El procedimiento `setref!` permite cambiar el valor contenido en una referencia. Si la referencia apunta a un `direct-target`, se utiliza esa misma referencia; si apunta a un `indirect-target`, se utiliza la referencia interna que este contiene para realizar la actualización mediante `primitive-setref!`.

```racket
; Setref es un procedimiento para cambiar el valor de una referencia
(define setref!
  (lambda (ref val)
    (let ((ref1 (cases target (primitive-deref ref)
                  (direct-target (e)
                                 ref)
                  (indirect-target (r)
                                   r))))
      (primitive-setref! ref1 (direct-target val)))))
```

####
**Implementación de let-exp con direct-target**

Se modifica la evaluación de `let-exp` para que, al extender el ambiente, cada expresión evaluada sea encapsulada en un `direct-target`. Esto asegura que las variables introducidas en el `let` sigan el esquema de direccionamiento definido.

```racket
(let-exp (lid lexpr expr)
         (let ((vexpr (map (lambda (x)
                             (direct-target (eval-expression x env)))
                           lexpr)))
           (eval-expression expr (extend-env lid (list->vector vexpr) env))))
```

####
**Configuración del ambiente inicial con targets**

El ambiente inicial predefinido, que contiene las variables `x`, `y`, `z` y `a`, `b`, `c`, se inicializa utilizando `direct-target` para cada uno de sus valores numéricos correspondientes.

```racket
; Entorno inicial predefinido (variables x,y,z y a,b,c con valores numéricos)
(define init-env
  (extend-env
   '(x y z)
   (list->vector
    (list (direct-target 1)
          (direct-target 2)
          (direct-target 3)))
   (extend-env
    '(a b c)
    (list->vector
     (list (direct-target 4)
           (direct-target 5)
           (direct-target 6)))
    (empty-env))))
```
