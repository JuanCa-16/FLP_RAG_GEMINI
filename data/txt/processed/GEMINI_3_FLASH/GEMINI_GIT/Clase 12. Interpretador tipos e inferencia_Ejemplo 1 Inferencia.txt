####
**Expresión para inferencia de tipos**
```racket
let
  func = proc(? f, ? x) (f +(x, 1) >(x, 0))
in (func proc(? a, ? b) a 10)
```
Esta expresión define un procedimiento `func` con parámetros `f` y `x` de tipos inicialmente desconocidos. El cuerpo del procedimiento realiza una aplicación de `f` sobre los resultados de una suma y una comparación. Finalmente, se invoca `func` pasando un procedimiento y el valor constante `10`.

####
**Variables de tipo y mapeo de expresiones**
Para el proceso de inferencia de tipos, se establecen las siguientes variables y sus correspondientes expresiones:

- `t0`: `let .. in ...` -> `int`
- `tfunc`: `proc(? f, ? x) ...` -> `(int * bool -> int) * int -> int`
- `tf`: `f` -> `int * bool -> int`
- `tx`: `x` -> `int`
- `t1`: `(f +(x, 1) >(x, 0))` -> `int`
- `t2`: `+(x, 1)` -> `int`
- `t3`: `>(x, 0)` -> `bool`
- `t4`: `(func proc (? a, ? b) a 10)` -> `int`
- `t5`: `proc (? a, ? b) a` -> `int * bool -> int`
- `ta`: `a` -> `int`
- `tb`: `b` -> `bool`

La variable que representa el tipo del resultado final de la expresión es `t0`.

####
**Ecuaciones de unificación y análisis de tipos**
A partir de las declaraciones y evaluaciones en el código, se deducen las siguientes relaciones para la inferencia:

**Declaraciones de proc**
- `tfunc = tf * tx -> t1`

**Evaluaciones**
- `tf = t2 * t3 -> t1`
- `tx * int -> t2` con `int * int -> int` implica que `tx = int` y `t2 = int`
- `tx * int -> t3` con `int * int -> bool` implica que `tx = int` y `t3 = bool`
- `tfunc = t5 * int -> t4`
- `t5 = tf`
- `tx = int`
- `t4 = t1`
- `t5 = ta * tb -> ta`
- `ta = t2`
- `tb = t3`
- `ta = t1`

**Let**
- `t0 = t4`

####
**Demostración de inferencia del tipo del resultado**
```racket
(type-of-expression
  (cases program
    (scan&parse
      " let
          func = proc(? f, ? x) (f +(x,1) >(x,0))
        in let
          res = (func proc (? a, ? b) a 10) in res ")
    (a-program (exp) exp))
  (empty-tenv))
```
El valor retornado por el sistema de tipos para esta expresión es:
`#(struct:tvar-type 16 #(#(struct:atomic-type int)))`

####
**Inferencia del tipo de la función func**
```racket
(type-to-external-form
  (type-of-expression
    (cases program
      (scan&parse
        " let
            func = proc(? f, ? x) (f +(x,1) >(x,0))
          in let
            res = (func proc (? a, ? b) a 10) in func ")
      (a-program (exp) exp))
    (empty-tenv)))
```
Al solicitar el formato externo del tipo de `func`, se obtiene la firma:
`((int * bool -> int) * int -> int)`
