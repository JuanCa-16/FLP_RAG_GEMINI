####
**Conceptos generales de los condicionales**

Los condicionales permiten un flujo de ejecución condicional: si se cumple una condición se ejecuta cierto número de instrucciones y, si no se cumple, se ejecutan otras. Para los condicionales se requieren `booleanos` para evaluar si una condición es verdadera o falsa.

- `Valores expresados`: Números y booleanos.
- `Valores denotados`: Números y booleanos.

####
**Especificación de la gramática para expresiones condicionales**

La producción gramatical para la expresión condicional se define de la siguiente manera:

`<expression> ::= "if" <expresion> "then" <expresion> "else" <expresion>`

Para permitir su declaración en el intérprete, se agrega la variante `if-exp` a la gramática:

```racket
(expression
  ("if" expression "then" expression "else" expression)
  if-exp)
```

####
**Ejemplo de uso de la expresión condicional**

Un ejemplo de la sintaxis utilizada para declarar una expresión condicional es el siguiente:

```racket
if > (x, 3) then 4 else 5
```

Este código evalúa si el valor de `x` es mayor a 3 para determinar si el resultado es 4 o 5.

####
**Modificación de eval-expression para el soporte de condicionales**

Para procesar las expresiones de tipo `if-exp`, se modifica el procedimiento `eval-expression` integrando la lógica de evaluación de la condición y el manejo de errores:

```racket
(if-exp (cond-exp true-exp false-exp)
  (let ((cond-value (eval-expression cond-exp env)))
    (if (boolean? cond-value)
        (if cond-value
            (eval-expression true-exp env)
            (eval-expression false-exp env))
        (eopl:error "The condition must be a boolean " cond-exp))))
```

En esta implementación, se evalúa primero `cond-exp` en el ambiente `env`. Se verifica si `cond-value` es un `booleano`; de ser así, se evalúa `true-exp` o `false-exp` según el resultado. En caso contrario, se reporta un error indicando que la condición debe ser un valor booleano.
