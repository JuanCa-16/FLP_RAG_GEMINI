####
**Implementación de objetos simples**

Los objetos simples son una implementación basada en listas de partes; cada parte representa una clase en una cadena de herencia. Por ejemplo:

```racket
class c1 extends object
  field x
  field y
  method initialize () 1

class c2 extends c1
  field z
  field w
  method initialize () 1

let o1 = new c1()
    o2 = new c2()
in ??
```

Cuando se instancia `o1`, se representa como:
```racket
(#(struct:a-part c1 #(0 0)))
```
Una parte contiene el nombre de la clase y los valores de los campos. Al observar la instancia `o2` de la clase `c2`:
```racket
(#(struct:a-part c2 #(0 0))
 #(struct:a-part c1 #(0 0)))
```
El primer elemento es la clase `c2` y el segundo la clase `c1`. Se asume `object` como la lista vacía.

####
**Jerarquía de herencia y representación de partes**

Considerando una cadena de herencia más extensa:

```racket
class c1 extends object
  field x
  field y
  method initialize () 1

class c2 extends c1
  field z
  field w
  method initialize () 1

class c3 extends c2
  field a
  field b
  method initialize () 1

let o3 = new c3()
in o3
```

En este caso, la representación interna es:
```racket
(#(struct:a-part c3 #(0 0))
 #(struct:a-part c2 #(0 0))
 #(struct:a-part c1 #(0 0)))
```

Esto construye la siguiente estructura lógica:

```mermaid
flowchart LR
    A["C3
    a = 0, b = 0"]
    B["C2
    z = 0, w = 0"]
    C["C1
    x = 0, y = 0"]
    D["object
    '()"]
    A --> B
    B --> C
    C --> D
```

El primer elemento siempre corresponde a la clase de menor jerarquía y la lista vacía representa a `object`. Esta implementación es similar a la utilizada por `C++` para representar objetos.

####
**Ejemplo de clases p1, p2 y p3 con métodos y super**

A continuación se definen clases con campos, inicialización y llamadas a métodos:

```racket
class p1 extends object
  field a
  field b
  method initialize (x, y)
    begin
      set a = x;
      set b = y
    end
  method sum (x)
    begin
      set a = +(a, 1);
      +(a, b, x, send self m1())
    end
  method m1 () 1

class p2 extends p1
  field c
  field d
  field e
  method initialize (x, y, z)
    begin
      super initialize (x, y);
      set c = +(x, 1);
      set d = +(y, 2);
      set e = z
    end
  method sum (s)
    +(c, d, e, send self m1())

class p3 extends p2
  field f
  field g
  method initialize (u)
    begin
      set f = u;
      set g = +(u, 1);
      super initialize (u, +(u, 1), +(u, 2))
    end
  method sum (a)
    +(a, super sum(+(a, 2)))
  method m1 () 10

let o1 = new p1(2, 3)
    o2 = new p2(3, 4, 5)
    o3 = new p3(6)
in list(send o1 sum(2), send o2 sum(3), send o3 sum(4))
```

El resultado de estas operaciones es `(9 16 39)`.

####
**Ambientes de los objetos o1, o2 y o3**

A continuación se muestra la representación de los objetos instanciados:

Para `o1 = new p1(2, 3)`:
```mermaid
flowchart LR
    A["c1
    a = 2, b = 3"]
    B["object"]
    A --> B
```
`(#(struct:a-part p1 #(2 3)))`

Para `o2 = new p2(3, 4, 5)`:
```mermaid
flowchart LR
    A["c2
    c = 4, d = 6, e = 5"]
    B["c1
    a = 3, b = 4"]
    C["object"]
    A --> B
    B --> C
```
`(#(struct:a-part p2 #(4 6 5)) #(struct:a-part p1 #(3 4)))`

Para `o3 = new p3(6)`:
```mermaid
flowchart LR
    A["c3
    f = 6, g = 7"]
    B["c2
    c = 7, d = 9, e = 8"]
    C["c1
    a = 6, b = 7"]
    D["object"]
    A --> B
    B --> C
    C --> D
```
`(#(struct:a-part p3 #(6 7)) #(struct:a-part p2 #(7 9 8)) #(struct:a-part p1 #(6 7)))`

####
**Ejecución de métodos y encapsulación**

Cuando se invoca un método, se genera un diagrama de ambientes independiente. Los llamados no se afectan entre sí, a menos que se modifiquen los campos del objeto.

Llamado `send o1 sum(2)`:
```mermaid
flowchart LR
    A["empty-env"]
    B["env1
    a, b: 2, 3"]
    C["envsum
    x, super, self: 2, object, o1"]
    A --> B
    B --> C
```

Sobre el ambiente `envsum` se ejecuta:
```racket
begin
  set a = +(a, 1);
  +(a, b, x, send self m1())
end
```

Al hacer `set a = +(a, 1)`, cambia la representación del objeto y el ambiente `env1`:
```mermaid
flowchart LR
    A["c1
    a = 3, b = 3"]
    B["object"]
    A --> B
```

```mermaid
flowchart LR
    A["empty-env"]
    B["env1
    a, b: 3, 3"]
    C["envsum
    x, super, self: 2, object, o1"]
    A --> B
    B --> C
```

Al ejecutar `+(a, b, x, send self m1())`, se evalúa `+(3, 3, 2, send o1 m1())`. El llamado `send o1 m1()` genera:
```mermaid
flowchart LR
    A["empty-env"]
    B["env1
    a, b: 3, 3"]
    C["envm1
    super, self: object, o1"]
    A --> B
    B --> C
```
El resultado es `+(3, 3, 2, 1) = 9`.

####
**Ejecución del método sum en o2**

Para el llamado `send o2 sum(3)`:
```mermaid
flowchart LR
    A["empty-env"]
    B["env1
    a, b: 3, 4"]
    C["env2
    c, d, e: 4, 6, 5"]
    D["envsum
    s, super, self: 3, p1, o2"]
    A --> B
    B --> C
    C --> D
```

Se ejecuta en el ambiente `envsum`:
```racket
+(c, d, e, send self m1())
```
Lo que equivale a `+(4, 6, 5, send o2 m1())`. Para `send o2 m1()`:
```mermaid
flowchart LR
    A["empty-env"]
    B["env1
    a, b: 3, 4"]
    C["envm1
    super, self: p1, o2"]
    A --> B
    B --> C
```
Se evalúa la expresión `1` (proveniente de `p1`). El resultado es `+(4, 6, 5, 1) = 16`.

####
**Ejecución del método sum en o3 con super**

Para el llamado `send o3 sum(4)`, el diagrama de ambientes es:
```mermaid
flowchart LR
    A["empty-env"]
    B["env1
    a, b: 6, 7"]
    C["env2
    c, d, e: 7, 9, 8"]
    D["env3
    f, g: 6, 7"]
    E["envsum
    a, super, self: 4, p2, o3"]
    A --> B
    B --> C
    C --> D
    D --> E
```

Se ejecuta el cuerpo del método definido en `p3`:
```racket
+(a, super sum(+(a, 2)))
```
Equivale a `+(4, super sum(6))`. El llamado `super sum(6)` invoca el método de `p2`:
```mermaid
flowchart LR
    A["empty-env"]
    B["env1
    a, b: 6, 7"]
    C["env2
    c, d, e: 7, 9, 8"]
    D["envsum
    s, super, self: 6, p1, o3"]
    A --> B
    B --> C
    C --> D
```

Sobre este `envsum` se evalúa `+(c, d, e, send self m1())`, es decir, `+(7, 9, 8, send o3 m1())`. Para `send o3 m1()`:
```mermaid
flowchart LR
    A["empty-env"]
    B["env1
    a, b: 6, 7"]
    C["env2
    c, d, e: 7, 9, 8"]
    D["env3
    f, g: 6, 7"]
    E["envm1
    super, self: p2, o3"]
    A --> B
    B --> C
    C --> D
    D --> E
```
Bajo `envm1` se ejecuta el método `m1` de `p3`, que retorna `10`.
Evaluación:
- `+(7, 9, 8, 10) = 34`
- `+(4, 34) = 38`
