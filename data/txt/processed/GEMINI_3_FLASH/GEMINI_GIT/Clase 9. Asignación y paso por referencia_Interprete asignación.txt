####
**Especificación léxica del intérprete**

La especificación léxica define los tokens y patrones mediante expresiones regulares para el lenguaje. Se ignoran los comentarios iniciados con `%` y los espacios en blanco.

```racket
(define lexical
  '((comment ("%" (arbno (not #\newline))) skip) ; Comentarios: % hasta newline (se ignoran)
    (whitespace (whitespace) skip)               ; Espacios en blanco (se ignoran)
    (number (digit (arbno digit)) number)        ; Números positivos: dígitos
    (number ("-" digit (arbno digit)) number)    ; Números negativos: - seguido de dígitos
    (identifier (letter (arbno (or letter digit))) symbol) ; Identificadores: letra seguida de letras/dígitos
    ))
```

####
**Gramática para el parser y construcción del AST**

La gramática define las producciones y los constructores para el Árbol de Sintaxis Abstracta (`AST`). Incluye expresiones para variables, literales, booleanos, primitivas, condicionales `if`, ligaduras `let`, procedimientos `proc`, recursión `letrec`, asignación `set` y secuenciación `begin`.

```racket
(define grammar
  '((program (expression) a-program) ; Programa: una expresión
    (expression (identifier) var-exp) ; Expresión: variable
    (expression (number) lit-exp)     ; Expresión: literal numérico
    (expression ("true") true-exp)    ; Expresión: booleano true
    (expression ("false") false-exp)  ; Expresión: booleano false
    (expression (primitive "(" (separated-list expression ",") ")") prim-exp) ; Expresión primitiva
    (expression ("if" expression "then" expression "else" expression) if-exp)
    (expression ("let" (arbno identifier "=" expression) "in" expression) let-exp)
    (expression ("proc" "(" (separated-list identifier ",") ")") proc-exp)
    (expression ("letrec" (arbno identifier "(" (separated-list identifier ",") ")" "=" expression) "in" expression) letrec-exp)
    (expression ("set" identifier "=" expression) set-exp)
    (expression ("begin" expression (arbno ";" expression) "end") begin-exp)
    (expression ("(" expression (arbno expression) ")") app-exp)
    (primitive ("+") add-prim)    ; Primitiva: suma
    (primitive ("-") sub-prim)    ; Primitiva: resta
    (primitive ("*") prod-prim)   ; Primitiva: multiplicación
    (primitive ("/") div-prim)    ; Primitiva: división
    (primitive ("and") and-prim)  ; Primitiva: and lógico
    (primitive ("or") or-prim)    ; Primitiva: or lógico
    (primitive ("<") less-prim)   ; Primitiva: menor que
    (primitive ("<=") lesseq-prim); Primitiva: menor o igual
    (primitive (">") more-prim)   ; Primitiva: mayor que
    (primitive (">=") moreeq-prim); Primitiva: mayor o igual
    (primitive ("==") eq-prim)    ; Primitiva: igualdad
    (primitive ("!=") neq-prim)   ; Primitiva: desigualdad
    ))
```

####
**Generación de datatypes, Scanner y Parser**

Se utilizan las funciones de `sllgen` para generar automáticamente los tipos de datos a partir de la gramática y crear las funciones de análisis léxico y sintáctico.

```racket
;; Generar datatypes automáticamente a partir de lexical/grammar
(sllgen:make-define-datatypes lexical grammar)

; Scanner: convierte string en lista de tokens
; scanner -> string -> lista de tokens
(define scanner
  (lambda (program)
    ((sllgen:make-string-scanner lexical grammar) program)))

; Parser: convierte string en AST usando la gramática
; parser: string -> ast
(define parser
  (lambda (program)
    ((sllgen:make-string-parser lexical grammar) program)))
```

####
**Definición del tipo de dato environment**

El entorno (`environment`) se define como una estructura de datos que puede estar vacía, extendida con nuevos bindings o extendida recursivamente para soportar procedimientos. Los valores se almacenan en un `vector` para permitir la mutación requerida en la asignación.

```racket
(define-datatype environment environment?
  (empty-env) ; Entorno vacío
  (extend-env ; Extender entorno con nuevos bindings
   (lid (list-of symbol?)) ; Lista de identificadores
   (lval vector?)          ; Lista de valores (usando vectores para mutación)
   (old-env environment?)) ; Entorno anterior
  (extend-recursively-env
   (procname (list-of symbol?))
   (argss (list-of (list-of symbol?)))
   (bodies (list-of expression?))
   (old-env environment?)))

; Predicado value?: cualquier valor es válido (implementación simplificada)
(define value?
  (lambda (v) #t))
```

####
**Aplicación del entorno y búsqueda de referencias**

Las funciones `apply-env` y `apply-env-ref` permiten buscar valores o referencias en el entorno. La búsqueda es recursiva y retorna una referencia (`a-ref`) que apunta a una posición dentro de un vector de valores.

```racket
; Aplicar entorno: busca un símbolo en el entorno y retorna su valor
; apply-env: environment -> valor
(define apply-env
  (lambda (env var)
    (deref (apply-env-ref env var))))

; apply-env-ref: environment -> referencia
(define apply-env-ref
  (lambda (env var)
    (cases environment env
      (empty-env ()
        (eopl:error "Variable not found" var)) ; Error si no se encuentra
      (extend-env (lid vec old-env)
        (letrec
            ((search-var
              (lambda (lid lval pos)
                (cond
                  [(null? lid) (apply-env-ref old-env var)] ; Buscar en entorno anterior
                  [(equal? (car lid) var) (a-ref pos vec)] ; Generar referencia al encontrar la variable
                  [else (search-var (cdr lid) lval (+ pos 1))]))))
          (search-var lid vec 0)))
      (extend-recursively-env (procnames llargs bodies old-env)
        (letrec
            ((search-proc
              (lambda (procs args bodies)
                (cond
                  [(null? procs) (apply-env-ref old-env var)]
                  [(eqv? (car procs) var)
                   (a-ref 0 (list->vector (list (closure (car args) (car bodies) env))))]
                  [else (search-proc (cdr procs) (cdr args) (cdr bodies))]))))
          (search-proc procnames llargs bodies))))))
```

####
**Evaluación del programa y entorno inicial**

`eval-program` actúa como el punto de entrada para la ejecución del `AST`. El `init-env` define un conjunto de variables predefinidas con sus respectivos valores.

```racket
; Evaluar programa: punto de entrada, evalúa la expresión del programa
; eval-program: program -> value
(define eval-program
  (lambda (pgm)
    (cases program pgm
      (a-program (exp)
                 (eval-expression exp init-env)))))

; Entorno inicial predefinido (variables x,y,z y a,b,c con valores numéricos)
(define init-env
  (extend-env
   '(x y z)
   (list->vector '(1 2 3))
   (extend-env
    '(a b c)
    (list->vector '(4 5 6))
    (empty-env))))
```

####
**Evaluación de expresiones (eval-expression)**

La función `eval-expression` procesa recursivamente los diferentes tipos de expresiones del lenguaje, incluyendo literales, variables, primitivas, condicionales y aplicaciones de procedimientos.

```racket
; Evaluar expresión: recursivamente evalúa según el tipo de expresión
; eval-expression: expression -> value
(define eval-expression
  (lambda (exp env)
    (cases expression exp
      (var-exp (id) (apply-env env id)) ; Variable: busca en entorno
      (lit-exp (datum) datum)           ; Literal: retorna el número
      (true-exp () #t)                  ; True: retorna #t
      (false-exp () #f)                 ; False: retorna #f
      (prim-exp (prim lrands)           ; Primitiva: evalúa argumentos y aplica
                (let ((values (map (lambda (x) (eval-expression x env)) lrands)))
                  (apply-primitive prim values)))
      (if-exp (cond-exp true-exp false-exp)
              (let ((cond-value (eval-expression cond-exp env)))
                (if (boolean? cond-value)
                    (if cond-value
                        (eval-expression true-exp env)
                        (eval-expression false-exp env))
                    (eopl:error "The condition must be a boolean " cond-exp))))
      (let-exp (lid lexpr expr)
               (let ((vexpr (map (lambda (x) (eval-expression x env)) lexpr)))
                 (eval-expression expr (extend-env lid (list->vector vexpr) env))))
      (proc-exp (lid exp) (closure lid exp env))
      (app-exp (rator rands)
               (let ((procv (eval-expression rator env))
                     (vrands (map (lambda (x) (eval-expression x env)) rands)))
                 (if (and (procVal? procv)
                          (= (length (procVal->lid procv)) (length vrands)))
                     (cases procVal procv
                       (closure (lid exp old-env)
                                (eval-expression exp (extend-env lid (list->vector vrands) old-env))))
                     (eopl:error "Not a procedure or incorrect of number of args"))))
      (letrec-exp (procnames llargs bodies expn)
                  (eval-expression expn (extend-recursively-env procnames llargs bodies env)))
      (begin-exp (exp lexp)
                 (letrec ((evaluation (lambda (lexp)
                                        (cond
                                          [(null? (cdr lexp)) (eval-expression (car lexp) env)]
                                          [else (begin (eval-expression (car lexp) env)
                                                       (evaluation (cdr lexp)))]))))
                   (evaluation (cons exp lexp))))
      (set-exp (id exp)
               (begin
                 (setref! (apply-env-ref env id) (eval-expression exp env))
                 (void)))
      (else "Not implemented yet"))))
```

####
**Implementación de operaciones primitivas**

Las operaciones primitivas se ejecutan aplicando funciones binarias acumulativas sobre una lista de valores mediante el procedimiento genérico `operation`.

```racket
; Operación genérica: aplica una función binaria acumulativa a una lista
; operation: (T,T)->T, lista de valores, T -> value
(define operation
  (lambda (f lst acc)
    (cond
      [(null? lst) acc] ; Caso base: retorna acumulador
      [else (operation f (cdr lst) (f (car lst) acc))])))

; Aplicar primitiva: ejecuta la operación correspondiente sobre los valores
(define apply-primitive
  (lambda (prim values)
    (cases primitive prim
      (add-prim () (operation + values 0))
      (sub-prim () (- (car values) (operation + (cdr values) 0)))
      (prod-prim () (operation * values 1))
      (div-prim () (/ (car values) (operation * (cdr values) 1)))
      (and-prim () (operation (lambda (a b) (and a b)) values #t))
      (or-prim () (operation (lambda (a b) (or a b)) values #f))
      (less-prim () (< (car values) (cadr values)))
      (lesseq-prim () (<= (car values) (cadr values)))
      (more-prim () (> (car values) (cadr values)))
      (moreeq-prim () (>= (car values) (cadr values)))
      (eq-prim () (= (car values) (cadr values)))
      (neq-prim () (not (= (car values) (cadr values)))))))
```

####
**Representación de procedimientos (Closures)**

Los procedimientos se representan mediante `closures`, que almacenan la lista de identificadores, el cuerpo de la expresión y el entorno en el momento de su definición.

```racket
; Representación de los procedimientos
(define-datatype procVal procVal?
  (closure
   (lid (list-of symbol?))
   (exp expression?)
   (old-env environment?)))

;; Extractor para lid en un closure
(define procVal->lid
  (lambda (cls)
    (cases procVal cls
      (closure (lid exp old-env) lid))))
```

####
**Manejo de referencias y mutación**

Para permitir la asignación de variables, el intérprete utiliza `referencias` que vinculan una posición específica con un `vector`. Esto permite que el valor almacenado cambie sin necesidad de reconstruir el entorno.

```racket
;; References
(define-datatype reference reference?
  (a-ref
   (position integer?)
   (vec vector?)))

;; deref: reference -> value
(define deref
  (lambda (ref)
    (primitive-deref ref)))

; primitive-deref: reference -> valor
(define primitive-deref
  (lambda (ref)
    (cases reference ref
      (a-ref (pos vec)
             (vector-ref vec pos)))))

; setref! es un procedimiento para cambiar el valor de una referencia
(define setref!
  (lambda (ref val)
    (primitive-setref! ref val)))

(define primitive-setref!
  (lambda (ref val)
    (cases reference ref
      (a-ref (pos vec)
             (vector-set! vec pos val)))))

;; Definición del valor nulo
(define-datatype nulltype nulltype?
  (void))
```
