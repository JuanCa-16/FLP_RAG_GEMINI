####
**Definición gramatical de ambientes**

La estructura de un `environment` (ambiente) se define mediante la siguiente gramática:

<environment> ::= '()
`empty-env()`
::= <symbol>* <values>* <environment>
`extend-env(lid lval old-env)`

Esta gramática especifica que un ambiente puede ser vacío o una extensión que contiene una lista de identificadores, una lista de valores y un ambiente previo.

####
**Implementación del tipo de dato environment en EOPL**

Utilizando la interfaz de programación de `eopl`, se define el tipo de dato `environment` y un predicado auxiliar `value?`.

```racket
#lang eopl

(define-datatype environment environment?
  (empty-env)
  (extend-env
    (lid (list-of symbol?))
    (lval (list-of value?))
    (old-env environment?)))

(define value?
  (lambda (x)
    #t))
```

El tipo de dato `environment` cuenta con dos variantes: `empty-env` para el ambiente vacío y `extend-env` para representar la vinculación de nombres y valores. El predicado `value?` se define para aceptar cualquier valor.

####
**Creación de un ambiente de ejemplo**

Se define un ambiente `env1` que contiene múltiples niveles de anidamiento utilizando la función `extend-env`.

```racket
(define env1
  (extend-env
    '(x y z)
    '(1 2 3)
    (extend-env
      '(a b c)
      '(4 5 6)
      (empty-env))))
```

En este ejemplo, `env1` vincula los símbolos `x`, `y`, `z` a los valores `1`, `2`, `3` respectivamente, sobre un ambiente previo que vincula `a`, `b`, `c` a `4`, `5`, `6`.

####
**Procedimiento para la búsqueda en ambientes**

La función `apply-env` permite recuperar el valor asociado a una variable dentro de un ambiente dado.

```racket
;;apply-env: environment -> value
(define apply-env
  (lambda (env var)
    (cases environment env
      (empty-env ()
        (eopl:error "No encuentro a " var))
      (extend-env (lid lval old-env)
        (letrec
          ((search-var
             (lambda (li lv)
               (cond
                 [(null? li) (apply-env old-env var)]
                 [(equal? (car li) var) (car lv)]
                 [else (search-var (cdr li) (cdr lv))]))))
          (search-var lid lval))))))
```

El procedimiento utiliza `cases` para determinar si el ambiente está vacío o extendido. En un `extend-env`, se emplea una función recursiva local `search-var` para buscar el símbolo en la lista actual de identificadores; si no se encuentra, la búsqueda continúa recursivamente en el ambiente previo `old-env`.

####
**Pruebas de recuperación de valores en el ambiente**

A continuación se presentan las llamadas para visualizar los valores asociados a diferentes símbolos dentro del ambiente `env1`.

```racket
(display (apply-env env1 'x))
(display "\n")
(display (apply-env env1 'y))
(display "\n")
(display (apply-env env1 'z))
(display "\n")
(display (apply-env env1 'a))
(display "\n")
(display (apply-env env1 'b))
(display "\n")
(display (apply-env env1 'c))
(display "\n")
(display (apply-env env1 'w))
```

Estas instrucciones buscan los símbolos `x`, `y`, `z`, `a`, `b`, `c` y `w` en `env1`. La búsqueda de un símbolo que no existe en el ambiente, como `w`, resultará en un error generado por `empty-env`.
