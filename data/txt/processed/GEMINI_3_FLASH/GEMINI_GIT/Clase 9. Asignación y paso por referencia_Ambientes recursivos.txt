####
**Ambientes recursivos**
Los `ambientes recursivos` son estrictamente para `procedimientos`, no vamos a permitir modificaciones a sus valores (es posible pero requiere un proceso en el cual debemos validar que sean `procedimientos`).

####
**Implementación de apply-env-ref**
```racket
; apply-env: environment -> referencia
(define apply-env-ref
  (lambda (env var)
    (cases environment env
      (empty-env ()
        (eopl:error "Variable not found" var)) ; Error si no se encuentra
      (extend-env (lid vec old-env)
        (letrec
          ((search-var ; Búsqueda recursiva en la lista de bindings
            (lambda (lid lval [pos 0])
              (cond
                [(null? lid) (apply-env-ref old-env var)] ; Buscar en entorno anterior
                [(equal? (car lid) var) (a-ref pos vec)] ; Voy a generar una referencia cuando encuentro la variable
                [else (search-var (cdr lid) vec (+ pos 1))])))) ; Continuar búsqueda
          (search-var lid vec)))
      (extend-recursively-env (procnames llargs bodies old-env)
        (letrec
          ((search-proc
            (lambda (procs args bodies)
              (cond
                [(null? procs) (apply-env-ref old-env var)]
                [(eqv? (car procs) var)
                 (a-ref 0
                   (list->vector
                     (list
                       (closure (car args) (car bodies) env))))]
                [else (search-proc (cdr procs) (cdr args) (cdr bodies))]))))
          (search-proc procnames llargs bodies))))))
```
La función `apply-env-ref` permite buscar una variable en un `environment` para obtener su referencia. En el caso de `extend-recursively-env`, se realiza una búsqueda del nombre del procedimiento y, al encontrarlo, se genera una `clausura` contenida en una referencia.

####
**Retorno de clausuras en ambientes recursivos**
En esta modificación al consultar el procedimiento, vamos a retornar una referencia con un único valor que es una `clausura`:

```racket
(a-ref 0
  (list->vector
    (list
      (closure (car args) (car bodies) env))))
```
Este fragmento ilustra cómo se construye la referencia `a-ref` que encapsula el `vector` con la `clausura` del procedimiento encontrado durante la búsqueda en el ambiente recursivo.
