####
**Introducción a los procedimientos en el lenguaje de programación**

Hasta este punto, el lenguaje de programación desarrollado cuenta con operaciones matemáticas (sumas, restas, multiplicaciones), booleanos para el manejo de condicionales y ligaduras locales. Estas últimas se denominan ligaduras porque, por ahora, no pueden cambiar su valor.

El siguiente paso en la extensión del interpretador es agregar `procedimientos`. Los procedimientos o funciones son fundamentales en la computación y en los lenguajes de programación, ya que permiten implementar tareas que se repiten múltiples veces mediante su invocación o llamado. Para incluir esta funcionalidad, se requieren dos componentes:

1. La creación del procedimiento: Se utiliza la sintaxis `proc`, seguida de las variables (por ejemplo, `x, y, z`) y el cuerpo que define la acción del procedimiento.
2. El llamado o aplicación: La ejecución de un procedimiento previamente creado.

####
**Especificación gramatical para procedimientos**

Para implementar los procedimientos, se realizan cambios en la gramática del lenguaje. Se definen dos estructuras principales: la creación (`proc-exp`) y la aplicación (`app-exp`).

**Gramática de la creación (proc)**
La sintaxis para definir un procedimiento es `proc`, seguido de una lista de identificadores entre paréntesis separados por comas y, finalmente, una expresión que representa el cuerpo.
- Notación: `proc` `(` (separados por comas `identifier`*) `)` `expression`
- En la implementación se denomina `proc-exp`. La lista de identificadores utiliza una cerradura de Kleene, lo que permite cero o más parámetros.

**Gramática de la aplicación (llamado)**
El llamado se define mediante la apertura de un paréntesis, seguido de una expresión que actúa como el `rator` (operador/procedimiento) y una lista de expresiones que actúan como los `rants` (operandos/argumentos).
- Notación: `(` `expression` `expression`* `)`
- En la implementación se denomina `app-exp`. La primera expresión es obligatoria y debe evaluar a un procedimiento.

####
**Evaluación de procedimientos y ejemplos**

Al evaluar un procedimiento, los argumentos de la invocación se vinculan con las variables definidas en la creación.

**Ejemplo 1**
Si definimos `f = proc(y, z) { + y, (- z, 5) }`, la evaluación de `f(2, 28)` se realiza asignando `y = 2` y `z = 28`:
- Expresión: `2 + (28 - 5)`
- Resultado: `2 + 23 = 25`

**Ejemplo 2**
Si definimos `f = proc(z) { * z, 2 }` y una función `y = proc(x, y) { + x, y }`, el llamado `y(f(4), f(3))` se resuelve evaluando primero los argumentos internos:
- `f(4)` resulta en `8`.
- `f(3)` resulta en `6`.
- El llamado final es `y(8, 6)`, que resulta en `14`.

Es importante recordar que en el proceso de evaluación se resuelven primero los paréntesis internos y luego los externos.

####
**El concepto de clausura y el alcance léxico**

En el paradigma funcional, se espera que si una función se llama con los mismos argumentos en cualquier parte del código, devuelva el mismo resultado. Para garantizar esto, es necesario manejar correctamente el ambiente donde se crea la función.

Considere el siguiente código:
```
let x = 5
in let f = proc(y, z) { + y, (- z, x) }
   in let x = 7
      in (f 2 28)
```
Aquí surge la pregunta: ¿qué valor debe tomar `x` dentro de `f`, el valor original (5) o el nuevo valor (7)? Para cumplir con la transparencia referencial, el valor correcto de `x` es aquel que existía en el momento en que se creó el procedimiento (en este caso, 5).

Para resolver esto y evitar ambigüedades, se introduce la `clausura` (o `closure`). Una clausura es la representación de un procedimiento en memoria y se encarga de almacenar tres elementos clave:
1. Los identificadores (`ids`) del procedimiento.
2. El cuerpo del procedimiento.
3. El ambiente donde fue creado el procedimiento.

Al guardar el ambiente de creación, la función "recuerda" los valores de las variables externas en ese momento, independientemente de dónde sea invocada posteriormente.

####
**Implementación de la clausura en el interpretador**

Para representar los procedimientos en el interpretador, se define un nuevo tipo de dato llamado `probal` (valor de procedimiento) que implementa la `clausura`.

**Definición del Data Type**
En Racket, la estructura de la clausura se define de la siguiente manera:

```racket
(define-datatype probal probal?
  (closure
    (ids (list-of symbol?))
    (body expression?)
    (env environment?)))
```

Esta estructura asegura que el procedimiento mantenga la lista de símbolos (parámetros), la expresión que constituye su cuerpo y el ambiente (`env`) necesario para su evaluación bajo alcance léxico.

####
**Modificaciones en evaluar-expresion para procedimientos**

Para procesar la creación de procedimientos en la función `evaluar-expresion`, se debe gestionar el caso de `proc-exp`.

```racket
;; Fragmento de lógica para evaluar-expresion
(define evaluar-expresion
  (lambda (exp env)
    (cases expression exp
      (proc-exp (ids body)
                (closure ids body env))
      (app-exp (rator rands)
               (let ((proc (evaluar-expresion rator env))
                     (args (map (lambda (x) (evaluar-expresion x env)) rands)))
                 (if (probal? proc)
                     (aplicar-procedimiento proc args)
                     (eopl:error "No es un procedimiento"))))
      ;; ... otros casos
      )))
```

Cuando se encuentra un `proc-exp`, el interpretador simplemente crea una `clausura` capturando los identificadores, el cuerpo y el ambiente actual. Por otro lado, `app-exp` se encarga de la aplicación del procedimiento, evaluando primero el operador (`rator`) y los argumentos (`rands`).

####
**Representación de la clausura en el ambiente**

Cuando se crea un procedimiento, la clausura resultante almacena el estado del ambiente en ese instante. Por ejemplo, en el siguiente código:

```
let a = proc(x, y) { x }
```

La representación interna de la clausura mostraría:
- `ids`: `(x, y)`
- `body`: `x`
- `env`: Una estructura que contiene las variables del ambiente inicial y cualquier ligadura previa (como `z = 425`).

Si el ambiente ha sido extendido múltiples veces, la clausura guardará toda esa cadena de referencias. Esto garantiza que, al ejecutar el procedimiento, este tenga acceso a todas las variables que eran visibles en su punto de definición. Es fundamental comprender que la evaluación de un procedimiento se realiza siempre a partir del ambiente guardado en su clausura.
